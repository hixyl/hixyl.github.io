4:T2621,<p>为了彻底理清 JavaScript 的集合体系，我们需要从最底层的<strong>内存管理机制</strong>出发。无论是 <code>Map</code> 还是 <code>Set</code>，它们最核心的区别不在于 API 的好用程度，而在于它们如何与系统的**垃圾回收（Garbage Collection, GC）**对话。</p>
<hr>
<h2 id="第一部分核心底层强引用-vs-弱引用"><a href="#第一部分核心底层强引用-vs-弱引用">第一部分：核心底层——强引用 vs 弱引用</a></h2>
<p>在 JavaScript 中，内存管理是自动的，但它是基于“可达性”逻辑的。</p>
<h3 id="1-强引用-strong-reference"><a href="#1-强引用-strong-reference">1. 强引用 (Strong Reference)</a></h3>
<p>这是 JS 的默认行为。只要一个变量指向一个对象，这个对象就是“可达”的，垃圾回收器绝不会碰它。</p>
<ul>
<li>
<p><strong>逻辑</strong>：<code>A -> Object</code>。只要 A 还在，Object 就在。</p>
</li>
<li>
<p><strong>代价</strong>：如果你忘记手动将 <code>A = null</code>，Object 会永远占用内存，这就是<strong>内存泄漏</strong>的根源。</p>
</li>
</ul>
<h3 id="2-弱引用-weak-reference"><a href="#2-弱引用-weak-reference">2. 弱引用 (Weak Reference)</a></h3>
<p>弱引用不会被垃圾回收器计入。它像是一个“旁观者”。</p>
<ul>
<li>
<p><strong>逻辑</strong>：<code>A --(weak)--> Object</code>。如果没有其他<strong>强引用</strong>指向这个 Object，即便 A 还在看着它，垃圾回收器也会在下次清理时直接把 Object 收走。</p>
</li>
<li>
<p><strong>代价</strong>：你无法遍历弱引用的集合，因为对象可能随时消失，导致结果不可预测。</p>
</li>
</ul>
<hr>
<h2 id="第二部分map-与-weakmap-的深度梳理"><a href="#第二部分map-与-weakmap-的深度梳理">第二部分：Map 与 WeakMap 的深度梳理</a></h2>
<p>你之前的疑惑集中在：<strong>既然逻辑一样，为什么要用 WeakMap？</strong></p>
<h3 id="1-场景回顾私有属性与-this"><a href="#1-场景回顾私有属性与-this">1. 场景回顾：私有属性与 <code>this</code></a></h3>
<p>在 <code>class</code> 或 <code>function</code> 中，我们利用 <code>WeakMap</code> 实现私有化。</p>
<ul>
<li><strong>代码示例（Function 写法）：</strong></li>
</ul>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">const</span> privates = <span class="hljs-keyword">new</span> <span class="hljs-title class_">WeakMap</span>();

<span class="hljs-keyword">function</span> <span class="hljs-title function_">User</span>(<span class="hljs-params">name</span>) {
  <span class="hljs-comment">// 这里的 this 指向 new 出来的实例对象</span>
  <span class="hljs-comment">// 我们把实例对象作为 Key，把私有数据作为 Value</span>
  privates.<span class="hljs-title function_">set</span>(<span class="hljs-variable language_">this</span>, { <span class="hljs-attr">name</span>: name }); 
}

<span class="hljs-title class_">User</span>.<span class="hljs-property"><span class="hljs-keyword">prototype</span></span>.<span class="hljs-property">getName</span> = <span class="hljs-keyword">function</span>(<span class="hljs-params"></span>) {
  <span class="hljs-keyword">return</span> privates.<span class="hljs-title function_">get</span>(<span class="hljs-variable language_">this</span>).<span class="hljs-property">name</span>;
};
</code></pre>
<ul>
<li>
<p><strong>为什么要用 WeakMap 而不是 Map？</strong></p>
<p>如果用 <code>Map</code>，当你执行 <code>user = null</code> 试图销毁用户时，<code>Map</code> 依然强引用着 <code>user</code> 实例作为 Key。结果是：<code>user</code> 实例和它的私有数据 <code>{ name }</code> 都会残留在内存中。</p>
<p>使用 <code>WeakMap</code>，当 <code>user = null</code> 后，唯一的强引用断开，<code>WeakMap</code> 里的这条记录会被 GC 自动抹除。</p>
</li>
</ul>
<h3 id="2-场景回顾dom-节点元数据"><a href="#2-场景回顾dom-节点元数据">2. 场景回顾：DOM 节点元数据</a></h3>
<ul>
<li>
<p><strong>逻辑</strong>：给 DOM 元素“贴标签”。</p>
</li>
<li>
<p><strong>优势</strong>：当 DOM 元素从页面移除（例如 Vue/React 组件卸载）时，<code>WeakMap</code> 里的标签会自动消失，无需手动清理 <code>Map.delete(el)</code>。</p>
</li>
</ul>
<hr>
<h2 id="第三部分set-与-weakset-的系统梳理"><a href="#第三部分set-与-weakset-的系统梳理">第三部分：Set 与 WeakSet 的系统梳理</a></h2>
<p><code>Set</code> 系列处理的是<strong>值的集合</strong>（类似数组，但值唯一），而不是键值对。</p>
<h3 id="1-set唯一值的容器"><a href="#1-set唯一值的容器">1. Set：唯一值的容器</a></h3>
<p><code>Set</code> 是强引用的，可以存储任何类型的值。</p>
<ul>
<li><strong>用法</strong>：去重、判断是否存在。</li>
</ul>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">const</span> tags = <span class="hljs-keyword">new</span> <span class="hljs-title class_">Set</span>([<span class="hljs-string">'js'</span>, <span class="hljs-string">'css'</span>, <span class="hljs-string">'js'</span>]); <span class="hljs-comment">// 结果只有 'js' 和 'css'</span>
tags.<span class="hljs-title function_">add</span>(<span class="hljs-string">'html'</span>);
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(tags.<span class="hljs-title function_">has</span>(<span class="hljs-string">'js'</span>)); <span class="hljs-comment">// true</span>
</code></pre>
<h3 id="2-weakset对象的临时通行证"><a href="#2-weakset对象的临时通行证">2. WeakSet：对象的“临时通行证”</a></h3>
<p><code>WeakSet</code> 只能存储<strong>对象</strong>，且对这些对象是弱引用。</p>
<ul>
<li>
<p><strong>底层逻辑</strong>：它不存储值，它只存储“对象的引用”。如果对象在外部被销毁，它在 <code>WeakSet</code> 里也会消失。</p>
</li>
<li>
<p><strong>特性</strong>：不可遍历，没有 <code>size</code>。</p>
</li>
</ul>
<h3 id="3-weakset-的典型用处防重入与打标签"><a href="#3-weakset-的典型用处防重入与打标签">3. WeakSet 的典型用处：防重入与打标签</a></h3>
<p>假设你有一个复杂的函数，你希望确保某些对象只被处理一次，且不干扰它们的回收。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">const</span> processedObjects = <span class="hljs-keyword">new</span> <span class="hljs-title class_">WeakSet</span>();

<span class="hljs-keyword">function</span> <span class="hljs-title function_">complexProcess</span>(<span class="hljs-params">obj</span>) {
  <span class="hljs-keyword">if</span> (processedObjects.<span class="hljs-title function_">has</span>(obj)) {
    <span class="hljs-keyword">return</span>; <span class="hljs-comment">// 如果已经处理过，直接跳过</span>
  }
  
  <span class="hljs-comment">// 执行复杂的业务逻辑...</span>
  <span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">"正在处理对象..."</span>, obj);
  
  <span class="hljs-comment">// 处理完打个标签</span>
  processedObjects.<span class="hljs-title function_">add</span>(obj);
}

<span class="hljs-keyword">let</span> data = { <span class="hljs-attr">id</span>: <span class="hljs-number">1</span> };
<span class="hljs-title function_">complexProcess</span>(data); <span class="hljs-comment">// 处理</span>
<span class="hljs-title function_">complexProcess</span>(data); <span class="hljs-comment">// 自动跳过</span>

data = <span class="hljs-literal">null</span>; <span class="hljs-comment">// 当 data 设为 null，WeakSet 自动释放对原对象的引用</span>
</code></pre>

<h3 id="关键结论如何做选择"><a href="#关键结论如何做选择">关键结论：如何做选择？</a></h3>
<ol>
<li>
<p><strong>如果你需要处理字符串、数字等基础类型</strong>：只能选 <code>Map</code> 或 <code>Set</code>。</p>
</li>
<li>
<p><strong>如果你需要遍历、统计数量</strong>：只能选 <code>Map</code> 或 <code>Set</code>。</p>
</li>
<li>
<p><strong>如果你在处理 DOM 节点或类实例，且不希望干扰它们的生死（GC）</strong>：请务必选择 <code>WeakMap</code> 或 <code>WeakSet</code>。</p>
</li>
