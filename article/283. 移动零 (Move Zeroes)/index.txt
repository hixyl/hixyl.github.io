1c:T39cd,<h2 id="移动零-move-zeroes"><a href="#移动零-move-zeroes">移动零 (Move Zeroes)</a></h2>
<h3 id="题目描述"><a href="#题目描述">题目描述</a></h3>
<p>给定一个数组 <code>nums</code>，编写一个函数将所有 <code>0</code> 移动到数组的末尾，同时保持非零元素的相对顺序。</p>
<p><strong>请注意</strong>，必须在不复制数组的情况下原地对数组进行操作。</p>
<p><strong>示例 1:</strong></p>
<ul>
<li>
<p><strong>输入:</strong> <code>nums = [0,1,0,3,12]</code></p>
</li>
<li>
<p><strong>输出:</strong> <code>[1,3,12,0,0]</code></p>
</li>
</ul>
<p><strong>示例 2:</strong></p>
<ul>
<li>
<p><strong>输入:</strong> <code>nums = [0]</code></p>
</li>
<li>
<p><strong>输出:</strong> <code>[0]</code></p>
</li>
</ul>
<p><strong>提示:</strong></p>
<ul>
<li>
<p><code>1 &#x3C;= nums.length &#x3C;= 10^4</code></p>
</li>
<li>
<p><code>-2^31 &#x3C;= nums[i] &#x3C;= 2^31 - 1</code></p>
</li>
</ul>
<p><strong>进阶：</strong></p>
<p>你能尽量减少完成的操作次数吗？</p>
<hr>
<h3 id="解法核心双指针交换法"><a href="#解法核心双指针交换法">解法核心：双指针交换法</a></h3>
<p>要实现“原地”且“保持相对顺序”，同时满足进阶要求“尽量减少操作次数”，最有效的机制是设立两个指针，在一次遍历中完成非零元素的提拔和零元素的后置。</p>
<p>我们可以为这两个指针赋予具体的角色：</p>
<ol>
<li>
<p><strong><code>fast</code>（探测指针）</strong>：它的任务是无条件向前推进，扫描整个数组，寻找非零元素。</p>
</li>
<li>
<p><strong><code>slow</code>（锚点指针）</strong>：它的任务是标记下一个非零元素应该被存放的安全位置。</p>
</li>
</ol>
<p><strong>运行机制：</strong></p>
<p>当 <code>fast</code> 遇到 <code>0</code> 时，说明当前位置不需要保留，<code>fast</code> 继续前进，而 <code>slow</code> 停留在原地，它指向的正是这个 <code>0</code>（也就是一个“空出的坑位”）。</p>
<p>当 <code>fast</code> 继续扫描，遇到下一个非零元素时，直接将 <code>fast</code> 指向的非零元素与 <code>slow</code> 指向的 <code>0</code> 进行<strong>互换</strong>。换完之后，<code>slow</code> 向前推进一步，准备迎接下一个非零元素。</p>
<h3 id="javascript-完整实现"><a href="#javascript-完整实现">JavaScript 完整实现</a></h3>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-comment">/**
 * <span class="hljs-doctag">@param</span> {<span class="hljs-type">number[]</span>} <span class="hljs-variable">nums</span>
 * <span class="hljs-doctag">@return</span> {<span class="hljs-type">void</span>} Do not return anything, modify nums in-place instead.
 */</span>
<span class="hljs-keyword">var</span> moveZeroes = <span class="hljs-keyword">function</span>(<span class="hljs-params">nums</span>) {
    <span class="hljs-keyword">let</span> slow = <span class="hljs-number">0</span>;

    <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> fast = <span class="hljs-number">0</span>; fast &#x3C; nums.<span class="hljs-property">length</span>; fast++) {
        <span class="hljs-comment">// 当探测指针 fast 发现非零元素时，准备处理</span>
        <span class="hljs-keyword">if</span> (nums[fast] !== <span class="hljs-number">0</span>) {
            <span class="hljs-comment">// 核心优化：如果 fast 和 slow 指向同一个位置，说明还没有遇到过 0</span>
            <span class="hljs-comment">// 此时它们指向同一个非零元素，不需要发生无意义的原地自换</span>
            <span class="hljs-keyword">if</span> (fast !== slow) {
                <span class="hljs-comment">// 将探测到的非零元素，与 slow 停留位置的 0 进行交换</span>
                <span class="hljs-keyword">let</span> temp = nums[slow];
                nums[slow] = nums[fast];
                nums[fast] = temp;
            }
            <span class="hljs-comment">// 只要处理过非零元素，锚点 slow 就必须向前移动一位</span>
            slow++;
        }
    }
};
</code></pre>
<hr>
<h3 id="详细用例剖析"><a href="#详细用例剖析">详细用例剖析</a></h3>
<p>我们以最容易让人产生疑问的场景为例，逐步拆解 <code>nums = [0, 1, 0, 3, 12]</code> 的执行全过程。</p>
<p><strong>初始状态：</strong> <code>nums = [0, 1, 0, 3, 12]</code>，<code>slow = 0</code>，<code>fast = 0</code></p>
<ul>
<li>
<p><strong>第 1 轮 (<code>fast = 0</code>)</strong>：</p>
<ul>
<li>
<p><code>nums[0]</code> 是 <code>0</code>。</p>
</li>
<li>
<p><code>fast</code> 探测到 <code>0</code>，忽略。<code>slow</code> 留在索引 0，它现在标记着第一个 0 的位置。</p>
</li>
<li>
<p>数组状态：<code>[0, 1, 0, 3, 12]</code></p>
</li>
</ul>
</li>
<li>
<p><strong>第 2 轮 (<code>fast = 1</code>)</strong>：</p>
<ul>
<li>
<p><code>nums[1]</code> 是 <code>1</code>（非零）。</p>
</li>
<li>
<p>触发条件。此时 <code>fast(1) !== slow(0)</code>，执行交换：将 <code>nums[1]</code> 的 <code>1</code> 与 <code>nums[0]</code> 的 <code>0</code> 互换。</p>
</li>
<li>
<p>交换后，<code>slow</code> 前进到 1。</p>
</li>
<li>
<p>数组状态：<code>[1, 0, 0, 3, 12]</code></p>
</li>
</ul>
</li>
<li>
<p><strong>第 3 轮 (<code>fast = 2</code>)</strong>：</p>
<ul>
<li>
<p><code>nums[2]</code> 是 <code>0</code>。</p>
</li>
<li>
<p>忽略。<code>slow</code> 留在索引 1，它标记着当前的第一个 0。</p>
</li>
<li>
<p>数组状态：<code>[1, 0, 0, 3, 12]</code></p>
</li>
</ul>
</li>
<li>
<p><strong>第 4 轮 (<code>fast = 3</code>)</strong>：</p>
<ul>
<li>
<p><code>nums[3]</code> 是 <code>3</code>（非零）。</p>
</li>
<li>
<p>触发条件。此时 <code>fast(3) !== slow(1)</code>，执行交换：将 <code>nums[3]</code> 的 <code>3</code> 与 <code>nums[1]</code> 的 <code>0</code> 互换。</p>
</li>
<li>
<p>交换后，<code>slow</code> 前进到 2。</p>
</li>
<li>
<p>数组状态：<code>[1, 3, 0, 0, 12]</code></p>
</li>
</ul>
</li>
<li>
<p><strong>第 5 轮 (<code>fast = 4</code>)</strong>：</p>
<ul>
<li>
<p><code>nums[4]</code> 是 <code>12</code>（非零）。</p>
</li>
<li>
<p>触发条件。此时 <code>fast(4) !== slow(2)</code>，执行交换：将 <code>nums[4]</code> 的 <code>12</code> 与 <code>nums[2]</code> 的 <code>0</code> 互换。</p>
</li>
<li>
<p>交换后，<code>slow</code> 前进到 3。</p>
</li>
<li>
<p>数组状态：<code>[1, 3, 12, 0, 0]</code></p>
</li>
</ul>
</li>
</ul>
<p>遍历结束，任务完成。你会发现，<code>slow</code> 和 <code>fast</code> 之间的区域，永远都是被推平的 <code>0</code>。<code>fast</code> 就像推土机一样把非零元素往前扔，把 <code>0</code> 往后滚。</p>
<hr>
<h3 id="评估与进阶分析"><a href="#评估与进阶分析">评估与进阶分析</a></h3>
<ul>
<li>
<p><strong>时间复杂度：<code>O(n)</code></strong>。无论数组内容如何，<code>fast</code> 指针精确地遍历数组一次，时间开销随数组长度线性增长。</p>
</li>
<li>
<p><strong>空间复杂度：<code>O(1)</code></strong>。全程只借用了 <code>fast</code>、<code>slow</code> 和 <code>temp</code> 几个临时变量，没有占用额外的集合空间。</p>
</li>
</ul>
<p><strong>关于“尽量减少操作次数”的深度对比：</strong></p>
<p>存在另一种常见的双指针思路是“先覆盖，后补零”：即 <code>fast</code> 遇到非零就直接赋值给 <code>slow</code> (<code>nums[slow] = nums[fast]</code>)，遍历完后，把 <code>slow</code> 后面的位置全部强行填为 <code>0</code>。</p>
<p>为什么我们采用的<strong>交换法+优化</strong>更好？</p>
<ol>
<li>
<p><strong>如果数组是 <code>[0, 0, 0, ..., 1]</code></strong>：</p>
<ul>
<li>
<p>覆盖法：执行 1 次写入（将 1 写到头部），然后要在末尾执行大量补 0 的写入操作。</p>
</li>
<li>
<p>交换法：只在最后一步执行 1 次交换（包含极少的读写）。</p>
</li>
</ul>
</li>
<li>
<p><strong>如果数组是 <code>[1, 2, 3, 4, 5]</code>（没有 0）</strong>：</p>
<ul>
<li>
<p>覆盖法：执行 5 次自我写入覆盖。</p>
</li>
<li>
<p>交换法（带 <code>fast !== slow</code> 判断）：判断拦截了所有无意义的交换，写操作次数直接降为 0。</p>
</li>
</ul>
</li>
</ol>
<p>通过加入指针位置是否重叠的判断条件，我们在所有极端场景下都把内存写入操作压缩到了真正的物理极限，这直接回应了题目最后的进阶要求。</p>

<h2 id="核心质疑双指针交换逻辑的稳健性分析"><a href="#核心质疑双指针交换逻辑的稳健性分析"><strong>核心质疑：双指针交换逻辑的稳健性分析</strong></a></h2>
<p><strong>问题描述：</strong></p>
<p>在执行“移动零”的双指针算法过程中，是否存在这样一种中间状态：<strong>慢指针 (<code>slow</code>) 指向的是一个非零元素，而快指针 (<code>fast</code>) 指向的是另一个非零元素，且此时 <code>fast > slow</code>？</strong></p>
<p><strong>质疑点：</strong></p>
<p>如果上述状态成立，代码中的交换逻辑 <code>[nums[slow], nums[fast]] = [nums[fast], nums[slow]]</code> 就会演变为“两个非零元素之间的互换”。</p>
<p>由于算法的要求是“保持非零元素的相对顺序”，一旦发生这种非零元素间的乱序交换，是否会导致最终数组的相对顺序被破坏？请论证在算法运行的任何时刻，只要触发交换，<code>slow</code> 所指向的一定是 <code>0</code>。</p>
<h3 id=""></h3>
<p>这是一个非常棒的深度思考！你观察到了算法逻辑中最核心的 <strong>“不变性”（Invariant）</strong> 问题。</p>
<p>直接给出结论：<strong>在程序运行过程中，只要 <code>fast</code> 和 <code>slow</code> 不相等（即 <code>fast > slow</code>），<code>slow</code> 指向的位置【一定】是一个 0。</strong></p>
<p>所以，你担心的“两个非零数互换导致乱套”的情况，在逻辑上是被阻断的。我们来看看为什么这个结论成立：</p>
<h3 id="1-指针的分道扬镳点"><a href="#1-指针的分道扬镳点">1. 指针的“分道扬镳”点</a></h3>
<p>在数组开头阶段，如果全是非零数（比如 <code>[1, 2, 3...]</code>），<code>fast</code> 和 <code>slow</code> 是<strong>并肩前行</strong>的。</p>
<ul>
<li>
<p><code>fast</code> 扫到 1，<code>slow</code> 也指向 1，处理完后两人一起跳到下一个。</p>
</li>
<li>
<p>这时候 <code>fast === slow</code>，触发不了交换逻辑。</p>
</li>
</ul>
<p><strong>重点来了：</strong> 只有当 <code>fast</code> 遇到了数组中的<strong>第一个 0</strong>，两个人的节奏才会打乱。</p>
<ul>
<li>
<p><code>fast</code> 看到 0，不进 <code>if</code> 逻辑，直接 <code>fast++</code> 跑向下一个。</p>
</li>
<li>
<p><code>slow</code> 却被挡在了这个 0 的位置，<strong>原地待命</strong>。</p>
</li>
</ul>
<h3 id="2-为什么-slow-不可能指向非零"><a href="#2-为什么-slow-不可能指向非零">2. 为什么 <code>slow</code> 不可能指向非零？</a></h3>
<p>一旦两人分开了，<code>slow</code> 就变成了一个 <strong>“坑位守护者”</strong>。</p>
<ul>
<li>
<p>此时 <code>slow</code> 指向的是那个刚才被跳过的 <strong>0</strong>。</p>
</li>
<li>
<p><code>fast</code> 在前面继续找。直到 <code>fast</code> 找到下一个非零数（比如 3）。</p>
</li>
<li>
<p>此时执行交换：把 <code>fast</code> 指向的 3 和 <code>slow</code> 指向的 0 交换。</p>
</li>
<li>
<p><strong>交换后：</strong> 3 到了前面，而 <strong>0 被换到了 <code>fast</code> 原本的位置上</strong>。</p>
</li>
<li>
<p><strong>关键动作：</strong> <code>slow++</code>。</p>
</li>
</ul>
<p><strong>关键点：</strong> <code>slow</code> 只往后挪一格。这一格原本是什么？</p>
<p>因为在遇到第一个 0 之后，<code>fast</code> 扫过的路径上如果还有 0，这些 0 都会被留在原地。<code>slow</code> 挪到的下一格，要么是下一个原本就在那里的 0，要么是之前交换留下的 0。</p>
<h3 id="3-逻辑推演用你的疑问做测试"><a href="#3-逻辑推演用你的疑问做测试">3. 逻辑推演（用你的疑问做测试）</a></h3>
<p>假设数组是 <code>[1, 0, 0, 2, 3]</code>：</p>
<ol>
<li>
<p><code>fast=0, slow=0</code>: 指向 1。<code>fast === slow</code>，不交换。<code>slow++</code> 变成 1。</p>
</li>
<li>
<p><code>fast=1, slow=1</code>: 指向 0。<code>fast</code> 发现是 0，不处理。<code>slow</code> 停在索引 1（它是 0）。</p>
</li>
<li>
<p><code>fast=2, slow=1</code>: 指向 0。<code>fast</code> 发现是 0，不处理。<code>slow</code> 停在索引 1（它是 0）。</p>
</li>
<li>
<p><code>fast=3, slow=1</code>: 指向 2。<strong>发现非零！</strong> 且 <code>fast(3) !== slow(1)</code>。</p>
<ul>
<li>
<p><strong>交换：</strong> <code>nums[1]</code>(0) 和 <code>nums[3]</code>(2) 交换。</p>
</li>
<li>
<p><strong>结果：</strong> <code>[1, 2, 0, 0, 3]</code>。</p>
</li>
<li>
<p><strong>此时 <code>slow</code> 变成 2。</strong></p>
</li>
</ul>
</li>
<li>
<p><code>fast=4, slow=2</code>: 指向 3。<strong>发现非零！</strong> 且 <code>fast(4) !== slow(2)</code>。</p>
<ul>
<li>
<p><strong>交换：</strong> <code>nums[2]</code>(0) 和 <code>nums[4]</code>(3) 交换。</p>
</li>
<li>
<p><strong>结果：</strong> <code>[1, 2, 3, 0, 0]</code>。</p>
</li>
</ul>
</li>
</ol>

<h3 id="总结"><a href="#总结">总结</a></h3>
<p>你可以把 <code>slow</code> 指针的行为理解为：</p>
<blockquote>
<p><strong>“我绝不轻易挪窝。我脚下必须踩着一个非零数我才肯走。如果我脚下是 0，我就死死守在这里，直到 <code>fast</code> 给我运来一个非零数把这个 0 换走，我才往后挪一步。”</strong></p>
</blockquote>
<p>由于 <code>slow</code> 是<strong>从左往右一个一个坑位填过来</strong>的，在它右边、<code>fast</code> 左边的区域，全是被跳过的 0 或者换过来的 0。</p>
<p><strong>所以：</strong></p>
<ul>
<li>
<p>当 <code>fast > slow</code> 时，<code>nums[slow]</code> <strong>必为 0</strong>。</p>
</li>
<li>
<p>当 <code>fast === slow</code> 时，逻辑不触发交换。</p>
</li>
</ul>
