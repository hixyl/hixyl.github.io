<h2 id="讲一讲-html-语义化"><a href="#讲一讲-html-语义化">🔥 讲一讲 HTML 语义化</a></h2>
<p>用正确的标签做正确的事情</p>

<li><strong>利于开发</strong>：结构清晰，可读性高，方便维护</li>

<ul>
<li>tdk（title、description、keyword）</li>

</li>
<li>历史：最开始后端写 HTML 用 table，后来美工用 div + css（无语义），前端用语义元素（ h1，p，article ）</li>
</ul>

<h2 id="在写-html-代码时语义化实践中应该注意什么"><a href="#在写-html-代码时语义化实践中应该注意什么"><strong>在写 HTML 代码时，语义化实践中应该注意什么？</strong></a></h2>
<ul>
<li>尽可能少的使用无语义的标签 <code>div</code> 和 <code>span</code></li>
<li>不要使用纯样式标签，如：<code>b</code>、<code>font</code>、<code>u</code> 等，改用 <code>css</code> 设置</li>
<li>需要强调的文本，可以包含在 <code>strong</code> 或 <code>em</code> 标签中，<code>strong</code> 默认样式是加粗（不要用 b），<code>em</code> 是斜体（不要用 <code>i</code> 标签）</li>
<li>使用表格时，标题要用 <code>caption</code>，表头用 <code>thead</code>，主体部分用 <code>tbody</code> 包围，尾部用 <code>tfoot</code> 包围。表头和一般单元格要区分开，表头用 <code>th</code>，单元格用 <code>td</code></li>
<li>每个 <code>input</code> 标签对应的说明文本都需要使用 <code>label</code> 标签，并且通过为 <code>input</code> 设置 <code>id</code> 属性，在 <code>lable</code> 标签中设置 <code>for=someld</code> 来让说明文本和相对应的 <code>input</code> 关联起来</li>
<li>不仅写 <code>HTML</code> 结构时，要用语义化标签，给元素写 <code>CSS</code> 类名时，JS 类名、方法名、变量命名等也要遵循语义化原则。不随意取名，不利于后期的代码重构和维护。同时，也最好不要用汉语拼音命名。</li>
</ul>
<h1 id="meta-viewport"><a href="#meta-viewport">Meta Viewport</a></h1>
<h2 id="meta-viewport-是做什么的怎么写"><a href="#meta-viewport-是做什么的怎么写"><strong>meta viewport 是做什么的，怎么写?</strong></a></h2>
<p><code>meta viewport</code> 是一个 HTML 元素，用于控制网页在移动设备上的显示和缩放行为。它通过设置 <code>viewport</code> 元素的属性，告诉浏览器如何调整页面的尺寸和缩放，以适应不同屏幕大小和分辨率的设备。</p>
<pre><code class="hljs language-html"><span class="hljs-tag">&#x3C;<span class="hljs-name">meta</span> <span class="hljs-attr">name</span>=<span class="hljs-string">"viewport"</span> <span class="hljs-attr">content</span>=<span class="hljs-string">"width=device-width, initial-scale=1.0, user-scalable=no"</span> /></span>
</code></pre>
<ul>
<li>name 为 viewport 表示供移动设备使用</li>
<li>content 定义了 viewport 的属性
<ul>
<li>width 表示显示宽度为设备宽度（兼容苹果）</li>
<li>initial-scale 表示设备与视口的缩放比率（兼容IE）</li>
</ul>
</li>
</ul>

<h1 id="web-quality-无障碍"><a href="#web-quality-无障碍"><strong>Web Quality (无障碍)</strong></a></h1>
<h2 id="说一下-web-quality"><a href="#说一下-web-quality"><strong>说一下 Web Quality</strong></a></h2>
<p>能够被残障人士使用的网站才能称得上一个易用的(易访问的)网站。</p>
<p>残障人士指的是那些带有残疾或者身体不健康的用户。</p>
<p>e.g. 使用 alt 属性:</p>
<p><code>&#x3C;img src="person.jpg" alt="this is a person"/> 有时候浏览器会无法显示图像。</code></p>
<p>具体的原因有:</p>
<ul>
<li>用户关闭了图像显示</li>
<li>浏览器是不支持图形显示的迷你浏览器</li>
<li>浏览器是语音浏览器(供盲人和弱视人群使用)</li>
</ul>
<p>如果使用了 alt 属性，那么浏览器至少可以显示或读出有关图像的描述。</p>
<h1 id="web-worker"><a href="#web-worker"><strong>Web Worker</strong></a></h1>
<h2 id="说一下-web-worker"><a href="#说一下-web-worker"><strong>说一下 Web Worker</strong></a></h2>
<p>在 HTML 页面中，如果在执行脚本时，页面的状态是不可响应的，直到脚本执行完成后，页面才变成可响应。web worker 是运行在后台的 JS，独立于其他脚本，不会影响页面你的性能。并且通过 postMessage 将结果回传到主线程。这样在进行复杂操作的时候，就不会阻塞主线程了。</p>
<p>如何创建 web worker:</p>
<ol>
<li>检测浏览器对于 web worker 的支持性</li>
<li>创建 web worker 文件(js，回传函数等)</li>
<li>创建 web worker 对象</li>
</ol>
<p>更多：</p>
<p><a href="https://developer.mozilla.org/en-US/docs/Web/API/Web_Workers_API/Using_web_workers">Using Web Workers - Web APIs | MDN</a></p>
<h1 id="其他"><a href="#其他">其他</a></h1>
<h2 id="什么是-data-url"><a href="#什么是-data-url">什么是 Data URL</a></h2>
<p>Data URL 是将图片转换为 base64 直接嵌入到了网页中，使用 <code>&#x3C;img src="data:[MIME type];base64"/></code> 这种方式引用图片，不需要再发请求获取图片。</p>
<h3 id="data-url-有缺点嘛"><a href="#data-url-有缺点嘛">Data URL 有缺点嘛？</a></h3>
<ul>
<li>base64 编码后的图片会比原来的体积大三分之一左右</li>
<li>Data URL 形式的图片不会缓存下来，每次访问页面都要被下载一次。
<ul>
<li>可以将 Data URL 写入到 CSS 文件中随着 CSS 被缓存下来。</li>
</ul>
</li>
</ul>
