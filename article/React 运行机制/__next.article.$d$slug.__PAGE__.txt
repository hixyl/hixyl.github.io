4:T3d34,<p>React 的核心不仅在于它的组件化思想，更在于其底层维持高性能和响应性的精妙算法设计。在面试和实际架构理解中，核心算法主要围绕 <strong>找出差异（Diffing）</strong>、<strong>任务调度（Fiber）</strong>、<strong>优先级管理（Lanes）</strong> 以及 <strong>状态闭包与存取（Hooks 机制）</strong> 展开。</p>
<p>为了从根本上理解 React 的运行机制，我们可以将这些算法解构为以下四个核心部分：</p>
<hr>
<h3 id="1-协调算法-reconciliation--diffing"><a href="#1-协调算法-reconciliation--diffing">1. 协调算法 (Reconciliation / Diffing)</a></h3>
<p>传统的树对比算法复杂度是 <span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><mrow><mi>O</mi><mo stretchy="false">(</mo><msup><mi>n</mi><mn>3</mn></msup><mo stretchy="false">)</mo></mrow><annotation encoding="application/x-tex">O(n^3)</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1.0641em;vertical-align:-0.25em;"></span><span class="mord mathnormal" style="margin-right:0.02778em;">O</span><span class="mopen">(</span><span class="mord"><span class="mord mathnormal">n</span><span class="msupsub"><span class="vlist-t"><span class="vlist-r"><span class="vlist" style="height:0.8141em;"><span style="top:-3.063em;margin-right:0.05em;"><span class="pstrut" style="height:2.7em;"></span><span class="sizing reset-size6 size3 mtight"><span class="mord mtight">3</span></span></span></span></span></span></span></span><span class="mclose">)</span></span></span></span>，这对于前端 DOM 树来说性能代价太高。React 的 Diff 算法基于三个“妥协性”的启发式前提，硬生生将复杂度降到了 <span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><mrow><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></mrow><annotation encoding="application/x-tex">O(n)</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal" style="margin-right:0.02778em;">O</span><span class="mopen">(</span><span class="mord mathnormal">n</span><span class="mclose">)</span></span></span></span>：</p>
<ol>
<li>
<p><strong>Tree Diff (层级对比):</strong> DOM 节点跨层级的移动操作少到可以忽略不计。因此，React 只会对同级节点进行比较。</p>
</li>
<li>
<p><strong>Component Diff (组件对比):</strong> 拥有相同类的两个组件会生成相似的树形结构，拥有不同类的两个组件会生成不同的树形结构。</p>
</li>
<li>
<p><strong>Element Diff (元素对比):</strong> 对于同一层级的一组子节点，它们可以通过唯一 <code>key</code> 进行区分。</p>
</li>
</ol>
<p><strong>Mermaid 图表：层级对比逻辑</strong></p>
<div class="mermaid">graph TD
    subgraph 旧虚拟DOM
    A1((A)) --> B1((B))
    A1 --> C1((C))
    C1 --> D1((D))
    end

    subgraph 新虚拟DOM
    A2((A)) --> B2((B))
    A2 --> C2((C))
    C2 --> E2((E))
    end

    A1 -.对比.- A2
    B1 -.对比.- B2
    C1 -.对比.- C2
    D1 -.销毁.- E2
</div>

<h3 id="2-可中断渲染架构-fiber-算法"><a href="#2-可中断渲染架构-fiber-算法">2. 可中断渲染架构 (Fiber 算法)</a></h3>
<p>在 React 15 及以前，由于采用递归遍历虚拟 DOM，一旦开始就无法中断。如果树很深，主线程会被长时间阻塞，导致掉帧（卡顿）。React 16 引入了 Fiber，本质是将<strong>递归变成了循环</strong>，将树的结构变成了<strong>单链表结构</strong>。</p>
<p>Fiber 的核心思想是<strong>时间切片（Time Slicing）</strong>：将渲染工作拆分成小任务，每执行完一个小任务就看看主线程是否有更高优先级的任务（如用户输入、动画），如果有，就让出控制权。</p>
<p><strong>Mermaid 图表：Fiber 的链表树结构</strong></p>
<div class="mermaid">graph TD
    Root((RootFiber)) -- child --> App((App))
    App -- child --> Header((Header))
    Header -- sibling --> Main((Main))
    Main -- sibling --> Footer((Footer))
    
    Header -. return .-> App
    Main -. return .-> App
    Footer -. return .-> App
    App -. return .-> Root
</div>
<p><strong>代码模拟：可中断的工作循环</strong></p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">let</span> nextUnitOfWork = <span class="hljs-literal">null</span>; <span class="hljs-comment">// 指向下一个 Fiber 节点</span>

<span class="hljs-comment">// 浏览器的 requestIdleCallback 会在浏览器空闲时调用此函数</span>
<span class="hljs-keyword">function</span> <span class="hljs-title function_">workLoop</span>(<span class="hljs-params">deadline</span>) {
  <span class="hljs-keyword">let</span> shouldYield = <span class="hljs-literal">false</span>; <span class="hljs-comment">// 是否需要让出控制权</span>
  
  <span class="hljs-comment">// 只要有任务且不需要让出控制权，就一直执行</span>
  <span class="hljs-keyword">while</span> (nextUnitOfWork &#x26;&#x26; !shouldYield) {
    nextUnitOfWork = <span class="hljs-title function_">performUnitOfWork</span>(nextUnitOfWork); <span class="hljs-comment">// 执行当前任务，并返回下一个任务</span>
    <span class="hljs-comment">// 检查剩余时间，如果不足则让出</span>
    shouldYield = deadline.<span class="hljs-title function_">timeRemaining</span>() &#x3C; <span class="hljs-number">1</span>; 
  }
  
  <span class="hljs-keyword">if</span> (nextUnitOfWork) {
    <span class="hljs-comment">// 任务没做完，请求下一次浏览器空闲时继续</span>

  }

<span class="hljs-keyword">function</span> <span class="hljs-title function_">performUnitOfWork</span>(<span class="hljs-params">fiber</span>) {
  <span class="hljs-comment">// 1. 创建 DOM，处理当前 fiber 节点的更新</span>
  <span class="hljs-comment">// 2. 将子节点转换为新的 fiber 节点（Diffing 发生在这里）</span>
  
  <span class="hljs-comment">// 3. 返回下一个要处理的 fiber (深度优先：child -> sibling -> return)</span>
  <span class="hljs-keyword">if</span> (fiber.<span class="hljs-property">child</span>) <span class="hljs-keyword">return</span> fiber.<span class="hljs-property">child</span>;

<h3 id="3-位运算调度优先级-lanes-模型"><a href="#3-位运算调度优先级-lanes-模型">3. 位运算调度优先级 (Lanes 模型)</a></h3>
<p>在并发模式下，不仅需要打断，还需要决定“先执行谁”。早期的 ExpirationTime（过期时间）模型在处理批量更新和复杂优先级时不够灵活。React 17+ 引入了 Lanes 模型，借鉴了车道的概念，利用<strong>32位整数的位运算</strong>来管理优先级。</p>
<p>位运算的优势在于速度极快，且能用按位与（<code>&#x26;</code>）、按位或（<code>|</code>）轻松处理批量的任务合并与分离。</p>
<p><strong>Mermaid 图表：优先级插队处理</strong></p>
<div class="mermaid">sequenceDiagram
    participant User as 用户
    participant Scheduler as React调度器
    participant Render as 渲染阶段
    
    User->>Scheduler: 发起低优先级更新(如获取数据)
    Scheduler->>Render: 分配低优先级Lane, 开始渲染...
    Render-->>Render: 渲染了一半节点
    User->>Scheduler: 发起高优先级更新(如点击按钮)
    Scheduler->>Render: 判定优先级更高, 中断当前渲染!
    Render->>Render: 放弃旧工作，重新处理高优先级Lane
    Render-->>User: 按钮响应完成
    Scheduler->>Render: 恢复处理之前的低优先级Lane
</div>
<p><strong>代码模拟：Lanes 位运算</strong></p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-comment">// 不同的数字占用不同的位（二进制表示）</span>
<span class="hljs-keyword">const</span> <span class="hljs-title class_">SyncLane</span> = <span class="hljs-number">0b00001</span>;       <span class="hljs-comment">// 最高优先级（如用户点击）</span>
<span class="hljs-keyword">const</span> <span class="hljs-title class_">InputContinuousLane</span> = <span class="hljs-number">0b00010</span>; <span class="hljs-comment">// 连续交互（如拖拽）</span>
<span class="hljs-keyword">const</span> <span class="hljs-title class_">DefaultLane</span> = <span class="hljs-number">0b00100</span>;    <span class="hljs-comment">// 默认（如请求数据返回）</span>
<span class="hljs-keyword">const</span> <span class="hljs-title class_">IdleLane</span> = <span class="hljs-number">0b10000</span>;       <span class="hljs-comment">// 最低（空闲时执行）</span>

<span class="hljs-keyword">let</span> pendingLanes = <span class="hljs-number">0b00000</span>; <span class="hljs-comment">// 当前待处理的所有任务</span>

<span class="hljs-comment">// 1. 添加任务 (按位或)</span>

<span class="hljs-comment">// 2. 提取最高优先级任务 (查找最低位的 1)</span>
<span class="hljs-comment">// 算法公式：lanes &#x26; -lanes</span>
<span class="hljs-keyword">const</span> highestPriorityLane = pendingLanes &#x26; -pendingLanes; 
<span class="hljs-comment">// 0b00101 &#x26; 0b11011 = 0b00001 (成功提取出 SyncLane)</span>

<span class="hljs-comment">// 3. 移除已完成的任务 (按位与和按位取反)</span>

<h3 id="4-hooks-执行机制-链表存储"><a href="#4-hooks-执行机制-链表存储">4. Hooks 执行机制 (链表存储)</a></h3>
<p>Hooks 赋予了函数组件状态，但函数每次渲染都会重新执行，React 是如何记住状态的？</p>
<p>答案是：<strong>每个 Fiber 节点上有一个 <code>memoizedState</code> 属性，它存放着一个由 Hooks 组成的单向链表。</strong></p>
<p>这就是为什么 React 严格规定 <strong>“不要在循环、条件或嵌套函数中调用 Hook”</strong>——因为 React 完全依赖 Hooks 的执行顺序来将状态与对应的 Hook 匹配。</p>
<p><strong>Mermaid 图表：Hooks 链表结构</strong></p>
<div class="mermaid">graph LR
    subgraph FiberNode
    State[memoizedState]
    end
    
    Hook1[Hook 1: useState]
    Hook2[Hook 2: useEffect]
    Hook3[Hook 3: useRef]
    
    State --> Hook1
    Hook1 -- next --> Hook2
    Hook2 -- next --> Hook3
    Hook3 -- next --> null
</div>
<p><strong>代码模拟：Hooks 的顺序调用机制</strong></p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">let</span> currentlyRenderingFiber = <span class="hljs-literal">null</span>;

  <span class="hljs-keyword">return</span> [hook.<span class="hljs-property">memoizedState</span>, dispatch];
}
