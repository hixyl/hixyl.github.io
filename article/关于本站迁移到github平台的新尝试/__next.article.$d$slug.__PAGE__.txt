4:T98a,<h2 id="前言"><a href="#前言">前言</a></h2>
<p>学习git命令后，体会到其神奇之处，一直想借助github平台和hexo框架，将之前臃肿的wordpress网站迁移上去。终于在这周周末，抽出了时间去做这件事。这篇文章将记录本站的hexo框架部署过程。</p>
<h2 id="hexo框架的安装"><a href="#hexo框架的安装">hexo框架的安装</a></h2>
<pre><code class="hljs language-bash">
<span class="hljs-built_in">sudo</span> npm install hexo-cli -g
hexo init
npm install hexo-deployer-git --save
</code></pre>
<h2 id="设置主题"><a href="#设置主题">设置主题</a></h2>
<p>本站使用Butterfly主题，个人感觉非常美观。</p>
<p>在根目录执行</p>
<pre><code class="hljs language-bash">git <span class="hljs-built_in">clone</span> -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly
npm install hexo-renderer-pug hexo-renderer-stylus --save
</code></pre>
<p>修改 Hexo 根目录下的 <code>_config.yml</code>，把主题改为<code>butterfly</code></p>
<pre><code class="hljs language-yaml"><span class="hljs-string">theme:butterfly</span>
</code></pre>
<p>在 hexo 的根目录创建一个文件 <code>_config.butterfly.yml</code>，并把主题目录的 <code>_config.yml</code> 内容复制到 <code>_config.butterfly.yml</code> 去。( 注意: 复制的是主题的 <code>_config.yml</code> ,而不是 hexo 的 <code>_config.yml</code>)</p>
<blockquote>
<p><strong>注意</strong>： 不要把主题目录的 <code>_config.yml</code> 删掉</p>
</blockquote>
<blockquote>
<p><strong>注意</strong>： 以后只需要在 <code>_config.butterfly.yml</code>进行配置就行。如果使用了 <code>_config.butterfly.yml</code>， 配置主题的 <code>_config.yml</code> 将不会有效果。</p>
</blockquote>
<p>Hexo会自动合并主题中的<code>_config.yml</code>和 <code>_config.butterfly.yml</code>里的配置，如果存在同名配置，会使用<code>_config.butterfly.yml</code>的配置，其优先度较高。</p>
<h2 id="压缩静态资源"><a href="#压缩静态资源">压缩静态资源</a></h2>
<p>运用<code>hexo-all-minifier</code>插件</p>
<pre><code class="hljs language-bash">npm install hexo-all-minifier --save
</code></pre>
<p>在<code>_config.yml</code>里添加</p>
