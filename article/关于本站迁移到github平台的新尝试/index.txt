    <html><head></head><body><h2>前言</h2>
<p>学习git命令后，体会到其神奇之处，一直想借助github平台和hexo框架，将之前臃肿的wordpress网站迁移上去。终于在这周周末，抽出了时间去做这件事。这篇文章将记录本站的hexo框架部署过程。</p>
<h2>hexo框架的安装</h2>
<pre><code class="language-bash">
sudo npm <span class="hljs-keyword">install</span> hexo-cli -g
hexo init
npm <span class="hljs-keyword">install</span> hexo-deployer-git --save
</code></pre>
<h2>设置主题</h2>
<p>本站使用Butterfly主题，个人感觉非常美观。</p>
<p>在根目录执行</p>
<pre><code class="language-bash">git <span class="hljs-keyword">clone</span> <span class="hljs-title">-b</span> <span class="hljs-keyword">master</span> <span class="hljs-title">https</span>://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly
npm install hexo-renderer-pug hexo-renderer-stylus --save
</code></pre>
<p>修改 Hexo 根目录下的 <code>_<span class="hljs-built_in">config</span>.yml</code>，把主题改为<code><span class="hljs-attribute">butterfly</span></code></p>
<pre><code class="language-yaml"><span class="hljs-symbol">theme:</span>butterfly
</code></pre>
<p>在 hexo 的根目录创建一个文件 <code>_<span class="hljs-built_in">config</span>.<span class="hljs-keyword">butterfly.yml</span></code>，并把主题目录的 <code>_<span class="hljs-built_in">config</span>.yml</code> 内容复制到 <code>_<span class="hljs-built_in">config</span>.<span class="hljs-keyword">butterfly.yml</span></code> 去。( 注意: 复制的是主题的 <code>_<span class="hljs-built_in">config</span>.yml</code> ,而不是 hexo 的 <code>_<span class="hljs-built_in">config</span>.yml</code>)</p>
<blockquote>
<p><strong>注意</strong>： 不要把主题目录的 <code>_<span class="hljs-built_in">config</span>.yml</code> 删掉</p>
</blockquote>
<blockquote>
<p><strong>注意</strong>： 以后只需要在 <code>_<span class="hljs-built_in">config</span>.<span class="hljs-keyword">butterfly.yml</span></code>进行配置就行。如果使用了 <code>_<span class="hljs-built_in">config</span>.<span class="hljs-keyword">butterfly.yml</span></code>， 配置主题的 <code>_<span class="hljs-built_in">config</span>.yml</code> 将不会有效果。</p>
</blockquote>
<p>Hexo会自动合并主题中的<code>_<span class="hljs-built_in">config</span>.yml</code>和 <code>_<span class="hljs-built_in">config</span>.<span class="hljs-keyword">butterfly.yml</span></code>里的配置，如果存在同名配置，会使用<code>_<span class="hljs-built_in">config</span>.<span class="hljs-keyword">butterfly.yml</span></code>的配置，其优先度较高。</p>
<h2>压缩静态资源</h2>
<p>运用<code>hexo-<span class="hljs-keyword">all</span>-minifier</code>插件</p>
<pre><code class="language-bash">npm install hexo-<span class="hljs-keyword">all</span>-minifier <span class="hljs-comment">--save</span>
</code></pre>
<p>在<code>_<span class="hljs-built_in">config</span>.yml</code>里添加</p>
<pre><code class="language-yaml"><span class="hljs-attr">all_minifier:</span> <span class="hljs-literal">true</span>
</code></pre>
