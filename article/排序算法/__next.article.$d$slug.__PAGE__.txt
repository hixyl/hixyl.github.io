<h3 id="排序算法的核心分野"><a href="#排序算法的核心分野">排序算法的核心分野</a></h3>
<p>在深入探讨之前，我们需要建立一个重要的认知：排序算法分为两大类。</p>
<ol>
<li>
<p><strong>比较排序 (Comparison Sorts)</strong>: 这类算法通过<strong>比较</strong>元素之间的大小关系来决定它们的顺序。我们熟知的冒泡排序、插入排序、选择排序、归并排序、堆排序以及<strong>快速排序</strong>都属于这一类。比较排序算法的平均时间复杂度存在一个理论下限，即 O(nlogn)。它们不关心元素的具体数值，只关心它们之间的相对大小。</p>
</li>
<li>
<p><strong>非比较排序 (Non-comparison Sorts)</strong>: 这类算法不通过比较来排序，而是利用元素自身的特性（比如数值大小、构成部分）来确定其位置。它们的效率可以突破 O(nlogn) 的限制，达到线性时间 O(n)。<strong>桶排序</strong>和<strong>基数排序</strong>是其中的杰出代表。这类算法通常对数据有额外的要求，比如必须是整数或可以映射为整数。</p>
</li>
</ol>
<p>理解这个分野是理解这三种算法各自定位和优劣势的关键。</p>
<hr>
<h3 id="快速排序-quick-sort"><a href="#快速排序-quick-sort">快速排序 (Quick Sort)</a></h3>
<p>快速排序是<strong>比较排序</strong>中的王者，因其在平均情况下的卓越性能而被广泛应用，例如 V8 引擎（JavaScript 的核心）内部的 <code>Array.prototype.sort</code> 在处理非稀疏数组时就主要基于快速排序的变体。</p>
<h4 id="核心思想分而治之-divide-and-conquer"><a href="#核心思想分而治之-divide-and-conquer">核心思想：分而治之 (Divide and Conquer)</a></h4>
<p>快速排序的逻辑可以用三个词概括：<strong>选轴、分区、递归</strong>。</p>
<ol>
<li>
<p><strong>选轴 (Pivot Selection)</strong>: 从待排序的数组中选择一个元素作为“基准”或“轴心”(Pivot)。这个选择至关重要，直接影响算法的效率。</p>
</li>
<li>
<p><strong>分区 (Partition)</strong>: 重新排列数组。将所有小于基准的元素移动到基准的左边，所有大于基准的元素移动到基准的右边。相等的元素可以放在任意一边。操作完成后，基准元素就处于其最终排序后的正确位置上。</p>
</li>
<li>
<p><strong>递归 (Recursion)</strong>: 对基准左边的子数组和右边的子数组分别重复上述过程，直到子数组的大小为 1 或 0，此时整个数组就有序了。</p>
</li>
</ol>
<h4 id="图解步骤"><a href="#图解步骤">图解步骤</a></h4>
<p>假设我们有数组 <code>[8, 3, 5, 4, 7, 2, 9]</code>，我们选择 <code>7</code> 作为基准 (Pivot)。</p>
<ol>
<li>
<p><strong>分区</strong>:</p>
<ul>
<li>
<p>创建两个指针，<code>left</code> 从头开始，<code>right</code> 从尾（不含基准）开始。</p>
</li>
<li>
<p><code>left</code> 向右移动，直到找到一个大于等于基准的数 (<code>8</code>)。</p>
</li>
<li>
<p><code>right</code> 向左移动，直到找到一个小于基准的数 (<code>2</code>)。</p>
</li>
<li>
<p>交换 <code>8</code> 和 <code>2</code>。数组变为 <code>[2, 3, 5, 4, 8, 7, 9]</code>。</p>
</li>
<li>
<p>继续移动指针，直到 <code>left</code> 和 <code>right</code> 交错。</p>
</li>
<li>
<p>最后将基准 <code>7</code> 与 <code>left</code> 指针指向的元素 <code>8</code> 交换。</p>
</li>
<li>
<p>数组变为 <code>[2, 3, 5, 4, 7, 8, 9]</code>。此时，<code>7</code> 已经就位。</p>
</li>
</ul>
</li>
<li>
<p><strong>递归</strong>:</p>
<ul>
<li>
<p>对 <code>7</code> 左边的子数组 <code>[2, 3, 5, 4]</code> 重复快速排序。</p>
</li>
<li>
<p>对 <code>7</code> 右边的子数组 <code>[8, 9]</code> 重复快速排序。</p>
</li>
</ul>
</li>
</ol>
<p>这个过程不断进行，直到所有元素都就位。</p>

<h4 id="复杂度与特性分析"><a href="#复杂度与特性分析">复杂度与特性分析</a></h4>
<ul>
<li>
<p><strong>时间复杂度</strong>:</p>
<ul>
<li>
<p><strong>平均情况: O(nlogn)</strong>。每次分区操作的时间是 O(n)。在理想情况下，每次基准都能将数组平分为两半，递归深度为 logn。总时间就是 O(nlogn)。随机化基准让算法极大概率达到这个性能。</p>
</li>
<li>
<p><strong>最坏情况: O(n2)</strong>。当每次选择的基准都是当前数组的最小值或最大值时（例如，对一个已经排好序的数组，每次都选最后一个元素做基准），递归树严重不平衡，深度变为 n，导致总时间为 O(n2)。</p>
</li>
<li>
<p><strong>最好情况: O(nlogn)</strong>。</p>
</li>
</ul>
</li>
<li>
<p><strong>空间复杂度</strong>:</p>
<ul>
<li>
<p><strong>平均情况: O(logn)</strong>。主要开销是递归调用栈的深度。</p>
</li>
<li>
<p><strong>最坏情况: O(n)</strong>。当递归树退化成链表时。</p>
</li>
</ul>
</li>
<li>
<p><strong>稳定性</strong>: <strong>不稳定</strong>。在分区过程中，与基准相等的多个元素的相对位置可能会被改变。例如 <code>[5, 8, 5, 2, 9]</code>，若以 <code>8</code> 为基准，第一个 <code>5</code> 可能被交换到后面。</p>
</li>
</ul>
<h4 id="适用场景"><a href="#适用场景">适用场景</a></h4>
<p>快速排序是<strong>通用场景下最快的内部排序算法之一</strong>。当你需要对一个内存中的大型、无序、元素类型不限（只要能比较）的数组进行排序时，它通常是首选。但要注意防范最坏情况的发生，随机化是简单有效的手段。</p>

<h3 id="桶排序-bucket-sort"><a href="#桶排序-bucket-sort">桶排序 (Bucket Sort)</a></h3>
<p>桶排序是<strong>非比较排序</strong>的代表，它的核心思想是“<strong>空间换时间</strong>”。</p>
<h4 id="核心思想映射与分散"><a href="#核心思想映射与分散">核心思想：映射与分散</a></h4>
<p>桶排序假设输入数据服从<strong>均匀分布</strong>，然后将数据根据其数值大小，通过一个映射函数，分配到有限数量的“桶”(Bucket)里。每个桶再分别进行排序（可以使用其他排序算法，如插入排序，甚至递归使用桶排序），最后按顺序将所有非空桶中的元素合并起来，得到有序的结果。</p>
<h4 id="图解步骤-1"><a href="#图解步骤-1">图解步骤</a></h4>
<p>假设我们有数组 <code>[0.78, 0.17, 0.39, 0.26, 0.72, 0.94, 0.21, 0.12]</code>，我们要把它们分到 5 个桶里。</p>
<ol>
<li>
<p><strong>创建桶</strong>: 创建 5 个空桶，编号 0 到 4。</p>
</li>
<li>
<p><strong>映射与入桶</strong>:</p>
<ul>
<li>
<p>映射函数可以是 <code>f(x) = floor(n * x)</code>，这里 <code>n=5</code>。</p>
</li>
<li>
<p><code>0.78</code> -> <code>floor(5 * 0.78)</code> -> <code>floor(3.9)</code> -> 桶 3</p>
</li>
<li>
<p><code>0.17</code> -> <code>floor(5 * 0.17)</code> -> <code>floor(0.85)</code> -> 桶 0</p>
</li>
<li>
<p>...以此类推，将所有元素放入对应桶中。</p>
</li>
<li>
<p>桶 0: <code>[0.17, 0.12]</code></p>
</li>
<li>
<p>桶 1: <code>[0.26, 0.21]</code></p>
</li>
<li>
<p>桶 2: <code>[0.39]</code></p>
</li>
<li>
<p>桶 3: <code>[0.78, 0.72]</code></p>
</li>
<li>
<p>桶 4: <code>[0.94]</code></p>
</li>
</ul>
</li>
<li>
<p><strong>桶内排序</strong>: 对每个非空桶分别进行排序。</p>
<ul>
<li>
<p>桶 0: <code>[0.12, 0.17]</code></p>
</li>
<li>
<p>桶 1: <code>[0.21, 0.26]</code></p>
</li>
<li>
<p>桶 2: <code>[0.39]</code></p>
</li>
<li>
<p>桶 3: <code>[0.72, 0.78]</code></p>
</li>
<li>
<p>桶 4: <code>[0.94]</code></p>
</li>
</ul>
</li>
<li>
<p><strong>合并</strong>: 按顺序拼接所有桶的元素。</p>
<ul>
<li><code>[0.12, 0.17, 0.21, 0.26, 0.39, 0.72, 0.78, 0.94]</code></li>
</ul>
</li>
</ol>

    <span class="hljs-comment">// 3. 将元素分配到桶中</span>
    <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> i = <span class="hljs-number">0</span>; i &#x3C; arr.<span class="hljs-property">length</span>; i++) {
        <span class="hljs-keyword">const</span> bucketIndex = <span class="hljs-title class_">Math</span>.<span class="hljs-title function_">floor</span>((arr[i] - min) / bucketSize);
        buckets[bucketIndex].<span class="hljs-title function_">push</span>(arr[i]);
    }

    <span class="hljs-comment">// 4. 对每个桶进行排序，并合并结果</span>
    <span class="hljs-keyword">const</span> sortedArr = [];
    <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> i = <span class="hljs-number">0</span>; i &#x3C; buckets.<span class="hljs-property">length</span>; i++) {
        <span class="hljs-keyword">if</span> (buckets[i].<span class="hljs-property">length</span> > <span class="hljs-number">0</span>) {
            <span class="hljs-comment">// 对每个桶进行排序，这里使用插入排序</span>
            <span class="hljs-title function_">insertionSort</span>(buckets[i]);
            <span class="hljs-comment">// 展开桶内元素并添加到结果数组</span>
            sortedArr.<span class="hljs-title function_">push</span>(...buckets[i]);
        }
    }

    <span class="hljs-keyword">return</span> sortedArr;
}

<h4 id="复杂度与特性分析-1"><a href="#复杂度与特性分析-1">复杂度与特性分析</a></h4>
<ul>
<li>
<p><strong>时间复杂度</strong>:</p>
<ul>
<li>
<p><strong>平均情况: O(n+k)</strong>。其中 <code>n</code> 是元素数量，<code>k</code> 是桶的数量。遍历一遍数组将元素放入桶中是 O(n)。如果数据均匀分布，每个桶里有约 <code>n/k</code> 个元素，对每个桶排序的时间为 O((n/k)log(n/k))（如果用快排）或 O((n/k)2)（如果用插入排序）。总时间约为 O(n)+kcdotO((n/k)2)。当 <code>k</code> 约等于 <code>n</code> 时，每个桶内只有常数个元素，排序时间为 O(1)，总时间趋近于 O(n)。</p>
</li>
<li>
<p><strong>最坏情况: O(n2)</strong> (或 O(nlogn)，取决于桶内排序算法)。当所有元素都落入同一个桶中，桶排序退化为桶内使用的排序算法的复杂度。</p>
</li>
</ul>
</li>
<li>
<p><strong>空间复杂度</strong>: <strong>O(n+k)</strong>。需要额外的空间来存储 <code>k</code> 个桶和 <code>n</code> 个元素。</p>
</li>
<li>
<p><strong>稳定性</strong>: <strong>稳定</strong> (取决于桶内排序算法的稳定性)。如果桶内排序算法是稳定的（如插入排序、计数排序），并且我们将元素追加到桶的末尾，那么整个桶排序就是稳定的。</p>
</li>
</ul>
<h4 id="适用场景-1"><a href="#适用场景-1">适用场景</a></h4>
<p>桶排序极度依赖于输入数据。它最适用于：</p>
<ol>
<li>
<p><strong>数据是浮点数或整数，且范围已知</strong>。</p>
</li>
<li>
<p><strong>数据在整个范围内大致呈均匀分布</strong>。如果数据分布极不均匀，其性能会急剧下降。</p>
</li>
</ol>
<blockquote>
<p>延伸思考：计数排序 (Counting Sort)</p>
<p>计数排序是桶排序的一个特例。当桶的大小为 1 时，即每个桶只对应一个确定的整数值，桶排序就变成了计数排序。计数排序的核心是统计每个整数出现的次数，然后根据统计结果直接写回原数组。它的时间复杂度为 O(n+r)，其中 r 是数据的范围（最大值-最小值）。它非常高效，但对数据范围敏感。</p>
</blockquote>
<hr>
<h3 id="基数排序-radix-sort"><a href="#基数排序-radix-sort">基数排序 (Radix Sort)</a></h3>
<p>基数排序同样是<strong>非比较排序</strong>，它非常巧妙，不比较整个数字的大小，而是按“位”来排序。它通常被用于排序<strong>整数</strong>，尤其是位数较短的大量整数。</p>
<h4 id="核心思想按位分配与收集"><a href="#核心思想按位分配与收集">核心思想：按位分配与收集</a></h4>
<p>基数排序将所有待排序的数字（通常是正整数）统一为相同的位数（位数不足的在前面补 0），然后从<strong>最低位 (Least Significant Digit, LSD)</strong> 开始，依次对每一位进行排序，直到<strong>最高位 (Most Significant Digit, MSD)</strong>。</p>
<p>关键在于，<strong>每一位的排序必须是稳定的</strong>。也就是说，在按某一位排序后，相同数值的元素必须保持它们在前一轮排序中的相对顺序。这通常通过<strong>计数排序</strong>作为其子过程来实现。</p>
<h4 id="图解步骤-2"><a href="#图解步骤-2">图解步骤</a></h4>
<p>假设我们有数组 <code>[170, 45, 75, 90, 802, 24, 2, 66]</code>。</p>
<ol>
<li>
<p><strong>按个位 (LSD) 排序</strong>:</p>
<ul>
<li>
<p><code>170</code> (0), <code>90</code> (0), <code>802</code> (2), <code>2</code> (2), <code>24</code> (4), <code>45</code> (5), <code>75</code> (5), <code>66</code> (6)</p>
</li>
<li>
<p>根据个位数，使用稳定的排序（如计数排序）重新排列。</p>
</li>
<li>
<p>结果: <code>[170, 90, 802, 2, 24, 45, 75, 66]</code> (注意 <code>170</code> 在 <code>90</code> 前面，<code>802</code> 在 <code>2</code> 前面)</p>
</li>
</ul>
</li>
<li>
<p><strong>按十位排序</strong>:</p>
<ul>
<li>
<p><code>802</code> (0), <code>2</code> (0), <code>24</code> (2), <code>45</code> (4), <code>66</code> (6), <code>170</code> (7), <code>75</code> (7), <code>90</code> (9)</p>
</li>
<li>
<p>根据十位数，对上一轮的结果进行稳定排序。</p>
</li>
<li>
<p>结果: <code>[802, 2, 24, 45, 66, 170, 75, 90]</code> (注意 <code>802</code> 仍在 <code>2</code> 前面，<code>170</code> 仍在 <code>75</code> 前面)</p>
</li>
</ul>
</li>
<li>
<p><strong>按百位排序</strong>:</p>
<ul>
<li>
<p><code>2</code> (0), <code>24</code> (0), <code>45</code> (0), <code>66</code> (0), <code>75</code> (0), <code>90</code> (0), <code>170</code> (1), <code>802</code> (8)</p>
</li>
<li>
<p>根据百位数，对上一轮的结果进行稳定排序。</p>
</li>
<li>
<p>结果: <code>[2, 24, 45, 66, 75, 90, 170, 802]</code></p>
</li>
</ul>
</li>
</ol>
<p>排序完成。</p>

<h4 id="复杂度与特性分析-2"><a href="#复杂度与特性分析-2">复杂度与特性分析</a></h4>
<ul>
<li>
<p><strong>时间复杂度</strong>: <strong>O(dcdot(n+k))</strong> 或简写为 <strong>O(dcdotn)</strong>。</p>
<ul>
<li>
<p><code>d</code>: 数组中最大数字的位数。</p>
</li>
<li>
<p><code>n</code>: 数组元素的个数。</p>
</li>
<li>
<p><code>k</code>: 基数，即桶的数量。对于十进制数，<code>k=10</code>。</p>
</li>
<li>
<p>每一轮分配和收集都是 O(n+k) 的。总共要进行 <code>d</code> 轮。由于 <code>k</code> 通常是常数（如10），所以复杂度可以看作是线性的 O(dcdotn)。</p>
</li>
</ul>
</li>
<li>
<p><strong>空间复杂度</strong>: <strong>O(n+k)</strong>。需要 <code>k</code> 个桶来存放 <code>n</code> 个元素。</p>
</li>
<li>
<p><strong>稳定性</strong>: <strong>稳定</strong>。基数排序的正确性<strong>强依赖于</strong>其内部排序的稳定性。我们的实现中，将元素顺序推入桶中，再顺序取出，保证了稳定性。</p>
</li>
</ul>
<h4 id="适用场景-2"><a href="#适用场景-2">适用场景</a></h4>
<p>基数排序是<strong>特定场景下的效率之王</strong>：</p>
<ol>
<li>
<p><strong>待排序的元素是整数</strong>（或可以被不失真地映射为整数，如特定格式的字符串）。</p>
</li>
<li>
<p><strong>元素的位数 <code>d</code> 相对较小</strong>。如果 <code>d</code> 过大（例如，64位长整数），<code>d</code> 这个常数因子会变得很大，可能反而不如 O(nlogn) 的快速排序。</p>
</li>
<li>
<p><strong>海量数据排序</strong>。当 <code>n</code> 非常大时，线性时间复杂度的优势会完全体现出来。例如，对全国所有人的年龄进行排序。</p>
</li>
</ol>

<h4 id="如何选择我的最终建议"><a href="#如何选择我的最终建议">如何选择？我的最终建议</a></h4>
<p>这三种排序算法并非相互替代的关系，而是针对不同问题域的“专科医生”。</p>
<ol>
<li>
<p><strong>默认和通用首选：快速排序 (Randomized)</strong></p>
<ul>
<li>当你面对一个未知的数据集，不确定其分布，且元素可能是任意可比较类型（不只是数字），<strong>随机化的快速排序是你的第一选择</strong>。它的平均性能极其出色，空间效率高（接近原地排序），且实现相对简单，是名副其实的“通用排序之王”。</li>
</ul>
</li>
<li>
<p><strong>数据分布均匀时的利器：桶排序</strong></p>
<ul>
<li>如果你的数据是数值型，并且你<strong>有理由相信它们大致均匀地分布在一个已知的范围内</strong>（例如，大量的随机生成的浮点数），桶排序可以提供惊人的线性时间性能。但一定要警惕数据倾斜，否则它会退化成一个低效的排序。</li>
</ul>
</li>
<li>
<p><strong>整数排序的终极武器：基数排序</strong></p>
<ul>
<li>当你的任务是<strong>对大量的整数进行排序</strong>时，基数排序是目前最快的算法。它的性能稳定，不受数据初始顺序和分布的影响，只要整数的位数不是天文数字，它就能以压倒性的线性时间优势胜出。特别是在处理身份证号、电话号码（转换后）这类位数固定的长整数时，它的威力无人能及。</li>
</ul>
</li>
</ol>
<p><strong>结论</strong>：没有“最好”的算法，只有“最适合”的算法。理解它们各自的底层逻辑和约束条件，是做出正确技术选型的根本。如果必须给出一个在工程实践中最常被信赖的答案，那就是<strong>经过优化的快速排序</strong>，因为它在普适性和平均性能之间取得了最佳的平衡。但对于特定问题，巧妙地运用桶排序或基数排序，能让你写出性能远超通用库的程序。</p>0:{"rsc":["$","$1","c",{"children":[[["$","$L2",null,{}],["$","article",null,{"className":"container page-module__y-5DuW__article-page","children":["$","div",null,{"className":"page-module__y-5DuW__article-layout","children":[["$","div",null,{"className":"page-module__y-5DuW__article-main","children":[["$","header",null,{"className":"page-module__y-5DuW__article-header","children":[["$","div",null,{"children":[["$","span",null,{"children":"📂"}],[["$","$L3","前端开发/算法",{"href":"/categories/%E5%89%8D%E7%AB%AF%E5%BC%80%E5%8F%91/%E7%AE%97%E6%B3%95/","className":"page-module__y-5DuW__meta-link","children":"前端开发 / 算法"}]]]}],["$","h1",null,{"className":"page-module__y-5DuW__article-title","children":"排序算法"}],["$","div",null,{"className":"page-module__y-5DuW__article-meta","children":[["$","time",null,{"dateTime":"2025-07-22T00:00:00.000Z","children":"2025年7月22日"}],["$","span",null,{"className":"page-module__y-5DuW__sep","children":"·"}],["$","span",null,{"children":[3932," 字"]}],["$","span",null,{"className":"page-module__y-5DuW__sep","children":"·"}],["$","span",null,{"children":["约 ",14," 分钟"]}]]}],["$","div",null,{"className":"page-module__y-5DuW__article-tags","children":[["$","span","Obsidian",{"className":"page-module__y-5DuW__tag","children":"Obsidian"}]]}]]}],["$","div",null,{"className":"prose page-module__y-5DuW__article-content","dangerouslySetInnerHTML":{"__html":"$4"}}],"$L5","$L6"]}],"$L7"]}]}],"$L8"],["$L9","$La"],"$Lb"]}],"isPartial":false,"staleTime":300,"varyParams":null,"buildId":"XAGPIwpwei9ME4jeaOns4"}
