4:T6d50,<p>好的，我们来系统且深入地探讨一下 JavaScript 中的深拷贝。这不仅仅是“如何实现”的问题，更关键的是理解“为何如此实现”以及各种方法的优缺点和适用场景。</p>
<h3 id="核心出发点为什么需要深拷贝"><a href="#核心出发点为什么需要深拷贝">核心出发点：为什么需要深拷贝？</a></h3>
<p>要理解深拷贝（Deep Copy），必须先理解 JavaScript 的数据类型和内存模型。</p>
<p>JavaScript 的数据类型分为两类：</p>
<ol>
<li>
<p>原始类型 (Primitive Types): string, number, boolean, null, undefined, symbol, bigint。</p>
<p>这些值直接存储在**栈 (Stack)**内存中。当你把一个原始类型的值赋给另一个变量时，你是在创建一个全新的、独立的值。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">let</span> a = <span class="hljs-number">100</span>;
<span class="hljs-keyword">let</span> b = a; <span class="hljs-comment">// b 得到了 a 的值的副本</span>
b = <span class="hljs-number">200</span>;   <span class="hljs-comment">// 修改 b 不会影响 a</span>
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(a); <span class="hljs-comment">// 输出 100</span>
</code></pre>
</li>
<li>
<p>引用类型 (Reference Types): Object (包括普通对象、数组 Array、函数 Function、日期 Date、正则表达式 RegExp 等)。</p>
<p>这些值的内容（数据本身）存储在**堆 (Heap)内存中，而变量名和指向该内容的内存地址则存储在栈 (Stack)**内存中。当你把一个引用类型的值赋给另一个变量时，你只是复制了这个“内存地址的指针”，而不是数据本身。两个变量最终指向的是同一个堆内存中的对象。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">let</span> obj1 = { <span class="hljs-attr">name</span>: <span class="hljs-string">'Alice'</span>, <span class="hljs-attr">details</span>: { <span class="hljs-attr">age</span>: <span class="hljs-number">30</span> } };
<span class="hljs-keyword">let</span> obj2 = obj1; <span class="hljs-comment">// obj2 复制了 obj1 的内存地址，它们指向同一个对象</span>

obj2.<span class="hljs-property">name</span> = <span class="hljs-string">'Bob'</span>; <span class="hljs-comment">// 通过 obj2 修改对象</span>
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(obj1.<span class="hljs-property">name</span>); <span class="hljs-comment">// 输出 'Bob'，因为 obj1 也被改变了</span>
</code></pre>
</li>
</ol>
<p>这个特性引出了<strong>浅拷贝 (Shallow Copy)</strong> 和 <strong>深拷贝 (Deep Copy)</strong> 的概念。</p>
<ul>
<li>
<p><strong>浅拷贝</strong>: 只复制对象的第一层属性。如果属性值是原始类型，就复制值；如果属性值是引用类型，就只复制其内存地址。这意味着拷贝后的对象和原对象的嵌套引用类型属性，仍然指向同一个对象。<code>Object.assign()</code> 和展开语法 <code>...</code> 都是浅拷贝。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">let</span> obj1 = { <span class="hljs-attr">name</span>: <span class="hljs-string">'Alice'</span>, <span class="hljs-attr">details</span>: { <span class="hljs-attr">age</span>: <span class="hljs-number">30</span> } };
<span class="hljs-keyword">let</span> obj2 = { ...obj1 }; <span class="hljs-comment">// 使用展开语法进行浅拷贝</span>

obj2.<span class="hljs-property">name</span> = <span class="hljs-string">'Bob'</span>; <span class="hljs-comment">// 修改第一层，不影响 obj1</span>
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(obj1.<span class="hljs-property">name</span>); <span class="hljs-comment">// 'Alice'</span>

obj2.<span class="hljs-property">details</span>.<span class="hljs-property">age</span> = <span class="hljs-number">35</span>; <span class="hljs-comment">// 修改嵌套对象</span>
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(obj1.<span class="hljs-property">details</span>.<span class="hljs-property">age</span>); <span class="hljs-comment">// 35，obj1 也被修改了！</span>
</code></pre>
</li>
<li>
<p><strong>深拷贝</strong>: 完全创建一个新的、独立的对象，递归地复制原对象的所有层级的属性。拷贝后的对象与原对象没有任何关联，修改任何一方都不会影响另一方。</p>
</li>
</ul>
<h3 id="实现深拷贝的几种方式"><a href="#实现深拷贝的几种方式">实现深拷贝的几种方式</a></h3>
<h4 id="方式一jsonstringify-简单但不完美的捷径"><a href="#方式一jsonstringify-简单但不完美的捷径">方式一：<code>JSON.stringify</code> (简单但不完美的捷径)</a></h4>
<p>这是最广为人知也最简单的“深拷贝”方法。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">function</span> <span class="hljs-title function_">jsonDeepCopy</span>(<span class="hljs-params">obj</span>) {
  <span class="hljs-keyword">return</span> <span class="hljs-title class_">JSON</span>.<span class="hljs-title function_">parse</span>(<span class="hljs-title class_">JSON</span>.<span class="hljs-title function_">stringify</span>(obj));

  <span class="hljs-attr">name</span>: <span class="hljs-string">'Alice'</span>,
  <span class="hljs-attr">details</span>: { <span class="hljs-attr">age</span>: <span class="hljs-number">30</span> },
  <span class="hljs-attr">hobbies</span>: [<span class="hljs-string">'reading'</span>, <span class="hljs-string">'coding'</span>],
  <span class="hljs-attr">joinDate</span>: <span class="hljs-keyword">new</span> <span class="hljs-title class_">Date</span>(),

  <span class="hljs-attr">nothing</span>: <span class="hljs-literal">undefined</span>

<span class="hljs-keyword">const</span> obj2 = <span class="hljs-title function_">jsonDeepCopy</span>(obj1);

</code></pre>
<p><strong>优点</strong>:</p>
<ul>
<li>
<p>非常简单，一行代码搞定。</p>
</li>
<li>
<p>能处理常见的 JSON 安全数据（字符串、数字、布尔、数组、普通对象）。</p>
</li>
</ul>
<p><strong>致命缺点</strong>:</p>
<ul>
<li>
<p><strong>忽略 <code>undefined</code> 和 <code>symbol</code></strong>: 如果对象的属性值是 <code>undefined</code> 或 <code>symbol</code>，该属性会在序列化过程中丢失。</p>
</li>
<li>
<p><strong>忽略函数</strong>: 属性值是函数时，该属性也会丢失。</p>
</li>
<li>
<p><strong>转换 <code>Date</code> 对象</strong>: <code>Date</code> 对象会被转换为 ISO 格式的字符串，而不是一个新的 <code>Date</code> 对象。</p>
</li>
<li>
<p><strong>无法处理 <code>RegExp</code></strong>: 正则表达式对象会变为空对象 <code>{}</code>。</p>
</li>
<li>
<p><strong>无法处理循环引用</strong>: 如果对象之间存在循环引用（例如 <code>a.b = b; b.a = a;</code>），该方法会直接抛出 <code>TypeError</code> 错误。</p>
</li>
<li>
<p><strong>无法处理 <code>Map</code>, <code>Set</code> 等数据结构</strong>。</p>
</li>
</ul>
<p><strong>结论</strong>：此方法仅适用于数据结构简单、确定不包含上述特殊类型且没有循环引用的场景。在生产环境中需要慎用。</p>

<h4 id="方式二递归实现-彻底理解原理"><a href="#方式二递归实现-彻底理解原理">方式二：递归实现 (彻底理解原理)</a></h4>
<p>要实现一个真正健壮的深拷贝，我们需要手动编写一个递归函数。下面我们从一个基础版本开始，逐步完善它，以处理各种边界情况。</p>
<p><strong>V1.0: 基础递归版本 (仅处理普通对象和数组)</strong></p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">function</span> <span class="hljs-title function_">basicDeepCopy</span>(<span class="hljs-params">obj</span>) {
  <span class="hljs-comment">// 如果是原始类型或者 null，直接返回</span>
  <span class="hljs-keyword">if</span> (<span class="hljs-keyword">typeof</span> obj !== <span class="hljs-string">'object'</span> || obj === <span class="hljs-literal">null</span>) {
    <span class="hljs-keyword">return</span> obj;
  }

  <span class="hljs-comment">// 判断是数组还是对象</span>
  <span class="hljs-keyword">let</span> newObj = <span class="hljs-title class_">Array</span>.<span class="hljs-title function_">isArray</span>(obj) ? [] : {};

  <span class="hljs-comment">// 遍历对象的属性（包括原型链上的）</span>
  <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> key <span class="hljs-keyword">in</span> obj) {
    <span class="hljs-comment">// 只拷贝对象自身的属性</span>
    <span class="hljs-keyword">if</span> (<span class="hljs-title class_">Object</span>.<span class="hljs-property"><span class="hljs-keyword">prototype</span></span>.<span class="hljs-property">hasOwnProperty</span>.<span class="hljs-title function_">call</span>(obj, key)) {
      <span class="hljs-comment">// 递归拷贝属性值</span>
      newObj[key] = <span class="hljs-title function_">basicDeepCopy</span>(obj[key]);
    }
  }

  <span class="hljs-keyword">return</span> newObj;
}
</code></pre>
<p>这个版本已经能正确处理嵌套的对象和数组了，但它仍然有 V1.0 的问题：<strong>循环引用</strong>。</p>
<p>如果 <code>obj.self = obj</code>，那么 <code>basicDeepCopy(obj)</code> 将会无限递归下去，直到栈溢出。</p>
<p><strong>V2.0: 解决循环引用问题</strong></p>
<p>解决方案是使用一个 <code>Map</code> (或 <code>WeakMap</code>) 来存储已经拷贝过的对象。在拷贝一个新对象之前，先检查 <code>Map</code> 中是否已经存在，如果存在，则直接返回其对应的拷贝，避免重复拷贝和无限递归。</p>
<p><code>WeakMap</code> 在这里是更好的选择，因为它对键是弱引用，当原对象没有其他引用时，垃圾回收机制可以正常回收它，避免内存泄漏。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">function</span> <span class="hljs-title function_">deepCopyWithCircularRef</span>(<span class="hljs-params">obj, cache = <span class="hljs-keyword">new</span> <span class="hljs-built_in">WeakMap</span>()</span>) {
  <span class="hljs-comment">// 原始类型或 null</span>
  <span class="hljs-keyword">if</span> (<span class="hljs-keyword">typeof</span> obj !== <span class="hljs-string">'object'</span> || obj === <span class="hljs-literal">null</span>) {
    <span class="hljs-keyword">return</span> obj;
  }

  <span class="hljs-comment">// 如果已经拷贝过，直接返回缓存中的拷贝</span>
  <span class="hljs-keyword">if</span> (cache.<span class="hljs-title function_">has</span>(obj)) {
    <span class="hljs-keyword">return</span> cache.<span class="hljs-title function_">get</span>(obj);
  }

  <span class="hljs-comment">// 判断数据类型</span>
  <span class="hljs-keyword">let</span> newObj = <span class="hljs-title class_">Array</span>.<span class="hljs-title function_">isArray</span>(obj) ? [] : {};

  <span class="hljs-comment">// 在递归之前，将新对象存入缓存</span>
  <span class="hljs-comment">// 这样即使在递归中遇到循环引用，也能从缓存中获取</span>
  cache.<span class="hljs-title function_">set</span>(obj, newObj);

  <span class="hljs-comment">// Reflect.ownKeys 可以获取所有类型的键名（包括 Symbol）</span>

  });

  <span class="hljs-keyword">return</span> newObj;
}
</code></pre>
<p>这个版本已经相当不错了，解决了最大的循环引用问题。但它还不够“彻底”，因为它没有处理 <code>Date</code>, <code>RegExp</code>, <code>Set</code>, <code>Map</code> 等其他引用类型。</p>
<p><strong>V3.0: 完整的深拷贝实现</strong></p>
<p>让我们扩展 V2.0，使其能够处理更多的数据类型。</p>

<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(obj2.<span class="hljs-property">self</span> === obj2); <span class="hljs-comment">// true</span>
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(obj1.<span class="hljs-property">self</span> === obj2.<span class="hljs-property">self</span>); <span class="hljs-comment">// false，循环引用也被正确处理</span>
</code></pre>
<p>这个 V3.0 版本已经非常健壮，它：</p>
<ul>
<li>
<p>正确处理了原始类型和函数。</p>
</li>
<li>
<p>使用 <code>WeakMap</code> 优雅地解决了循环引用问题。</p>
</li>
<li>
<p>能够处理 <code>Date</code>, <code>RegExp</code>, <code>Set</code>, <code>Map</code> 等多种内置对象。</p>
</li>
<li>
<p>通过 <code>Reflect.ownKeys</code> 遍历所有类型的键。</p>
</li>
<li>
<p>通过 <code>new obj.constructor()</code> 保留了对象的原型链。</p>
</li>
</ul>
<hr>
<h4 id="方式三structuredclone-现代浏览器的内置武器"><a href="#方式三structuredclone-现代浏览器的内置武器">方式三：<code>structuredClone()</code> (现代浏览器的内置武器)</a></h4>
<p>现代浏览器和 Node.js (v17+) 提供了一个原生的、标准的深拷贝 API：<code>structuredClone()</code>。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">const</span> obj1 = {
  <span class="hljs-attr">name</span>: <span class="hljs-string">'Alice'</span>,
  <span class="hljs-attr">details</span>: { <span class="hljs-attr">age</span>: <span class="hljs-number">30</span> },
  <span class="hljs-attr">joinDate</span>: <span class="hljs-keyword">new</span> <span class="hljs-title class_">Date</span>(),
  <span class="hljs-attr">regex</span>: <span class="hljs-regexp">/abc/g</span>,
  <span class="hljs-attr">map</span>: <span class="hljs-keyword">new</span> <span class="hljs-title class_">Map</span>([[<span class="hljs-string">'a'</span>, <span class="hljs-number">1</span>]])
};
obj1.<span class="hljs-property">circular</span> = obj1; <span class="hljs-comment">// 循环引用</span>

<span class="hljs-keyword">const</span> obj2 = <span class="hljs-title function_">structuredClone</span>(obj1);

obj2.<span class="hljs-property">details</span>.<span class="hljs-property">age</span> = <span class="hljs-number">35</span>;
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(obj1.<span class="hljs-property">details</span>.<span class="hljs-property">age</span>); <span class="hljs-comment">// 30</span>
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(obj2.<span class="hljs-property">details</span>.<span class="hljs-property">age</span>); <span class="hljs-comment">// 35</span>

<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(obj2.<span class="hljs-property">circular</span> === obj2); <span class="hljs-comment">// true</span>
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(obj2.<span class="hljs-property">circular</span> === obj1.<span class="hljs-property">circular</span>); <span class="hljs-comment">// false</span>
</code></pre>
<p><code>structuredClone</code> 使用了<a href="https://developer.mozilla.org/en-US/docs/Web/API/Web_Workers_API/Structured_clone_algorithm">结构化克隆算法</a>，这个算法本来是用于在 Web Workers 之间传递数据的。它非常强大：</p>
<ul>
<li>
<p><strong>优点</strong>:</p>
<ul>
<li>
<p><strong>原生标准</strong>：这是官方推荐的方式，无需引入任何库。</p>
</li>
<li>
<p><strong>性能优秀</strong>：通常比手写的 JavaScript 实现更快。</p>
</li>
<li>
<p><strong>功能强大</strong>：支持循环引用、支持大量内置类型（<code>Date</code>, <code>RegExp</code>, <code>Map</code>, <code>Set</code>, <code>Blob</code>, <code>File</code>, <code>ImageData</code> 等）。</p>
</li>
</ul>
</li>
<li>
<p><strong>缺点</strong>:</p>
<ul>
<li>
<p><strong>不支持函数</strong>: 和 <code>JSON.stringify</code> 一样，如果尝试克隆包含函数的对象，会抛出 <code>DataCloneError</code> 错误。</p>
</li>
<li>
<p><strong>不克隆原型链</strong>: 它不会复制对象的原型链。</p>
</li>
<li>
<p><strong>兼容性</strong>: 在一些旧的浏览器或 Node.js 版本中不可用。</p>
</li>
</ul>
</li>
</ul>
<hr>
<h4 id="方式四使用成熟的第三方库"><a href="#方式四使用成熟的第三方库">方式四：使用成熟的第三方库</a></h4>
<p>比如 <code>lodash</code> 库的 <code>_.cloneDeep()</code> 方法。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-comment">// 需要先安装 lodash: npm install lodash</span>
<span class="hljs-keyword">import</span> _ <span class="hljs-keyword">from</span> <span class="hljs-string">'lodash'</span>;

<span class="hljs-keyword">const</span> obj1 = { <span class="hljs-comment">/* ... 复杂对象 ... */</span> };
<span class="hljs-keyword">const</span> obj2 = _.<span class="hljs-title function_">cloneDeep</span>(obj1);
</code></pre>
<p><strong>优点</strong>:</p>
<ul>
<li>
<p><strong>极其健壮</strong>：经过了大量的测试，处理了无数开发者可能都想不到的边界情况。</p>
</li>
<li>
<p><strong>兼容性好</strong>：可以在几乎所有的 JavaScript 环境中运行。</p>
</li>
<li>
<p><strong>功能全面</strong>：通常比手写的实现或 <code>structuredClone</code> 支持更多不常见的类型。</p>
</li>
</ul>
<p><strong>缺点</strong>:</p>
<ul>
<li><strong>增加依赖</strong>：需要为项目引入一个额外的库，增加了打包体积。</li>
</ul>
<h3 id="总结与最终推荐"><a href="#总结与最终推荐">总结与最终推荐</a></h3>
<p>理解了以上所有方法后，我们可以给出清晰的决策路径：</p>
<ol>
<li>
<p>最佳实践与首选: structuredClone()</p>
<p>在你的目标环境（浏览器、Node.js）支持的情况下，这应该是你的第一选择。它是原生的、高效的，并且功能足够强大，可以满足绝大多数的深拷贝需求。不能拷贝函数通常不是问题，因为拷贝函数本身通常是一种不好的实践（函数是行为，不是数据）。</p>
</li>
<li>
<p>兼容性与终极方案: Lodash 的 _.cloneDeep()</p>
<p>如果你的项目需要兼容旧版浏览器，或者你需要拷贝一些 structuredClone 也不支持的极其复杂的对象，那么使用一个像 Lodash 这样经过实战检验的库是最安全、最省心的选择。</p>
</li>
<li>
<p>学习与面试: 手写递归实现</p>
<p>理解并能够写出 V3.0 版本的深拷贝函数，是衡量一个 JavaScript 开发者基础是否扎实的绝佳标尺。它能全面考察你对数据类型、内存模型、递归、循环引用、Map/WeakMap 以及各种内置对象特性的理解。</p>
</li>
<li>
<p>临时与特定场景: JSON.parse(JSON.stringify())</p>
<p>只有当你完全确定你的数据是 "JSON-safe" 的（不包含函数、undefined、Symbol、Date等），并且没有循环引用时，才可以使用这个方法图个方便。在不确定的情况下，避免使用它。</p>
</li>
