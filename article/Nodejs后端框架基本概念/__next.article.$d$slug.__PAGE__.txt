<h3 id="一-核心概念什么是-nodejs-与后端框架"><a href="#一-核心概念什么是-nodejs-与后端框架">一、 核心概念：什么是 Node.js 与后端框架？</a></h3>
<h4 id="1-nodejs-的本质"><a href="#1-nodejs-的本质">1. Node.js 的本质</a></h4>
<p>Node.js 不是一门语言，也不是一个框架，而是一个 <strong>JavaScript 运行时环境 (Runtime)</strong>。它让 JavaScript 脱离了浏览器，能够在服务器端运行。</p>
<p>从底层架构来看，Node.js 主要由两部分组成：</p>
<ul>
<li>
<p><strong>V8 引擎：</strong> Google 开发的 JavaScript 引擎，负责将 JS 代码编译成机器码并执行。</p>
</li>
<li>
<p><strong>libuv 库：</strong> 一个多平台支持的 C 库，负责提供<strong>异步 I/O</strong> 的能力和<strong>事件循环 (Event Loop)</strong> 机制。</p>
</li>
</ul>
<p><strong>核心工作流：</strong> 你的 JS 代码是单线程执行的。当遇到 I/O 操作（如读写文件、网络请求、数据库查询）时，Node.js 会将这些耗时任务交给操作系统的底层线程池（通过 libuv）去处理，主线程继续向下执行。当底层任务完成后，将回调函数推入事件队列，主线程在空闲时取出并执行。这就是<strong>单线程、非阻塞 I/O、事件驱动</strong>的本质。</p>
<h4 id="2-为什么需要后端框架"><a href="#2-为什么需要后端框架">2. 为什么需要后端框架？</a></h4>
<p>Node.js 提供了底层的 <code>http</code> 模块来搭建服务器，但原生的 API 非常底层，处理路由解析、参数提取、请求头处理等繁琐且易错。</p>
<p>后端框架（如 Express、Koa、NestJS）的作用是封装这些底层 API，提供标准化的处理流程。</p>
<p><strong>最常用的框架：Express.js</strong></p>
<p>对于初中级工程师以及中小型项目，Express 是最基础且工业界使用最广泛的框架。它的核心思想只有两个：</p>
<ol>
<li>
<p><strong>路由 (Routing)：</strong> 将特定的 HTTP 方法 (GET/POST) 和 URL 路径映射到对应的处理函数。</p>
</li>
<li>
<p><strong>中间件 (Middleware)：</strong> 一个流水线模式。请求进入后，按照定义顺序依次经过一系列函数（处理身份验证、日志、解析 Body 等），最终到达路由处理函数并返回响应。</p>
</li>
</ol>

<h3 id="三-nodejs-面试核心考点系统梳理"><a href="#三-nodejs-面试核心考点系统梳理">三、 Node.js 面试核心考点系统梳理</a></h3>
<p>针对初中级工程师，面试官考察的核心是你是否真正理解了 Node.js 的运行机制，而不仅仅是会调用 API。</p>
<h4 id="1-解释-nodejs-的事件循环-event-loop它与浏览器的事件循环有什么区别"><a href="#1-解释-nodejs-的事件循环-event-loop它与浏览器的事件循环有什么区别">1. 解释 Node.js 的事件循环 (Event Loop)？它与浏览器的事件循环有什么区别？</a></h4>
<ul>
<li>
<p><strong>解答：</strong></p>
<p>Node.js 的事件循环是由 libuv 实现的，它包含多个阶段（Phases），按照顺序反复执行：</p>
<ol>
<li>
<p><strong>Timers (定时器阶段):</strong> 执行 <code>setTimeout</code> 和 <code>setInterval</code> 的回调。</p>
</li>
<li>
<p><strong>Pending Callbacks:</strong> 执行一些系统级别的回调（如 TCP 错误）。</p>
</li>
<li>
<p><strong>Idle, Prepare:</strong> 仅系统内部使用。</p>
</li>
<li>
<p><strong>Poll (轮询阶段):</strong> <strong>核心阶段</strong>。获取新的 I/O 事件，执行 I/O 相关的回调（如文件读写完成、网络请求到达）。如果当前队列为空，且没有设置 setImmediate，Node.js 会在这个阶段阻塞等待新的事件到来。</p>
</li>
<li>
<p><strong>Check (检查阶段):</strong> 执行 <code>setImmediate</code> 的回调。</p>
</li>
<li>
<p><strong>Close Callbacks:</strong> 执行关闭事件的回调（如 <code>socket.on('close', ...)</code>）。</p>
</li>
</ol>
</li>
<li>
<p><strong>微任务 (Microtasks)：</strong> <code>Promise.then</code> 和 <code>process.nextTick</code>。它们不在上述六个阶段内，而是在<strong>每个阶段切换的间隙</strong>（或者每个宏任务执行后）被立即清空执行。<code>process.nextTick</code> 的优先级高于 Promise。</p>
</li>
<li>
<p><strong>与浏览器的区别：</strong> 浏览器的事件循环相对简单，主要是宏任务队列和微任务队列的交替执行。Node.js 引入了多个阶段的概念，对不同类型的异步操作进行了更细粒度的分类和优先级划分。</p>
</li>
</ul>
<h4 id="2-nodejs-是单线程的为什么能处理高并发"><a href="#2-nodejs-是单线程的为什么能处理高并发">2. Node.js 是单线程的，为什么能处理高并发？</a></h4>
<ul>
<li>
<p><strong>解答：</strong></p>
<p>这里的“单线程”仅仅指 <strong>JavaScript 代码的执行主线程</strong>是单线程的。</p>
<p>Node.js 处理高并发的逻辑是：当主线程遇到 I/O 操作（如查数据库）时，不会停下来等待结果（非阻塞），而是将任务交给底层的 <strong>libuv 线程池</strong>（默认 4 个线程，可配置）或操作系统的异步接口。主线程立刻去处理下一个网络请求。当 I/O 任务完成后，带着结果回到事件队列中排队，主线程空闲时再执行后续的回调逻辑。</p>
<p><strong>结论：</strong> Node.js 依靠“单线程的主逻辑调度 + 底层多线程/异步 I/O 的任务执行 + 事件循环的回收机制”，以极低的内存消耗实现了高并发的网络请求处理。</p>
</li>
</ul>
<h4 id="3-什么是内存泄漏在-nodejs-中通常是怎么引起的如何排查"><a href="#3-什么是内存泄漏在-nodejs-中通常是怎么引起的如何排查">3. 什么是内存泄漏？在 Node.js 中通常是怎么引起的？如何排查？</a></h4>
<ul>
<li>
<p><strong>解答：</strong></p>
<p>内存泄漏是指程序中已动态分配的堆内存由于某种原因未释放或无法释放，导致系统内存持续消耗最终崩溃（OOM - Out of Memory）。</p>
<p><strong>常见原因：</strong></p>
<ol>
<li>
<p><strong>全局变量滥用：</strong> 挂载在 <code>global</code> 上的大对象一直不被清理。</p>
</li>
<li>
<p><strong>闭包：</strong> 闭包作用域内保持了对外部庞大对象的引用，导致垃圾回收器 (GC) 无法回收。</p>
</li>
<li>
<p><strong>事件监听器未清理：</strong> 使用 <code>EventEmitter</code> 注册了大量事件但未调用 <code>removeListener</code>。</p>
</li>
<li>
<p><strong>缓存无上限：</strong> 在内存中实现缓存（如普通 Object 或 Map），但没有设置过期时间或最大容量。</p>
<p><strong>排查思路：</strong></p>
<p>使用工具（如 Node.js 内置的 <code>--inspect</code> 配合 Chrome DevTools，或者 <code>heapdump</code> 库）在内存占用正常和偏高时分别抓取 Heap Snapshot（堆快照），通过对比两次快照中新增的对象，找出没有被释放的变量持有者。</p>
</li>
</ol>
</li>
</ul>
<h4 id="4-express-中间件的-next-有什么作用如果不调用会怎样"><a href="#4-express-中间件的-next-有什么作用如果不调用会怎样">4. Express 中间件的 <code>next()</code> 有什么作用？如果不调用会怎样？</a></h4>
<ul>
<li>
<p><strong>解答：</strong></p>
<p>在 Express 中，一个请求可能需要经过验证身份、解析数据、写入日志等多个步骤，每个步骤就是一个中间件。<code>next()</code> 是一个函数，调用它意味着将控制权移交给<strong>流水线上的下一个中间件或路由处理函数</strong>。</p>
<p>如果在一个中间件中<strong>既没有调用 <code>next()</code>，也没有结束请求</strong>（如使用 <code>res.send()</code> 或 <code>res.json()</code> 返回响应），该请求将被<strong>挂起 (Pending)</strong>，客户端会一直处于等待状态直到超时。如果给 <code>next</code> 传递了参数，如 <code>next(new Error())</code>，Express 会跳过后续的常规中间件，直接将请求交给全局错误处理中间件。</p>
</li>
</ul>
