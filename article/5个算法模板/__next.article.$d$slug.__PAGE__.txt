4:T35d7,<p>想要用一个“万能模板”直接套用 LeetCode Hot 100 的所有题目是不切实际的，因为算法的本质是针对不同数据结构和业务场景的时间/空间复杂度优化。图的搜索、数组的排序、树的遍历在物理内存和逻辑推演上是完全不同的维度。</p>
<p>不过，如果我们将 Hot 100 的题目进行解构，会发现它们高度集中在几个核心的<strong>逻辑模式</strong>上。掌握这几个模式的“骨架”，然后根据具体题目填充血肉，才是通关的最优解。</p>
<p>以下是涵盖 Hot 100 中近 80% 题目的 5 大核心套路模板（JavaScript 实现）及其底层思路：</p>
<h2 id="1-数组字符串滑动窗口与双指针模板"><a href="#1-数组字符串滑动窗口与双指针模板">1. 数组/字符串：滑动窗口与双指针模板</a></h2>
<p><strong>核心思路</strong>：这类题目通常要求在连续区间内寻找最值（最长子串、最小覆盖子串等）。与其使用 $O(N^2)$ 的暴力双重循环，不如维护一个像毛毛虫一样爬行的窗口。右指针主动扩张寻找可行解，左指针被动收缩优化可行解。</p>
<p><strong>适用题目</strong>：无重复字符的最长子串、最小覆盖子串、盛最多水的容器、三数之和。</p>

<h2 id="2-穷举与搜索回溯算法dfs模板"><a href="#2-穷举与搜索回溯算法dfs模板">2. 穷举与搜索：回溯算法（DFS）模板</a></h2>
<p><strong>核心思路</strong>：解决“全排列”、“组合”、“子集”问题的本质是遍历一棵决策树。回溯的核心就三步：做选择 -> 递归进入下一层 -> 撤销选择。它穷尽了所有可能性。</p>
<p><strong>适用题目</strong>：全排列、子集、组合总和、电话号码的字母组合、括号生成。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">function</span> <span class="hljs-title function_">backtrackGenerator</span>(<span class="hljs-params">nums</span>) {
    <span class="hljs-keyword">const</span> res = [];
    
    <span class="hljs-comment">// path: 当前走过的路径（已做的选择）</span>
    <span class="hljs-comment">// startIndex: 防止组合重复（排列问题通常不需要，或者用 visited 数组）</span>
    <span class="hljs-keyword">function</span> <span class="hljs-title function_">backtrack</span>(<span class="hljs-params">path, startIndex</span>) {
        <span class="hljs-comment">// 1. 触发结束条件（到达树的叶子节点）</span>
        <span class="hljs-keyword">if</span> (<span class="hljs-comment">/* 满足某个条件，如 path.length === nums.length */</span>) {
            res.<span class="hljs-title function_">push</span>([...path]); <span class="hljs-comment">// 必须深拷贝，否则后续的 pop 会清空结果</span>
            <span class="hljs-keyword">return</span>; 
        }

        <span class="hljs-comment">// 2. 遍历当前层的所有选择</span>
        <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> i = startIndex; i &#x3C; nums.<span class="hljs-property">length</span>; i++) {
            <span class="hljs-comment">// 剪枝逻辑：跳过不合法的选择</span>
            <span class="hljs-comment">// if (!isValid(nums[i])) continue;</span>

            <span class="hljs-comment">// 3. 做选择</span>
            path.<span class="hljs-title function_">push</span>(nums[i]);
            
            <span class="hljs-comment">// 4. 进入下一层决策树</span>
            <span class="hljs-title function_">backtrack</span>(path, i + <span class="hljs-number">1</span>); <span class="hljs-comment">// 组合问题传 i+1，如果是可以重复选的传 i</span>
            
            <span class="hljs-comment">// 5. 撤销选择（回溯，恢复状态）</span>
            path.<span class="hljs-title function_">pop</span>();
        }
    }

    <span class="hljs-title function_">backtrack</span>([], <span class="hljs-number">0</span>);
    <span class="hljs-keyword">return</span> res;
}

<h2 id="3-树结构二叉树递归遍历模板"><a href="#3-树结构二叉树递归遍历模板">3. 树结构：二叉树递归遍历模板</a></h2>
<p><strong>核心思路</strong>：二叉树题目的本质是<strong>降维打击</strong>。把一个庞大的树结构问题，分解为一个节点与其左右子节点的问题。关键在于明确当前节点需要做什么，以及这个动作应该发生在遍历前（前序）、遍历中（中序）还是遍历后（后序）。</p>
<p><strong>适用题目</strong>：翻转二叉树、二叉树的最大深度、二叉树的最近公共祖先、二叉树展开为链表。</p>

<h2 id="4-有序空间二分查找模板"><a href="#4-有序空间二分查找模板">4. 有序空间：二分查找模板</a></h2>
<p><strong>核心思路</strong>：二分查找不仅用于“在有序数组中找数字”，更强大的是用来寻找“满足某种条件的边界”。只要发现问题具备“单调性”（比如：某个值可行，比它小的值一定可行，比它大的值一定不可行），就可以用二分。</p>
<p><strong>适用题目</strong>：搜索旋转排序数组、寻找两个正序数组的中位数、在排序数组中查找元素的第一个和最后一个位置。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">function</span> <span class="hljs-title function_">binarySearch</span>(<span class="hljs-params">nums, target</span>) {
    <span class="hljs-comment">// 1. 定义搜索区间 [left, right]</span>
    <span class="hljs-keyword">let</span> left = <span class="hljs-number">0</span>;
    <span class="hljs-keyword">let</span> right = nums.<span class="hljs-property">length</span> - <span class="hljs-number">1</span>;

    <span class="hljs-comment">// 2. 当 left &#x3C;= right 时区间不为空</span>
    <span class="hljs-keyword">while</span> (left &#x3C;= right) {
        <span class="hljs-comment">// 防止 (left + right) 溢出，并向下取整</span>
        <span class="hljs-keyword">let</span> mid = left + <span class="hljs-title class_">Math</span>.<span class="hljs-title function_">floor</span>((right - left) / <span class="hljs-number">2</span>);

        <span class="hljs-keyword">if</span> (nums[mid] === target) {
            <span class="hljs-comment">// 找到目标值</span>
            <span class="hljs-keyword">return</span> mid; 
            <span class="hljs-comment">// 如果是寻找左边界，这里改为 right = mid - 1;</span>
            <span class="hljs-comment">// 如果是寻找右边界，这里改为 left = mid + 1;</span>
        } <span class="hljs-keyword">else</span> <span class="hljs-keyword">if</span> (nums[mid] &#x3C; target) {
            <span class="hljs-comment">// 目标在右侧，收缩左边界</span>
            left = mid + <span class="hljs-number">1</span>;
        } <span class="hljs-keyword">else</span> <span class="hljs-keyword">if</span> (nums[mid] > target) {
            <span class="hljs-comment">// 目标在左侧，收缩右边界</span>
            right = mid - <span class="hljs-number">1</span>;
        }
    }
    
    <span class="hljs-keyword">return</span> -<span class="hljs-number">1</span>; <span class="hljs-comment">// 或返回 left 作为插入位置</span>
}

<h2 id="5-链表结构虚拟头节点dummy-head与快慢指针模板"><a href="#5-链表结构虚拟头节点dummy-head与快慢指针模板">5. 链表结构：虚拟头节点（Dummy Head）与快慢指针模板</a></h2>
<p><strong>核心思路</strong>：链表题的难点往往在于处理头节点被删除或修改的边界情况，以及无法预知链表长度。引入一个指向原始头节点的 <code>dummy</code> 节点可以抹平所有边界差异；快慢指针则可以优雅地解决环形检测、寻找中点等问题。</p>
<p><strong>适用题目</strong>：反转链表、合并两个有序链表、删除链表的倒数第 N 个结点、环形链表。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">function</span> <span class="hljs-title function_">processLinkedList</span>(<span class="hljs-params">head</span>) {

    <span class="hljs-comment">// 5. 返回时忽略 dummy 本身</span>
