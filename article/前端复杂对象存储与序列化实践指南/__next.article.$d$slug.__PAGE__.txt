4:T317c,<h2 id="一-核心痛点为什么原生-json-会引发-bug"><a href="#一-核心痛点为什么原生-json-会引发-bug">一、 核心痛点：为什么原生 JSON 会引发 Bug？</a></h2>
<p>前端在处理状态持久化时，通常习惯使用 <code>JSON.stringify()</code> 将对象转换为字符串并存入 <code>localStorage</code> 或 <code>sessionStorage</code>。但 JSON 本质上是一种跨语言的轻量级<strong>文本数据交换格式</strong>，它并不是为了完整保存 JavaScript 内存中的对象状态而设计的。</p>
<p>这就导致了两个难以回避的客观边界：</p>
<ol>
<li>
<p><strong>类型丢失与静默变异</strong>：</p>
<ul>
<li>
<p><code>undefined</code>、<code>Function</code>、<code>Symbol</code> 会在转换时被直接忽略。</p>
</li>
<li>
<p><code>NaN</code>、<code>Infinity</code> 会被强制转换为 <code>null</code>。</p>
</li>
<li>
<p><code>Date</code> 对象会被转换为 ISO 字符串，且解析时无法自动还原为 Date 实例。</p>
</li>
<li>
<p><code>Map</code>、<code>Set</code>、<code>RegExp</code>、<code>Error</code> 等复杂对象会被转换为无意义的空对象 <code>{}</code>。</p>
</li>
</ul>
</li>
<li>
<p><strong>循环引用导致崩溃</strong>：</p>
<ul>
<li>当对象结构中存在相互引用（例如对象 A 的属性指向 B，B 的属性又指回 A）时，<code>JSON.stringify</code> 无法处理这种内存指针的闭环，会直接抛出 <code>TypeError: Converting circular structure to JSON</code> 异常。</li>
</ul>
</li>
</ol>
<hr>
<h2 id="二-解决方案与技术路径"><a href="#二-解决方案与技术路径">二、 解决方案与技术路径</a></h2>
<p>要突破上述边界，工程实践中通常有三种演进路径。我们需要根据具体的业务场景、数据规模以及对包体积的敏感度来做决策。</p>
<h3 id="路径一原生-api-的深度定制轻量级干预"><a href="#路径一原生-api-的深度定制轻量级干预">路径一：原生 API 的深度定制（轻量级干预）</a></h3>
<p>利用 <code>JSON.stringify</code> 和 <code>JSON.parse</code> 内置的 <code>replacer</code> 和 <code>reviver</code> 参数，建立一套自定义的序列化规则，用“魔法标记”来记录丢失的类型信息。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-comment">// 1. 序列化：将特殊类型转换为带标记的普通对象</span>

    <span class="hljs-keyword">if</span> (origValue <span class="hljs-keyword">instanceof</span> <span class="hljs-title class_">Date</span>) <span class="hljs-keyword">return</span> { <span class="hljs-attr">_type</span>: <span class="hljs-string">'Date'</span>, <span class="hljs-attr">val</span>: origValue.<span class="hljs-title function_">getTime</span>() };

    <span class="hljs-keyword">if</span> (origValue <span class="hljs-keyword">instanceof</span> <span class="hljs-title class_">RegExp</span>) <span class="hljs-keyword">return</span> { <span class="hljs-attr">_type</span>: <span class="hljs-string">'RegExp'</span>, <span class="hljs-attr">val</span>: origValue.<span class="hljs-property">source</span>, <span class="hljs-attr">flags</span>: origValue.<span class="hljs-property">flags</span> };

<ul>
<li>
<p><strong>优势</strong>：零外部依赖，完全可控，不增加打包体积。</p>
</li>
<li>
<p><strong>劣势</strong>：无法处理循环引用；需要手动维护大量的类型映射，容易遗漏边界条件。</p>
</li>
</ul>
<h3 id="路径二引入专业序列化库极致的字符串转换"><a href="#路径二引入专业序列化库极致的字符串转换">路径二：引入专业序列化库（极致的字符串转换）</a></h3>
<p>将类型推断和内存指针的扁平化处理交给成熟的第三方工具，实现代码层的“完美且简洁”。</p>
<ol>
<li>
<p><strong><code>superjson</code>（专注类型保留）</strong>：</p>
<p>在底层构建元数据映射，将复杂对象转换为带有类型标签的字符串。完美支持 <code>Date</code>、<code>Map</code>、<code>Set</code>、<code>BigInt</code> 等，API 完全对标原生 JSON。</p>
</li>
<li>
<p><strong><code>devalue</code>（专注解决循环引用）</strong>：</p>
<p>不仅保留类型，还能精确重建对象间的内存互相引用关系。常用于复杂状态机或服务端渲染（SSR）的数据脱水与注水。</p>
</li>
</ol>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">import</span> superjson <span class="hljs-keyword">from</span> <span class="hljs-string">'superjson'</span>;
<span class="hljs-keyword">import</span> * <span class="hljs-keyword">as</span> devalue <span class="hljs-keyword">from</span> <span class="hljs-string">'devalue'</span>;

<span class="hljs-comment">// superjson 转换</span>
<span class="hljs-keyword">const</span> str1 = superjson.<span class="hljs-title function_">stringify</span>(obj);
<span class="hljs-keyword">const</span> restored1 = superjson.<span class="hljs-title function_">parse</span>(str1);

<span class="hljs-comment">// devalue 转换（处理循环引用）</span>
<span class="hljs-keyword">const</span> str2 = devalue.<span class="hljs-title function_">stringify</span>(obj);
<span class="hljs-keyword">const</span> restored2 = devalue.<span class="hljs-title function_">parse</span>(str2);
</code></pre>
<ul>
<li>
<p><strong>优势</strong>：开发体验极佳，彻底解决类型丢失和循环引用报错。</p>
</li>
<li>
<p><strong>劣势</strong>：带来了额外的包体积；由于需要构建和解析元数据，CPU 运算耗时高于原生 JSON。在大规模数据高频读写时，可能产生微小的性能损耗。</p>
</li>
</ul>
<h3 id="路径三降维打击indexeddb-与结构化克隆"><a href="#路径三降维打击indexeddb-与结构化克隆">路径三：降维打击——IndexedDB 与结构化克隆</a></h3>
<p>如果最终目的是“存储数据”而非“通过网络传输字符串”，最严谨的做法是放弃强制字符串转换，直接使用浏览器底层的 <code>IndexedDB</code>。</p>
<p>它默认采用<strong>结构化克隆算法 (Structured Clone Algorithm)</strong>：</p>
<ul>
<li>
<p>原生支持直接存入 <code>Map</code>、<code>Set</code>、<code>Date</code>、<code>ArrayBuffer</code> 甚至 <code>Blob</code>。</p>
</li>
<li>
<p>原生支持并正确解析循环引用。</p>
</li>
</ul>
<p>为了抹平底层 API 的复杂性，通常配合 <code>localforage</code> 等封装库使用：</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">import</span> localforage <span class="hljs-keyword">from</span> <span class="hljs-string">'localforage'</span>;

<span class="hljs-comment">// 极其复杂且包含循环引用的对象</span>
<span class="hljs-keyword">const</span> complexState = { <span class="hljs-attr">map</span>: <span class="hljs-keyword">new</span> <span class="hljs-title class_">Map</span>(), <span class="hljs-attr">date</span>: <span class="hljs-keyword">new</span> <span class="hljs-title class_">Date</span>() };
complexState.<span class="hljs-property">self</span> = complexState;

<span class="hljs-comment">// 直接写入，无需转换</span>
<span class="hljs-keyword">await</span> localforage.<span class="hljs-title function_">setItem</span>(<span class="hljs-string">'app_state'</span>, complexState);

<span class="hljs-comment">// 直接读取，保持内存结构和类型完好</span>
<span class="hljs-keyword">const</span> state = <span class="hljs-keyword">await</span> localforage.<span class="hljs-title function_">getItem</span>(<span class="hljs-string">'app_state'</span>);
</code></pre>
<ul>
<li>
<p><strong>优势</strong>：零 CPU 序列化开销，是最符合计算机系统底层逻辑的复杂数据存储方案。</p>
</li>
<li>
<p><strong>劣势</strong>：需要处理异步（Promise）逻辑；无法用于必须要求同步读取的极早期渲染阶段。</p>
</li>
</ul>
