    <html><head></head><body><h1>rsync 用法教程</h1>
<p>作者： <a href="https://www.ruanyifeng.com/">阮一峰</a></p>
<p>日期： <a href="https://www.ruanyifeng.com/blog/2020/08/">2020年8月26日</a></p>
<h2>一、简介</h2>
<p>rsync 是一个常用的 Linux 应用程序，用于文件同步。</p>
<p>它可以在本地计算机与远程计算机之间，或者两个本地目录之间同步文件（但不支持两台远程计算机之间的同步）。它也可以当作文件复制工具，替代<code><span class="hljs-keyword">cp</span></code>和<code><span class="hljs-built_in">mv</span></code>命令。</p>
<p><img src="https://www.wangbase.com/blogimg/asset/202008/bg2020082602.jpg" alt=""></p>
<p>它名称里面的<code><span class="hljs-attribute">r</span></code>指的是 remote，rsync 其实就是"远程同步"（remote sync）的意思。与其他文件传输工具（如 FTP 或 scp）不同，rsync 的最大特点是会检查发送方和接收方已有的文件，仅传输有变动的部分（默认规则是文件大小或修改时间有变动）。</p>
<h2>二、安装</h2>
<p>如果本机或者远程计算机没有安装 rsync，可以用下面的命令安装。</p>
<blockquote>
<pre><code class="language-bash">
<span class="hljs-comment"># Debian</span>
<span class="hljs-variable">$ </span>sudo apt-get install rsync

<span class="hljs-comment"># Red Hat</span>
<span class="hljs-variable">$ </span>sudo yum install rsync

<span class="hljs-comment"># Arch Linux</span>
<span class="hljs-variable">$ </span>sudo pacman -S rsync
</code></pre>
</blockquote>
<p>注意，传输的双方都必须安装 rsync。</p>
<h2>三、基本用法</h2>
<h3>3.1 <code><span class="hljs-deletion">-r</span></code> 参数</h3>
<p>本机使用 rsync 命令时，可以作为<code><span class="hljs-keyword">cp</span></code>和<code><span class="hljs-built_in">mv</span></code>命令的替代方法，将源目录同步到目标目录。</p>
<blockquote>
<pre><code class="language-bash"><span class="hljs-meta prompt_">
$ </span><span class="language-bash">rsync -r <span class="hljs-built_in">source</span> destination</span>
</code></pre>
</blockquote>
<p>上面命令中，<code><span class="hljs-deletion">-r</span></code>表示递归，即包含子目录。注意，<code><span class="hljs-deletion">-r</span></code>是必须的，否则 rsync 运行不会成功。<code><span class="hljs-built_in">source</span></code>目录表示源目录，<code><span class="hljs-attribute">destination</span></code>表示目标目录。</p>
<p>如果有多个文件或目录需要同步，可以写成下面这样。</p>
<blockquote>
<pre><code class="language-bash">
<span class="hljs-variable">$ </span>rsync -r source1 source2 destination
</code></pre>
</blockquote>
<p>上面命令中，<code>source1</code>、<code>source2</code>都会被同步到<code><span class="hljs-attribute">destination</span></code>目录。</p>
<h3>3.2 <code>-<span class="hljs-selector-tag">a</span></code> 参数</h3>
<p><code>-<span class="hljs-selector-tag">a</span></code>参数可以替代<code><span class="hljs-deletion">-r</span></code>，除了可以递归同步以外，还可以同步元信息（比如修改时间、权限等）。由于 rsync 默认使用文件大小和修改时间决定文件是否需要更新，所以<code>-<span class="hljs-selector-tag">a</span></code>比<code><span class="hljs-deletion">-r</span></code>更有用。下面的用法才是常见的写法。</p>
<blockquote>
<pre><code class="language-bash"><span class="hljs-meta prompt_">
$ </span><span class="language-bash">rsync -a <span class="hljs-built_in">source</span> destination</span>
</code></pre>
</blockquote>
<p>目标目录<code><span class="hljs-attribute">destination</span></code>如果不存在，rsync 会自动创建。执行上面的命令后，源目录<code><span class="hljs-built_in">source</span></code>被完整地复制到了目标目录<code><span class="hljs-attribute">destination</span></code>下面，即形成了<code>destination/source</code>的目录结构。</p>
<p>如果只想同步源目录<code><span class="hljs-built_in">source</span></code>里面的内容到目标目录<code><span class="hljs-attribute">destination</span></code>，则需要在源目录后面加上斜杠。</p>
<blockquote>
<pre><code class="language-bash"><span class="hljs-meta prompt_">
$ </span><span class="language-bash">rsync -a <span class="hljs-built_in">source</span>/ destination</span>
</code></pre>
</blockquote>
<p>上面命令执行后，<code><span class="hljs-built_in">source</span></code>目录里面的内容，就都被复制到了<code><span class="hljs-attribute">destination</span></code>目录里面，并不会在<code><span class="hljs-attribute">destination</span></code>下面创建一个<code><span class="hljs-built_in">source</span></code>子目录。</p>
<h3>3.3 <code><span class="hljs-deletion">-n</span></code> 参数</h3>
<p>如果不确定 rsync 执行后会产生什么结果，可以先用<code><span class="hljs-deletion">-n</span></code>或<code><span class="hljs-comment">--dry-run</span></code>参数模拟执行的结果。</p>
<blockquote>
<pre><code class="language-bash"><span class="hljs-meta prompt_">
$ </span><span class="language-bash">rsync -anv <span class="hljs-built_in">source</span>/ destination</span>
</code></pre>
</blockquote>
<p>上面命令中，<code><span class="hljs-deletion">-n</span></code>参数模拟命令执行的结果，并不真的执行命令。<code><span class="hljs-deletion">-v</span></code>参数则是将结果输出到终端，这样就可以看到哪些内容会被同步。</p>
<h3>3.4 <code>--<span class="hljs-keyword">delete</span></code> 参数</h3>
<p>默认情况下，rsync 只确保源目录的所有内容（明确排除的文件除外）都复制到目标目录。它不会使两个目录保持相同，并且不会删除文件。如果要使得目标目录成为源目录的镜像副本，则必须使用<code>--<span class="hljs-keyword">delete</span></code>参数，这将删除只存在于目标目录、不存在于源目录的文件。</p>
<blockquote>
<pre><code class="language-bash">
$ rsync -av --<span class="hljs-keyword">delete</span> <span class="hljs-keyword">source</span>/ destination
</code></pre>
</blockquote>
<p>上面命令中，<code>--<span class="hljs-keyword">delete</span></code>参数会使得<code><span class="hljs-attribute">destination</span></code>成为<code><span class="hljs-built_in">source</span></code>的一个镜像。</p>
<h2>四、排除文件</h2>
<h3>4.1 <code><span class="hljs-comment">--exclude</span></code> 参数</h3>
<p>有时，我们希望同步时排除某些文件或目录，这时可以用<code><span class="hljs-comment">--exclude</span></code>参数指定排除模式。</p>
<blockquote>
<pre><code class="language-bash"><span class="hljs-meta prompt_">
$ </span><span class="language-bash">rsync -av --exclude=<span class="hljs-string">'*.txt'</span> <span class="hljs-built_in">source</span>/ destination</span>
<span class="hljs-meta prompt_"># </span><span class="language-bash">或者</span>
<span class="hljs-meta prompt_">$ </span><span class="language-bash">rsync -av --exclude <span class="hljs-string">'*.txt'</span> <span class="hljs-built_in">source</span>/ destination</span>
</code></pre>
</blockquote>
<p>上面命令排除了所有 TXT 文件。</p>
<p>注意，rsync 会同步以"点"开头的隐藏文件，如果要排除隐藏文件，可以这样写<code><span class="hljs-attr">--exclude</span>=<span class="hljs-string">".*"</span></code>。</p>
<p>如果要排除某个目录里面的所有文件，但不希望排除目录本身，可以写成下面这样。</p>
<blockquote>
<pre><code class="language-bash">
$ rsync -av --<span class="hljs-keyword">exclude</span> <span class="hljs-string">'dir1/*'</span> <span class="hljs-keyword">source</span>/ destination
</code></pre>
</blockquote>
<p>多个排除模式，可以用多个<code><span class="hljs-comment">--exclude</span></code>参数。</p>
<blockquote>
<pre><code class="language-bash">
$ rsync -av --<span class="hljs-keyword">exclude</span> <span class="hljs-string">'file1.txt'</span> --<span class="hljs-keyword">exclude</span> <span class="hljs-string">'dir1/*'</span> <span class="hljs-keyword">source</span>/ destination
</code></pre>
</blockquote>
<p>多个排除模式也可以利用 Bash 的大扩号的扩展功能，只用一个<code><span class="hljs-comment">--exclude</span></code>参数。</p>
<blockquote>
<pre><code class="language-bash">
$ rsync -av --<span class="hljs-keyword">exclude</span>={<span class="hljs-string">'file1.txt'</span>,<span class="hljs-string">'dir1/*'</span>} <span class="hljs-keyword">source</span>/ destination
</code></pre>
</blockquote>
<p>如果排除模式很多，可以将它们写入一个文件，每个模式一行，然后用<code>--<span class="hljs-keyword">exclude</span>-<span class="hljs-keyword">from</span></code>参数指定这个文件。</p>
<blockquote>
<pre><code class="language-bash">
$ rsync -av --<span class="hljs-keyword">exclude</span>-<span class="hljs-keyword">from</span>=<span class="hljs-string">'exclude-file.txt'</span> <span class="hljs-keyword">source</span>/ destination
</code></pre>
</blockquote>
<h3>4.2 <code>-<span class="hljs-keyword">-include</span></code> 参数</h3>
<p><code>-<span class="hljs-keyword">-include</span></code>参数用来指定必须同步的文件模式，往往与<code><span class="hljs-comment">--exclude</span></code>结合使用。</p>
<blockquote>
<pre><code class="language-bash">
$ rsync -av --<span class="hljs-keyword">include</span>=<span class="hljs-string">"*.txt"</span> --<span class="hljs-keyword">exclude</span>=<span class="hljs-string">'*'</span> <span class="hljs-keyword">source</span>/ destination
</code></pre>
</blockquote>
<p>上面命令指定同步时，排除所有文件，但是会包括 TXT 文件。</p>
<h2>五、远程同步</h2>
<h3>5.1 SSH 协议</h3>
<p>rsync 除了支持本地两个目录之间的同步，也支持远程同步。它可以将本地内容，同步到远程服务器。</p>
<blockquote>
<pre><code class="language-bash">
<span class="hljs-variable">$ </span>rsync -av source/ username<span class="hljs-variable">@remote_host</span><span class="hljs-symbol">:destination</span>
</code></pre>
</blockquote>
<p>也可以将远程内容同步到本地。</p>
<blockquote>
<pre><code class="language-bash">
<span class="hljs-variable">$ </span>rsync -av username<span class="hljs-variable">@remote_host</span><span class="hljs-symbol">:source/</span> destination
</code></pre>
</blockquote>
<p>rsync 默认使用 SSH 进行远程登录和数据传输。</p>
<p>由于早期 rsync 不使用 SSH 协议，需要用<code><span class="hljs-deletion">-e</span></code>参数指定协议，后来才改的。所以，下面<code><span class="hljs-deletion">-e ssh</span></code>可以省略。</p>
<blockquote>
<pre><code class="language-bash">
<span class="hljs-variable">$ </span>rsync -av -e ssh source/ user<span class="hljs-variable">@remote_host</span><span class="hljs-symbol">:/destination</span>
</code></pre>
</blockquote>
<p>但是，如果 ssh 命令有附加的参数，则必须使用<code><span class="hljs-deletion">-e</span></code>参数指定所要执行的 SSH 命令。</p>
<blockquote>
<pre><code class="language-bash">
<span class="hljs-variable">$ </span>rsync -av -e <span class="hljs-string">'ssh -p 2234'</span> source/ user<span class="hljs-variable">@remote_host</span><span class="hljs-symbol">:/destination</span>
</code></pre>
</blockquote>
<p>上面命令中，<code><span class="hljs-deletion">-e</span></code>参数指定 SSH 使用2234端口。</p>
<h3>5.2 rsync 协议</h3>
<p>除了使用 SSH，如果另一台服务器安装并运行了 rsync 守护程序，则也可以用<code>rsync:<span class="hljs-regexp">//</span></code>协议（默认端口873）进行传输。具体写法是服务器与目标目录之间使用双冒号分隔<code>::</code>。</p>
<blockquote>
<pre><code class="language-bash">
<span class="hljs-variable">$ </span>rsync -av source/ <span class="hljs-number">192.168</span>.<span class="hljs-number">122.32</span><span class="hljs-symbol">:</span><span class="hljs-symbol">:module/destination</span>
</code></pre>
</blockquote>
<p>注意，上面地址中的<code><span class="hljs-keyword">module</span></code>并不是实际路径名，而是 rsync 守护程序指定的一个资源名，由管理员分配。</p>
<p>如果想知道 rsync 守护程序分配的所有 module 列表，可以执行下面命令。</p>
<blockquote>
<pre><code class="language-bash">
$ rsync rsync:<span class="hljs-regexp">//</span><span class="hljs-number">192.168</span>.<span class="hljs-number">122.32</span>
</code></pre>
</blockquote>
<p>rsync 协议除了使用双冒号，也可以直接用<code>rsync:<span class="hljs-regexp">//</span></code>协议指定地址。</p>
<blockquote>
<pre><code class="language-bash">
$ rsync -av <span class="hljs-keyword">source</span><span class="hljs-regexp">/ rsync:/</span><span class="hljs-regexp">/192.168.122.32/m</span>odule/destination
</code></pre>
</blockquote>
<h2>六、增量备份</h2>
<p>rsync 的最大特点就是它可以完成增量备份，也就是默认只复制有变动的文件。</p>
<p>除了源目录与目标目录直接比较，rsync 还支持使用基准目录，即将源目录与基准目录之间变动的部分，同步到目标目录。</p>
<p>具体做法是，第一次同步是全量备份，所有文件在基准目录里面同步一份。以后每一次同步都是增量备份，只同步源目录与基准目录之间有变动的部分，将这部分保存在一个新的目标目录。这个新的目标目录之中，也是包含所有文件，但实际上，只有那些变动过的文件是存在于该目录，其他没有变动的文件都是指向基准目录文件的硬链接。</p>
<p><code><span class="hljs-comment">--link-dest</span></code>参数用来指定同步时的基准目录。</p>
<blockquote>
<pre><code class="language-bash">
$ rsync -a --<span class="hljs-keyword">delete</span> --link-dest <span class="hljs-regexp">/compare/</span>path <span class="hljs-regexp">/source/</span>path <span class="hljs-regexp">/target/</span>path
</code></pre>
</blockquote>
<p>上面命令中，<code><span class="hljs-comment">--link-dest</span></code>参数指定基准目录<code><span class="hljs-regexp">/compare/</span>path</code>，然后源目录<code><span class="hljs-regexp">/source/</span>path</code>跟基准目录进行比较，找出变动的文件，将它们拷贝到目标目录<code><span class="hljs-regexp">/target/</span>path</code>。那些没变动的文件则会生成硬链接。这个命令的第一次备份时是全量备份，后面就都是增量备份了。</p>
<p>下面是一个脚本示例，备份用户的主目录。</p>
<blockquote>
<pre><code class="language-bash">
<span class="hljs-comment">#!/bin/bash</span>

<span class="hljs-comment"># A script to perform incremental backups using rsync</span>

<span class="hljs-built_in">set</span> -o errexit
<span class="hljs-built_in">set</span> -o nounset
<span class="hljs-built_in">set</span> -o pipefail

readonly <span class="hljs-attribute">SOURCE_DIR</span>=<span class="hljs-string">"<span class="hljs-variable">${HOME}</span>"</span>
readonly <span class="hljs-attribute">BACKUP_DIR</span>=<span class="hljs-string">"/mnt/data/backups"</span>
readonly <span class="hljs-attribute">DATETIME</span>=<span class="hljs-string">"<span class="hljs-variable">$(date '+%Y-%m-%d_%H:%M:%S')</span>"</span>
readonly <span class="hljs-attribute">BACKUP_PATH</span>=<span class="hljs-string">"<span class="hljs-variable">${BACKUP_DIR}</span>/<span class="hljs-variable">${DATETIME}</span>"</span>
readonly <span class="hljs-attribute">LATEST_LINK</span>=<span class="hljs-string">"<span class="hljs-variable">${BACKUP_DIR}</span>/latest"</span>

mkdir -p <span class="hljs-string">"<span class="hljs-variable">${BACKUP_DIR}</span>"</span>

rsync -av --delete \
  <span class="hljs-string">"<span class="hljs-variable">${SOURCE_DIR}</span>/"</span> \
  --link-dest <span class="hljs-string">"<span class="hljs-variable">${LATEST_LINK}</span>"</span> \
  <span class="hljs-attribute">--exclude</span>=<span class="hljs-string">".cache"</span> \
  <span class="hljs-string">"<span class="hljs-variable">${BACKUP_PATH}</span>"</span>

rm -rf <span class="hljs-string">"<span class="hljs-variable">${LATEST_LINK}</span>"</span>
ln -s <span class="hljs-string">"<span class="hljs-variable">${BACKUP_PATH}</span>"</span> <span class="hljs-string">"<span class="hljs-variable">${LATEST_LINK}</span>"</span>
</code></pre>
</blockquote>
<p>上面脚本中，每一次同步都会生成一个新目录<code><span class="hljs-variable">${BACKUP_DIR}</span>/<span class="hljs-variable">${DATETIME}</span></code>，并将软链接<code><span class="hljs-variable">${BACKUP_DIR}</span>/latest</code>指向这个目录。下一次备份时，就将<code><span class="hljs-variable">${BACKUP_DIR}</span>/latest</code>作为基准目录，生成新的备份目录。最后，再将软链接<code><span class="hljs-variable">${BACKUP_DIR}</span>/latest</code>指向新的备份目录。</p>
<h2>七、配置项</h2>
<p><code>-<span class="hljs-selector-tag">a</span></code>、<code><span class="hljs-comment">--archive</span></code>参数表示存档模式，保存所有的元数据，比如修改时间（modification time）、权限、所有者等，并且软链接也会同步过去。</p>
<p><code><span class="hljs-comment">--append</span></code>参数指定文件接着上次中断的地方，继续传输。</p>
<p><code>--<span class="hljs-built_in">append</span>-<span class="hljs-built_in">verify</span></code>参数跟<code><span class="hljs-comment">--append</span></code>参数类似，但会对传输完成后的文件进行一次校验。如果校验失败，将重新发送整个文件。</p>
<p><code>-<span class="hljs-selector-tag">b</span></code>、<code><span class="hljs-comment">--backup</span></code>参数指定在删除或更新目标目录已经存在的文件时，将该文件更名后进行备份，默认行为是删除。更名规则是添加由<code><span class="hljs-comment">--suffix</span></code>参数指定的文件后缀名，默认是<code>~</code>。</p>
<p><code><span class="hljs-comment">--backup-dir</span></code>参数指定文件备份时存放的目录，比如<code>--backup-dir=<span class="hljs-regexp">/path/</span>to/backups</code>。</p>
<p><code><span class="hljs-comment">--bwlimit</span></code>参数指定带宽限制，默认单位是 KB/s，比如<code><span class="hljs-attr">--bwlimit</span>=<span class="hljs-number">100</span></code>。</p>
<p><code><span class="hljs-deletion">-c</span></code>、<code><span class="hljs-comment">--checksum</span></code>参数改变<code><span class="hljs-attribute">rsync</span></code>的校验方式。默认情况下，rsync 只检查文件的大小和最后修改日期是否发生变化，如果发生变化，就重新传输；使用这个参数以后，则通过判断文件内容的校验和，决定是否重新传输。</p>
<p><code>--<span class="hljs-keyword">delete</span></code>参数删除只存在于目标目录、不存在于源目标的文件，即保证目标目录是源目标的镜像。</p>
<p><code><span class="hljs-deletion">-e</span></code>参数指定使用 SSH 协议传输数据。</p>
<p><code><span class="hljs-comment">--exclude</span></code>参数指定排除不进行同步的文件，比如<code><span class="hljs-attr">--exclude</span>=<span class="hljs-string">"*.iso"</span></code>。</p>
<p><code>--<span class="hljs-keyword">exclude</span>-<span class="hljs-keyword">from</span></code>参数指定一个本地文件，里面是需要排除的文件模式，每个模式一行。</p>
<p><code><span class="hljs-comment">--existing</span></code>、<code><span class="hljs-comment">--ignore-non-existing</span></code>参数表示不同步目标目录中不存在的文件和目录。</p>
<p><code><span class="hljs-deletion">-h</span></code>参数表示以人类可读的格式输出。</p>
<p><code><span class="hljs-deletion">-h</span></code>、<code><span class="hljs-comment">--help</span></code>参数返回帮助信息。</p>
<p><code>-<span class="hljs-selector-tag">i</span></code>参数表示输出源目录与目标目录之间文件差异的详细情况。</p>
<p><code><span class="hljs-comment">--ignore-existing</span></code>参数表示只要该文件在目标目录中已经存在，就跳过去，不再同步这些文件。</p>
<p><code>-<span class="hljs-keyword">-include</span></code>参数指定同步时要包括的文件，一般与<code><span class="hljs-comment">--exclude</span></code>结合使用。</p>
<p><code><span class="hljs-comment">--link-dest</span></code>参数指定增量备份的基准目录。</p>
<p><code><span class="hljs-deletion">-m</span></code>参数指定不同步空目录。</p>
<p><code>--max-size</code>参数设置传输的最大文件的大小限制，比如不超过200KB（<code><span class="hljs-attr">--max-size</span>=<span class="hljs-string">'200k'</span></code>）。</p>
<p><code>--min-size</code>参数设置传输的最小文件的大小限制，比如不小于10KB（<code>--min-size=<span class="hljs-number">10</span>k</code>）。</p>
<p><code><span class="hljs-deletion">-n</span></code>参数或<code><span class="hljs-comment">--dry-run</span></code>参数模拟将要执行的操作，而并不真的执行。配合<code><span class="hljs-deletion">-v</span></code>参数使用，可以看到哪些内容会被同步过去。</p>
<p><code>-<span class="hljs-selector-tag">P</span></code>参数是<code><span class="hljs-comment">--progress</span></code>和<code><span class="hljs-comment">--partial</span></code>这两个参数的结合。</p>
<p><code><span class="hljs-comment">--partial</span></code>参数允许恢复中断的传输。不使用该参数时，<code><span class="hljs-attribute">rsync</span></code>会删除传输到一半被打断的文件；使用该参数后，传输到一半的文件也会同步到目标目录，下次同步时再恢复中断的传输。一般需要与<code><span class="hljs-comment">--append</span></code>或<code>--<span class="hljs-built_in">append</span>-<span class="hljs-built_in">verify</span></code>配合使用。</p>
<p><code><span class="hljs-comment">--partial-dir</span></code>参数指定将传输到一半的文件保存到一个临时目录，比如<code>--<span class="hljs-keyword">partial</span>-dir=.rsync-<span class="hljs-keyword">partial</span></code>。一般需要与<code><span class="hljs-comment">--append</span></code>或<code>--<span class="hljs-built_in">append</span>-<span class="hljs-built_in">verify</span></code>配合使用。</p>
<p><code><span class="hljs-comment">--progress</span></code>参数表示显示进展。</p>
<p><code><span class="hljs-deletion">-r</span></code>参数表示递归，即包含子目录。</p>
<p><code>--<span class="hljs-built_in">remove</span>-<span class="hljs-keyword">source</span>-<span class="hljs-keyword">files</span></code>参数表示传输成功后，删除发送方的文件。</p>
<p><code>--<span class="hljs-built_in">size</span>-<span class="hljs-keyword">only</span></code>参数表示只同步大小有变化的文件，不考虑文件修改时间的差异。</p>
<p><code><span class="hljs-comment">--suffix</span></code>参数指定文件名备份时，对文件名添加的后缀，默认是<code>~</code>。</p>
<p><code><span class="hljs-deletion">-u</span></code>、<code><span class="hljs-comment">--update</span></code>参数表示同步时跳过目标目录中修改时间更新的文件，即不同步这些有更新的时间戳的文件。</p>
<p><code><span class="hljs-deletion">-v</span></code>参数表示输出细节。<code><span class="hljs-deletion">-vv</span></code>表示输出更详细的信息，<code><span class="hljs-deletion">-vvv</span></code>表示输出最详细的信息。</p>
<p><code><span class="hljs-comment">--version</span></code>参数返回 rsync 的版本。</p>
<p><code><span class="hljs-deletion">-z</span></code>参数指定同步时压缩数据。</p>
