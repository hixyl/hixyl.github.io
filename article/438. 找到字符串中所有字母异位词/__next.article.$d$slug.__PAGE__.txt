<h4 id="题目描述"><a href="#题目描述">题目描述</a></h4>
<p>给定两个字符串 <code>s</code> 和 <code>p</code>，找到 <code>s</code> 中所有 <code>p</code> 的 <strong>异位词</strong> 的子串，返回这些子串的起始索引。不考虑答案输出的顺序。</p>
<p><strong>异位词</strong> 指由相同字母重排列而成的字符串（包括相同的字符串）。</p>
<hr>
<h4 id="示例"><a href="#示例">示例</a></h4>
<p><strong>示例 1:1</strong></p>
<pre><code class="hljs language-makefile"><span class="hljs-section">输入: s = "cbaebabacd", p = "abc"</span>
<span class="hljs-section">输出: [0,6]</span>
<span class="hljs-section">解释:</span>
起始索引等于 0 的子串是 <span class="hljs-string">"cba"</span>, 它是 <span class="hljs-string">"abc"</span> 的异位词。
起始索引等于 6 的子串是 <span class="hljs-string">"bac"</span>, 它是 <span class="hljs-string">"abc"</span> 的异位词。
</code></pre>
<p><strong>示例 2:2</strong></p>
<pre><code class="hljs language-makefile"><span class="hljs-section">输入: s = "abab", p = "ab"</span>
<span class="hljs-section">输出: [0,1,2]</span>
<span class="hljs-section">解释:</span>
起始索引等于 0 的子串是 <span class="hljs-string">"ab"</span>, 它是 <span class="hljs-string">"ab"</span> 的异位词。
起始索引等于 1 的子串是 <span class="hljs-string">"ba"</span>, 它是 <span class="hljs-string">"ab"</span> 的异位词。
起始索引等于 2 的子串是 <span class="hljs-string">"ab"</span>, 它是 <span class="hljs-string">"ab"</span> 的异位词。
</code></pre>
<hr>
<h4 id="提示3"><a href="#提示3">提示:3</a></h4>
<ul>
<li>
<p><code>1 &#x3C;= s.length, p.length &#x3C;= 3 * 10^4</code></p>
</li>
<li>
<p><code>s</code> 和 <code>p</code> 仅包含小写字母</p>
</li>
</ul>
<hr>
<h3 id="解题思路"><a href="#解题思路">解题思路</a></h3>
<p>这道题是“滑动窗口”算法的典型应用场景。核心思想是在字符串 <code>s</code> 上维护一个固定大小（与 <code>p</code> 的长度相同）的窗口，然后不断向右滑动这个窗口。在滑动的每一步，我们都检查窗口内的子串是否是 <code>p</code> 的一个异位词。</p>
<p>一个子串是 <code>p</code> 的异位词，等价于这个子串中所有字符的种类和数量与 <code>p</code> 完全一致。</p>
<p>直接的暴力解法是遍历 <code>s</code> 的所有长度为 <code>p.length</code> 的子串，然后对每个子串进行排序或计次来判断是否为异位词，但这样做效率太低。</p>
<p>滑动窗口算法的精髓在于，当窗口向右移动一格时，我们不需要重新统计整个窗口，而只需要进行<strong>增量更新</strong>：在窗口中<strong>增加</strong>右边新进入的字符，并<strong>移除</strong>左边滑出去的字符。这样可以将每次判断的复杂度从 O(p.length) 降为 O(1)。</p>
<p>为了高效地判断窗口内的字符构成是否符合要求，我们可以使用哈希表（或数组）来进行计数。</p>
<p><strong>具体步骤如下：</strong></p>
<ol>
<li>
<p><strong>初始化需求列表 (<code>need</code>)</strong>：</p>
<ul>
<li>创建一个哈希表 <code>need</code>，用来统计目标字符串 <code>p</code> 中每个字符出现的次数。</li>
</ul>
</li>
<li>
<p><strong>初始化滑动窗口 (<code>window</code>)</strong>：</p>
<ul>
<li>
<p>同样创建一个哈希表 <code>window</code>，用于统计当前在滑动窗口内的子串的字符频率。</p>
</li>
<li>
<p>设置两个指针 <code>left</code> 和 <code>right</code>，初始都指向 <code>s</code> 的开头，表示窗口的左右边界。</p>
</li>
<li>
<p>引入一个计数器 <code>valid</code>，用于记录当前 <code>window</code> 中有多少种字符的数量已经满足了 <code>need</code> 的要求。当 <code>valid</code> 的值等于 <code>need</code> 中不同字符的总数时，就意味着我们找到了一个异位词。</p>
</li>
</ul>
</li>
<li>
<p><strong>滑动与判断</strong>：</p>
<ul>
<li>
<p>不断向右移动 <code>right</code> 指针，将 <code>s[right]</code> 纳入窗口，并更新 <code>window</code> 中的计数。</p>
</li>
<li>
<p>如果 <code>s[right]</code> 是 <code>p</code> 中需要的字符，并且在加入后，它在 <code>window</code> 中的数量恰好等于在 <code>need</code> 中的数量，那么 <code>valid</code> 加一。</p>
</li>
<li>
<p>当窗口的大小 (<code>right - left</code>) 达到 <code>p.length</code> 时，说明窗口已满，需要开始判断和收缩。</p>
<ul>
<li>
<p><strong>判断</strong>：如果此时 <code>valid</code> 的值等于 <code>need</code> 中不同字符的总数，说明当前窗口内的子串是一个异位词，将 <code>left</code> 的值存入结果列表。</p>
</li>
<li>
<p><strong>收缩</strong>：将 <code>left</code> 指针右移一格，准备让窗口向右滑动。在移动前，需要更新 <code>window</code> 和 <code>valid</code> 的状态：如果即将移出窗口的字符 <code>s[left]</code> 是 <code>p</code> 所需的，并且它在 <code>window</code> 中的数量原本是“达标”的，那么在移出后 <code>valid</code> 就需要减一。同时，<code>window</code> 中对应字符的计数也减一。</p>
</li>
</ul>
</li>
</ul>
</li>
<li>
<p><strong>循环</strong>：重复第 3 步，直到 <code>right</code> 指针越过字符串 <code>s</code> 的末尾。</p>
</li>
</ol>
<h3 id="代码实现-javascript"><a href="#代码实现-javascript">代码实现 (JavaScript)</a></h3>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-comment">/**
 * <span class="hljs-doctag">@param</span> {<span class="hljs-type">string</span>} <span class="hljs-variable">s</span>
 * <span class="hljs-doctag">@param</span> {<span class="hljs-type">string</span>} <span class="hljs-variable">p</span>
 * <span class="hljs-doctag">@return</span> {<span class="hljs-type">number[]</span>}
 */</span>
<span class="hljs-keyword">var</span> findAnagrams = <span class="hljs-keyword">function</span>(<span class="hljs-params">s, p</span>) {
    <span class="hljs-comment">// 最终返回的结果数组</span>
    <span class="hljs-keyword">const</span> result = [];
    <span class="hljs-comment">// needMap 用于记录目标字符串 p 中需要的字符及其数量</span>
    <span class="hljs-keyword">const</span> needMap = <span class="hljs-keyword">new</span> <span class="hljs-title class_">Map</span>();
    <span class="hljs-comment">// windowMap 用于记录当前滑动窗口中已有字符的数量</span>
    <span class="hljs-keyword">const</span> windowMap = <span class="hljs-keyword">new</span> <span class="hljs-title class_">Map</span>();

    <span class="hljs-comment">// 1. 初始化 needMap</span>
    <span class="hljs-keyword">for</span> (<span class="hljs-keyword">const</span> char <span class="hljs-keyword">of</span> p) {
        needMap.<span class="hljs-title function_">set</span>(char, (needMap.<span class="hljs-title function_">get</span>(char) || <span class="hljs-number">0</span>) + <span class="hljs-number">1</span>);
    }

    <span class="hljs-comment">// 记录 p 中不同字符的种类数，用于判断窗口是否满足条件</span>
    <span class="hljs-keyword">const</span> needSize = needMap.<span class="hljs-property">size</span>;
    <span class="hljs-comment">// validCount 用于记录当前窗口中，已经满足 needMap 数量要求的字符种类数</span>
    <span class="hljs-keyword">let</span> validCount = <span class="hljs-number">0</span>;
    <span class="hljs-comment">// left 和 right 是滑动窗口的左右指针</span>
    <span class="hljs-keyword">let</span> left = <span class="hljs-number">0</span>, right = <span class="hljs-number">0</span>;

    <span class="hljs-comment">// 2. 开始滑动窗口，right 指针负责扩大窗口</span>
    <span class="hljs-keyword">while</span> (right &#x3C; s.<span class="hljs-property">length</span>) {
        <span class="hljs-comment">// c 是即将移入窗口的字符</span>
        <span class="hljs-keyword">const</span> c = s[right];
        right++;

        <span class="hljs-comment">// 3. [窗口扩张] 更新窗口内数据</span>
        <span class="hljs-comment">// 如果 c 是 p 中需要的字符，则更新 windowMap 和 validCount</span>
        <span class="hljs-keyword">if</span> (needMap.<span class="hljs-title function_">has</span>(c)) {
            windowMap.<span class="hljs-title function_">set</span>(c, (windowMap.<span class="hljs-title function_">get</span>(c) || <span class="hljs-number">0</span>) + <span class="hljs-number">1</span>);
            <span class="hljs-comment">// 当窗口中 c 的数量等于 p 中需要的数量时，一个字符种类已满足要求</span>
            <span class="hljs-keyword">if</span> (windowMap.<span class="hljs-title function_">get</span>(c) === needMap.<span class="hljs-title function_">get</span>(c)) {
                validCount++;
            }
        }

        <span class="hljs-comment">// 4. [窗口收缩] 判断是否需要收缩窗口</span>
        <span class="hljs-comment">// 当窗口大小 (right - left) 等于 p 的长度时，就需要判断并收缩</span>
        <span class="hljs-keyword">while</span> (right - left >= p.<span class="hljs-property">length</span>) {
            
            <span class="hljs-comment">// 4a. 检查当前窗口是否是异位词</span>
            <span class="hljs-keyword">if</span> (validCount === needSize) {
                result.<span class="hljs-title function_">push</span>(left);
            }

            <span class="hljs-comment">// d 是即将移出窗口的字符</span>
            <span class="hljs-keyword">const</span> d = s[left];
            left++;

            <span class="hljs-comment">// 4b. [收缩更新] 如果移出的字符 d 是 p 中需要的，则需要更新 windowMap 和 validCount</span>
            <span class="hljs-keyword">if</span> (needMap.<span class="hljs-title function_">has</span>(d)) {
                <span class="hljs-comment">// 如果移出前，d 的数量正好是满足需求的数量，那么 validCount 需要减 1</span>
                <span class="hljs-keyword">if</span> (windowMap.<span class="hljs-title function_">get</span>(d) === needMap.<span class="hljs-title function_">get</span>(d)) {
                    validCount--;
                }
                <span class="hljs-comment">// 窗口中 d 的数量减 1</span>
                windowMap.<span class="hljs-title function_">set</span>(d, windowMap.<span class="hljs-title function_">get</span>(d) - <span class="hljs-number">1</span>);
            }
        }
    }

    <span class="hljs-keyword">return</span> result;
};

<h3 id="复杂度分析"><a href="#复杂度分析">复杂度分析</a></h3>
<ul>
<li>
<p><strong>时间复杂度</strong>: O(L_s+L_p)，其中 L_s 是字符串 <code>s</code> 的长度，L_p 是字符串 <code>p</code> 的长度。我们首先需要 O(L_p) 的时间来构建 <code>need</code> 哈希表。之后，<code>left</code> 和 <code>right</code> 指针都各自最多遍历 <code>s</code> 一次，所以滑动窗口部分的时间复杂度是 O(L_s)。</p>
</li>
<li>
<p><strong>空间复杂度</strong>: O(K)，其中 K 是字符集的数量。在这里，<code>s</code> 和 <code>p</code> 只包含小写字母，所以 <code>K</code> 是 26。<code>needMap</code> 和 <code>windowMap</code> 最多存储 <code>K</code> 个键值对。因此，空间复杂度可以看作是 O(1)。</p>
</li>
