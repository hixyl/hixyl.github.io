4:T349e,<p>使用 <code>micro-app</code> 搭建微前端架构是一个非常务实的选择。与 <code>qiankun</code> 等基于路由劫持的框架不同，<code>micro-app</code> 的核心逻辑是基于 <strong>Web Components</strong>。</p>
<p>这意味着，你引入子应用的方式，就像在 HTML 中写一个 <code>&#x3C;div></code> 或 <code>&#x3C;iframe></code> 一样自然。这种“类原生”的隔离机制大大降低了上手门槛。</p>
<p>为了让你快速跑通流程，我们使用 <strong>React + Vite</strong> 来搭建一个最简模型：</p>
<ul>
<li>
<p><strong>主应用 (Host):</strong> 运行在 <code>http://localhost:3000</code></p>
</li>
<li>
<p><strong>子应用 (Child):</strong> 运行在 <code>http://localhost:4000</code></p>
</li>
</ul>
<hr>
<h3 id="第一步初始化项目结构"><a href="#第一步初始化项目结构">第一步：初始化项目结构</a></h3>

<p>Bash</p>

<p>Bash</p>

<h3 id="第三步配置子应用-child-app"><a href="#第三步配置子应用-child-app">第三步：配置子应用 (Child App)</a></h3>
<p>子应用本质上就是一个普通的 React 应用，但为了能被主应用加载，我们需要解决最核心的 <strong>跨域 (CORS)</strong> 问题。</p>

<h3 id="深入分析可能遇到的问题"><a href="#深入分析可能遇到的问题">深入分析：可能遇到的问题</a></h3>
<p>在实际开发中，你可能会遇到以下两个最常见的问题：</p>
<h4 id="1-typescript-报错"><a href="#1-typescript-报错">1. TypeScript 报错</a></h4>
<p>因为 <code>&#x3C;micro-app></code> 不是标准的 HTML 标签，React 的 TS 类型定义中不包含它。</p>
<p>解决方法： 在主应用的 src/vite-env.d.ts 或新建 custom.d.ts 中添加：</p>
<p>TypeScript</p>
<pre><code class="hljs language-ts"><span class="hljs-keyword">declare</span> <span class="hljs-keyword">namespace</span> <span class="hljs-title class_">JSX</span> {
  <span class="hljs-keyword">interface</span> <span class="hljs-title class_">IntrinsicElements</span> {
    <span class="hljs-string">'micro-app'</span>: <span class="hljs-built_in">any</span>
  }
}
</code></pre>
<h4 id="2-图片资源加载-404"><a href="#2-图片资源加载-404">2. 图片/资源加载 404</a></h4>
<p>当子应用被主应用加载时，如果子应用里的图片使用的是相对路径（如 <code>./assets/logo.png</code>），浏览器会基于<strong>主应用</strong>的域名（localhost:3000）去请求，导致 404。</p>
<p><strong>解决方法：</strong></p>
<ul>
<li>
<p><strong>Vite 场景：</strong> <code>micro-app</code> 的 iframe 模式通常能自动处理路径补全。</p>
</li>
<li>
<p><strong>非 iframe 模式：</strong> 你需要在子应用中设置 <code>publicPath</code>。在 <code>child-app/src/main.tsx</code> 中：</p>
<p>TypeScript</p>
<pre><code class="hljs language-ts"><span class="hljs-comment">// 如果在微前端环境中，设置 public path</span>
<span class="hljs-keyword">if</span> (<span class="hljs-variable language_">window</span>.<span class="hljs-property">__MICRO_APP_ENVIRONMENT__</span>) {
  <span class="hljs-comment">// 这一步通常构建工具配合，Vite 中较少直接操作 __webpack_public_path__</span>
  <span class="hljs-comment">// 但 micro-app 会自动处理 link 和 script 标签的路径</span>
}
</code></pre>
<p><em>最稳健的做法：</em> 在子应用中使用绝对路径引入图片，或者确保构建工具将资源处理为 Base64（如果是小图）。</p>
</li>
</ul>
<h3 id="总结"><a href="#总结">总结</a></h3>
<p>这个架构的逻辑闭环在于：<strong>主应用通过 Fetch 请求拉取子应用的 HTML，解析出其中的 CSS 和 JS 标签，然后在沙箱环境中执行这些 JS，最后将生成的 DOM 插入到 <code>&#x3C;micro-app></code> 标签内。</strong></p>
