    <html><head></head><body><p>在linux下，~/.ssh目录是ssh的配置文件路径，其中./ssh/config文件可以用于自定义ssh服务器。</p>
<h2>1.概述</h2>
<p>通常，当通过 SSH 连接到远程服务器时，您将指定远程用户名、主机名和端口。例如，要从命令行以名为port<code><span class="hljs-attribute">john</span></code>的主机的用户身份登录，您可以键入：<code><span class="hljs-attribute">dev</span>.example.com``<span class="hljs-number">2322</span></code></p>
<pre><code>ssh john@dev<span class="hljs-selector-class">.example</span><span class="hljs-selector-class">.com</span> -<span class="hljs-selector-tag">p</span> <span class="hljs-number">2322</span>
</code></pre>
<p>要使用与上面命令中提供的相同选项连接到服务器，只需键入<code><span class="hljs-attribute">ssh dev</span></code>，同时将以下行放入您的<code><span class="hljs-string">"~/.ssh/config</span></code>文件中：</p>
<pre><code class="language-ini">Host dev
    HostName dev.example.com
    <span class="hljs-keyword">User</span> <span class="hljs-title">john</span>
    Port <span class="hljs-number">2322</span>
</code></pre>
<p>现在，当您键入 时，ssh 客户端将读取配置文件并使用为主机<code><span class="hljs-attribute">ssh dev</span></code>指定的连接详细信息：<code><span class="hljs-attribute">dev</span></code></p>
<h2>2.SSH配置文件示例</h2>
<p>文件路径&nbsp;<code>~<span class="hljs-regexp">/.ssh/</span>config</code></p>
<pre><code class="language-ini">Host targaryen
    HostName <span class="hljs-number">192.168</span>.<span class="hljs-number">1.10</span>
    <span class="hljs-keyword">User</span> <span class="hljs-title">daenerys</span>
    Port <span class="hljs-number">7654</span>
    IdentityFile ~/.ssh/targaryen.key

Host tyrell
    HostName <span class="hljs-number">192.168</span>.<span class="hljs-number">10.20</span>

Host martell
    HostName <span class="hljs-number">192.168</span>.<span class="hljs-number">10.50</span>

Host *ell
    <span class="hljs-keyword">user</span> <span class="hljs-title">oberyn</span>

Host * !martell
    LogLevel <span class="hljs-literal">INF</span>O

Host *
    <span class="hljs-keyword">User</span> <span class="hljs-title">root</span>
    Compression yes
</code></pre>
<ul>
<li>
<p>当您键入<code><span class="hljs-attribute">ssh targaryen</span></code>时，ssh 客户端会读取文件并应用第一个匹配项中的选项，即<code><span class="hljs-attribute">Host targaryen</span></code>.&nbsp;然后它一个接一个地检查下一个节是否有匹配的模式。下一个匹配的是<code>Host * !martell</code>（表示除 之外的所有主机<code><span class="hljs-attribute">martell</span></code>），它将应用此节中的连接选项。最后一个定义<code><span class="hljs-attribute">Host</span> *</code>也匹配，但 ssh 客户端将仅采用该<code><span class="hljs-attribute">Compression</span></code>选项，因为该<code><span class="hljs-keyword">User</span></code>选项已在<code><span class="hljs-attribute">Host targaryen</span></code>节中定义。键入时使用的完整选项列表<code><span class="hljs-attribute">ssh targaryen</span></code>如下：</p>
<pre><code class="language-ini">HostName 192.168.1.10<span class="hljs-built_in">
User </span>daenerys<span class="hljs-built_in">
Port </span>7654
IdentityFile ~/.ssh/targaryen.key
LogLevel <span class="hljs-built_in">INFO</span>
Compression <span class="hljs-literal">yes</span>
</code></pre>
</li>
<li>
<p>运行<code><span class="hljs-attribute">ssh tyrell</span></code>匹配的主机模式时：<code><span class="hljs-attribute">Host tyrell</span></code>、<code><span class="hljs-attribute">Host</span> *ell</code>和.&nbsp;在这种情况下使用的选项是：<code>Host * !martell<span class="hljs-string">``</span>Host *</code></p>
<pre><code class="language-ini">HostName 192.168.10.20<span class="hljs-built_in">
User </span>oberyn
LogLevel <span class="hljs-built_in">INFO</span>
Compression <span class="hljs-literal">yes</span>
</code></pre>
</li>
<li>
<p>如果您运行<code><span class="hljs-attribute">ssh martell</span></code>，则匹配的主机模式是<code><span class="hljs-attribute">Host martell</span></code>：<code><span class="hljs-attribute">Host</span> *ell</code>和<code><span class="hljs-attribute">Host</span> *</code>。在这种情况下使用的选项是：</p>
<pre><code class="language-ini">HostName <span class="hljs-number">192.168</span>.<span class="hljs-number">10.50</span>
<span class="hljs-keyword">User</span> <span class="hljs-title">oberyn</span>
Compression yes
</code></pre>
</li>
<li>
<p>对于所有其他连接，ssh 客户端将使用<code>Host * !martell</code>和<code><span class="hljs-attribute">Host</span> *</code>部分中指定的选项。</p>
</li>
</ul>
<p>&nbsp;</p>
<h2>3.SSH读取顺序</h2>
<p>（1）从命令行指定的选项。 （2）<code>~<span class="hljs-regexp">/.ssh/</span>config</code>中定义的选项。 （3）<code><span class="hljs-regexp">/etc/</span>ssh/ssh_config</code>中定义的选项。 如果要覆盖单个选项，可以在命令行上指定它。例如，如果您有以下定义：</p>
<pre><code class="language-ini">Host dev
    HostName dev.example.com
    <span class="hljs-keyword">User</span> <span class="hljs-title">john</span>
    Port <span class="hljs-number">2322</span>
</code></pre>
<p>并且您想使用所有其他选项，但以用户身份连接，<code><span class="hljs-attribute">root</span></code>而不是<code><span class="hljs-attribute">john</span></code>简单地在命令行上指定用户：</p>
<pre><code><span class="hljs-attribute">ssh</span> -o <span class="hljs-string">"User=root"</span> dev
</code></pre>
<p>( <code><span class="hljs-deletion">-F</span></code>)<code><span class="hljs-attribute">configfile</span></code>选项允许您指定替代的每用户配置文件。 要告诉<code><span class="hljs-attribute">ssh</span></code>客户端忽略 ssh 配置文件中指定的所有选项，请使用： <code>ssh <span class="hljs-operator">-</span>F <span class="hljs-operator">/</span>dev<span class="hljs-operator">/</span><span class="hljs-keyword">null</span> <span class="hljs-keyword">user</span><span class="hljs-variable">@example</span>.com</code></p>
