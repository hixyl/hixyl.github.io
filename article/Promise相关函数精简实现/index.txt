1d:T2a39,<p>理解 Promise 的核心在于理解它是一个<strong>状态机</strong>和<strong>回调队列</strong>的集合。为了达到“去粗取精”的效果，我们不需要实现完整的 Promises/A+ 规范（那过于冗长），而是抓住其并发控制和状态流转的本质逻辑。</p>
<p>以下是 Promise 最核心的几个静态方法及构造逻辑的精简实现：</p>

<h2 id="1-核心机制promiseall"><a href="#1-核心机制promiseall">1. 核心机制：Promise.all</a></h2>
<p>这是面试和实战中最常见的逻辑。它的精髓在于：<strong>并行执行，串行等待，一错全错</strong>。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-title class_">Promise</span>.<span class="hljs-property">myAll</span> = <span class="hljs-keyword">function</span>(<span class="hljs-params">promises</span>) {
  <span class="hljs-keyword">return</span> <span class="hljs-keyword">new</span> <span class="hljs-title class_">Promise</span>(<span class="hljs-function">(<span class="hljs-params">resolve, reject</span>) =></span> {
    <span class="hljs-keyword">const</span> results = [];
    <span class="hljs-keyword">let</span> count = <span class="hljs-number">0</span>;
    <span class="hljs-keyword">const</span> len = promises.<span class="hljs-property">length</span>;

    <span class="hljs-keyword">if</span> (len === <span class="hljs-number">0</span>) <span class="hljs-keyword">return</span> <span class="hljs-title function_">resolve</span>([]);

    promises.<span class="hljs-title function_">forEach</span>(<span class="hljs-function">(<span class="hljs-params">p, i</span>) =></span> {
      <span class="hljs-comment">// 包装成 Promise 以处理非 Promise 值</span>

        <span class="hljs-function"><span class="hljs-params">res</span> =></span> {
          results[i] = res; <span class="hljs-comment">// 保证结果顺序</span>

        },

      );
    });
  });
};

<h2 id="2-竞速机制promiserace"><a href="#2-竞速机制promiserace">2. 竞速机制：Promise.race</a></h2>
<p>它的逻辑最纯粹：<strong>谁先完成（无论成功失败），就用谁的结果</strong>。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-title class_">Promise</span>.<span class="hljs-property">myRace</span> = <span class="hljs-keyword">function</span>(<span class="hljs-params">promises</span>) {
  <span class="hljs-keyword">return</span> <span class="hljs-keyword">new</span> <span class="hljs-title class_">Promise</span>(<span class="hljs-function">(<span class="hljs-params">resolve, reject</span>) =></span> {
    promises.<span class="hljs-title function_">forEach</span>(<span class="hljs-function"><span class="hljs-params">p</span> =></span> {
      <span class="hljs-comment">// 第一个改变状态的 Promise 会直接触发外层 Promise 的 resolve/reject</span>
      <span class="hljs-title class_">Promise</span>.<span class="hljs-title function_">resolve</span>(p).<span class="hljs-title function_">then</span>(resolve, reject);
    });
  });
};
</code></pre>

<h2 id="3-容错机制promiseallsettled"><a href="#3-容错机制promiseallsettled">3. 容错机制：Promise.allSettled</a></h2>
<p>与 <code>all</code> 不同，它不关心胜负，只关心“是否结束”。这在需要获取所有异步结果（无论好坏）时非常有用。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-title class_">Promise</span>.<span class="hljs-property">myAllSettled</span> = <span class="hljs-keyword">function</span>(<span class="hljs-params">promises</span>) {

        <span class="hljs-function"><span class="hljs-params">reason</span> =></span> ({ <span class="hljs-attr">status</span>: <span class="hljs-string">'rejected'</span>, reason })

    )
  );

</code></pre>

<p>这是“全败才败，一胜即胜”的逻辑。</p>

<h3 id="总结"><a href="#总结">总结</a></h3>
<ul>
<li>
<p><strong>all</strong>: 所有人必须成功，一个失败就完蛋。</p>
</li>
<li>
<p><strong>race</strong>: 只看第一名，不问对错。</p>
</li>
<li>
<p><strong>allSettled</strong>: 所有人报个平安（无论死活）。</p>
</li>
<li>
<p><strong>any</strong>: 只要有一个活着的就行，全死了才放弃。</p>
</li>
</ul>
