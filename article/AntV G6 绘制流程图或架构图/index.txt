1d:T379e,<p>使用 G6 绘制<strong>流程图（Flowchart）或架构图（Architecture Diagram）</strong>，与绘制社交网络图（力导向图）的逻辑完全不同。</p>
<p>社交网络图追求的是“节点不重叠，看着均匀”；而流程图和架构图追求的是**“层级分明、方向统一、模块嵌套”**。</p>
<p>要实现这一点，你需要掌握三个核心概念的组合拳：<strong>Dagre 布局</strong> + <strong>Polyline 连线</strong> + <strong>Combo 分组</strong>。</p>
<hr>
<h3 id="1-核心思维模型"><a href="#1-核心思维模型">1. 核心思维模型</a></h3>
<p>在 G6 中生成架构图，不是“画”出来的，而是“算”出来的。</p>
<ul>
<li>
<p><strong>方向感：</strong> 使用 <strong>Dagre 层次布局算法</strong>。它可以自动把节点按 <code>上->下</code> 或 <code>左->右</code> 的依赖关系排列，不需要你手动指定 (x, y)。</p>
</li>
<li>
<p><strong>规整感：</strong> 边（Edge）不能是直线，必须是 <strong>折线（Polyline）</strong>，且通常是正交的（直角拐弯），看起来才像工程图。</p>
</li>
<li>
<p><strong>嵌套感：</strong> 架构图中常见的“VPC包含子网，子网包含服务器”，在 G6 中对应的概念叫 <strong>Combo</strong>。</p>
</li>
</ul>
<hr>
<h3 id="2-关键配置详解基于-g6-v5"><a href="#2-关键配置详解基于-g6-v5">2. 关键配置详解（基于 G6 v5）</a></h3>
<h4 id="a-布局dagre-是唯一的真神"><a href="#a-布局dagre-是唯一的真神">A. 布局：Dagre 是唯一的真神</a></h4>
<p>不要试图自己去算坐标，一定要用 Dagre。它是专门用于有向图（Directed Graph）分层布局的算法。</p>
<p>JavaScript</p>
<pre><code class="hljs language-ruby">const layout = {
  <span class="hljs-symbol">type:</span> <span class="hljs-string">'dagre'</span>,
  <span class="hljs-symbol">rankdir:</span> <span class="hljs-string">'TB'</span>, <span class="hljs-regexp">//</span> 布局方向：<span class="hljs-string">'TB'</span> (<span class="hljs-title class_">Top</span>-<span class="hljs-title class_">Bottom</span>) 自上而下，或 <span class="hljs-string">'LR'</span> (<span class="hljs-title class_">Left</span>-<span class="hljs-title class_">Right</span>) 从左到右
  <span class="hljs-symbol">align:</span> <span class="hljs-string">'UL'</span>,   <span class="hljs-regexp">//</span> 对齐方式：<span class="hljs-string">'UL'</span> (<span class="hljs-title class_">Upper</span> <span class="hljs-title class_">Left</span>)

<h4 id="b-节点使用矩形与锚点"><a href="#b-节点使用矩形与锚点">B. 节点：使用矩形与锚点</a></h4>
<p>架构图的节点通常是矩形（Rect），并且连线最好连接在节点的“上下左右”四个中点上，而不是圆心，这样更整洁。</p>
<ul>
<li>
<p><strong>Shape:</strong> <code>rect</code></p>
</li>
<li>
<p><strong>Ports (连接桩):</strong> 必须指定连接点，否则线会穿过矩形中心。</p>
</li>
</ul>
<h4 id="c-边正交折线"><a href="#c-边正交折线">C. 边：正交折线</a></h4>
<p>流程图的线必须拐直角。</p>
<ul>
<li>
<p><strong>Type:</strong> <code>polyline</code> (折线)</p>
</li>
<li>
<p><strong>Router:</strong> <code>orth</code> (正交路由)，让线自动寻找直角路径避开节点。</p>
</li>
</ul>

<h4 id="2-解决连线乱穿问题"><a href="#2-解决连线乱穿问题">2. 解决“连线乱穿”问题</a></h4>
<p>Dagre 布局有时会导致连线穿过节点。</p>
<ul>
<li>
<p><strong>方案：</strong> 增加 <code>layout</code> 中的 <code>ranksep</code>（层间距）。</p>
</li>
<li>
<p><strong>高级方案：</strong> 如果 G6 的内置路由（orth）效果不好，可以尝试 G6 的插件或自定义 Edge，强制连线先从 Right 端口出来，再寻找路径。</p>
</li>
</ul>
<h4 id="3-复杂的节点内容html-node"><a href="#3-复杂的节点内容html-node">3. 复杂的节点内容（HTML Node）</a></h4>
<p>如果你的架构图节点需要显示：IP 地址、CPU 占用率进度条、报警图标。普通的 <code>rect</code> 形状很难画。</p>
<ul>
<li>
<p>使用 React/HTML 节点：</p>
<p>在 v5 中，你可以定义一个 React 组件作为节点。</p>
<p>JavaScript</p>
<pre><code class="hljs language-javascript"><span class="hljs-keyword">import</span> { <span class="hljs-title class_">ReactNode</span> } <span class="hljs-keyword">from</span> <span class="hljs-string">'@antv/g6-extension-react'</span>;

</li>

<h3 id="5-总结与建议"><a href="#5-总结与建议">5. 总结与建议</a></h3>

<li>

</li>
<li>

</li>
</ul>
