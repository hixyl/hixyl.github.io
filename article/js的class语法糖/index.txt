1a:T1f0f,<p>JavaScript 的 <code>class</code> 是一个非常精妙的“谎言”。</p>
<p>表面上，它让 JavaScript 看起来像 Java 或 C++ 这种基于类（Class-based）的语言，但其内核依然是那个自由、甚至有些“原始”的<strong>原型继承（Prototypical Inheritance）</strong>。</p>
<p>要彻底理解它，我们需要撕开这层糖衣，看看底层的内存结构和执行逻辑。</p>
<hr>
<h2 id="1-语法糖背后的真相"><a href="#1-语法糖背后的真相">1. 语法糖背后的真相</a></h2>
<p>在 2015 年（ES6）之前，我们通过构造函数来模拟类。<code>class</code> 语法的出现，本质上是将繁琐的原型链操作封装成了更符合人类直觉的关键词。</p>
<h3 id="核心对比"><a href="#核心对比">核心对比</a></h3>
<p>当你写下这段代码时：</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">class</span> <span class="hljs-title class_">Human</span> {
  <span class="hljs-title function_">constructor</span>(<span class="hljs-params">name</span>) {
    <span class="hljs-variable language_">this</span>.<span class="hljs-property">name</span> = name;
  }
  <span class="hljs-title function_">sayHello</span>(<span class="hljs-params"></span>) {
    <span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">`Hi, I'm <span class="hljs-subst">${name}</span>`</span>);
  }
}
</code></pre>
<p>它的底层逻辑等同于：</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">function</span> <span class="hljs-title function_">Human</span>(<span class="hljs-params">name</span>) {
  <span class="hljs-variable language_">this</span>.<span class="hljs-property">name</span> = name;
}
<span class="hljs-title class_">Human</span>.<span class="hljs-property"><span class="hljs-keyword">prototype</span></span>.<span class="hljs-property">sayHello</span> = <span class="hljs-keyword">function</span>(<span class="hljs-params"></span>) {
  <span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">`Hi, I'm <span class="hljs-subst">${<span class="hljs-variable language_">this</span>.name}</span>`</span>);
};
</code></pre>
<p><strong>关键差异点：</strong></p>
<ul>
<li>
<p><strong>强制性：</strong> <code>class</code> 必须配合 <code>new</code> 使用。如果你直接调用 <code>Human()</code>，JavaScript 会报错；而传统的构造函数则会把 <code>this</code> 绑定到全局对象（非严格模式下），这往往是 Bug 的温床。</p>
</li>
<li>
<p><strong>不可枚举性：</strong> <code>class</code> 中定义的方法默认是不可枚举的（non-enumerable），这符合“方法不应出现在 <code>for...in</code> 循环中”的工程直觉。</p>
</li>
</ul>
<hr>
<h2 id="2-深度剖析new-到底做了什么"><a href="#2-深度剖析new-到底做了什么">2. 深度剖析：<code>new</code> 到底做了什么？</a></h2>
<p>理解 <code>class</code> 的第一步是理解 <code>new</code> 操作符。当我们执行 <code>const me = new Human('Alex')</code> 时，底层发生了四件事：</p>
<ol>
<li>
<p><strong>创建一个空对象</strong>：<code>{}</code>。</p>
</li>
<li>
<p><strong>原型关联</strong>：将这个空对象的 <code>[[Prototype]]</code>（即隐藏属性 <code>__proto__</code>）指向 <code>Human.prototype</code>。</p>
</li>
<li>
<p><strong>绑定 <code>this</code> 并执行</strong>：运行 <code>constructor</code> 里的代码，此时 <code>this</code> 就指向这个新对象。</p>
</li>
<li>
<p><strong>自动返回</strong>：如果构造函数没有返回其他对象，则默认返回这个新创建的对象。</p>
</li>
</ol>
<hr>
<h2 id="3-继承的本质原型链的链接"><a href="#3-继承的本质原型链的链接">3. 继承的本质：原型链的链接</a></h2>
<p>在 JavaScript 中，继承不是“拷贝”，而是“委托”。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">class</span> <span class="hljs-title class_">Developer</span> <span class="hljs-keyword">extends</span> <span class="hljs-title class_ inherited__">Human</span> {
  <span class="hljs-title function_">constructor</span>(<span class="hljs-params">name, lang</span>) {
    <span class="hljs-variable language_">super</span>(name); <span class="hljs-comment">// 必须先调用 super</span>
    <span class="hljs-variable language_">this</span>.<span class="hljs-property">lang</span> = lang;
  }
}
</code></pre>
<h3 id="super-的魔法"><a href="#super-的魔法"><code>super</code> 的魔法</a></h3>
<p>在 <code>extends</code> 发生时，存在两条原型链的链接：</p>
<ol>
<li>
<p><strong>实例原型链</strong>：<code>Developer.prototype.__proto__ === Human.prototype</code>。这保证了 <code>Developer</code> 的实例可以找到 <code>Human</code> 的方法。</p>
</li>
<li>
<p><strong>构造函数原型链</strong>：<code>Developer.__proto__ === Human</code>。这保证了 <code>Developer</code> 可以继承 <code>Human</code> 的<strong>静态方法</strong>。</p>
</li>
</ol>
<p><strong>为什么必须先写 <code>super()</code>？</strong></p>
<p>在基于类的语言中，子类实例是先创建的。但在 JS 的 <code>class</code> 继承中，子类的 <code>this</code> 是由父类构造函数生成的。不调用 <code>super()</code>，子类就没有 <code>this</code>。</p>
<hr>
<h2 id="4-现代特性私有属性与内存"><a href="#4-现代特性私有属性与内存">4. 现代特性：私有属性与内存</a></h2>
<p>JavaScript 最近引入了 <code>#</code> 开头的私有属性（如 <code>#secret</code>）。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">class</span> <span class="hljs-title class_">BankAccount</span> {
  #balance = <span class="hljs-number">0</span>; <span class="hljs-comment">// 私有属性</span>
  
  <span class="hljs-title function_">deposit</span>(<span class="hljs-params">amount</span>) {
    <span class="hljs-variable language_">this</span>.#balance += amount;
  }
}
</code></pre>
<p><strong>底层原理：</strong></p>
<p>不同于 TypeScript 只是在编译时做检查，JS 原生的 <code>#</code> 属性是在 <strong>V8 引擎层面</strong>实现的。这些属性不在普通的对象键值对中存储，而是存在一个外部的哈希表（WeakMap 结构的变体）中，以对象实例为键。这意味着你无法通过 <code>account['#balance']</code> 这种黑产手段访问它，实现了真正的封装。</p>
<hr>
<h2 id="5-静态成员与块级作用域"><a href="#5-静态成员与块级作用域">5. 静态成员与块级作用域</a></h2>
<ul>
<li>
<p><strong><code>static</code> 关键字</strong>：直接定义在构造函数本身上，而不是 <code>prototype</code> 上。它们属于“类”这个对象，而不属于“实例”。</p>
</li>
<li>
<p><strong>静态初始化块</strong>：这是较新的特性，允许你在类定义时执行复杂的初始化逻辑，且能访问私有属性。</p>
</li>
</ul>
<hr>
<h2 id="总结如何构建直觉"><a href="#总结如何构建直觉">总结：如何构建直觉？</a></h2>
<p>你可以把 JS 的 <code>class</code> 想象成一个<strong>蓝图工厂</strong>：</p>
<ul>
<li>
<p><code>constructor</code> 是<strong>初始化车间</strong>，负责给每个产品打上唯一的标签（属性）。</p>
</li>
<li>
<p><code>prototype</code> 是<strong>共享工具库</strong>，所有产品共享一套说明书和工具（方法），不占用额外内存。</p>
</li>
<li>
<p><code>__proto__</code> 是<strong>寻路指针</strong>，当产品发现自己不会某个技能时，通过指针去工具库里找。</p>
</li>
</ul>
