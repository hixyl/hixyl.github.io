<h2 id="1-创建swap分区"><a href="#1-创建swap分区">1. 创建swap分区</a></h2>
<p>Linux VPS主机设置swap空间的基本命令如下：</p>
<pre><code class="hljs language-bash"><span class="hljs-comment">#查看swap分区的大小</span>
free -h 
<span class="hljs-comment">#删除所有SWAP分区</span>
swapoff -a
<span class="hljs-comment">#创建swap分区的文件，其中bs是每块的大小，count是块的数量，bs*count，就是swap文件的大小了，这里就是1M*2048=2G </span>
<span class="hljs-built_in">dd</span> <span class="hljs-keyword">if</span>=/dev/zero of=/root/swapfile bs=1M count=2048
<span class="hljs-comment">#格式化交换分区文件： </span>
mkswap /root/swapfile
<span class="hljs-comment">#启用swap分区文件： </span>
swapon /root/swapfile
<span class="hljs-comment">#添加开机启动，修改 /etc/fstab 这个文件，添加或者修改这一行：</span>
/root/swapfile swap swap defaults 0 0 <span class="hljs-comment">#适用于redhat或centos系统</span>
/root/swapfile none swap sw  0 0  <span class="hljs-comment">#适用于Ubuntu系统</span>
<span class="hljs-comment">#或者直接执行以下命令写入/etc/fstab</span>
<span class="hljs-built_in">echo</span> <span class="hljs-string">"/root/swapfile swap swap defaults 0 0"</span> >>/etc/fstab <span class="hljs-comment">#适用于redhat或ce</span>
</code></pre>

<h2 id="2-linux真实使用内存查看"><a href="#2-linux真实使用内存查看">2. Linux真实使用内存查看</a></h2>
<p>Linux的内存管理机制与Windows有所不同，linux系统中，是进程优先使用内存，而不是磁盘，这样会加快读取速度。当有新的进程启动时，再从内存中开辟出一定空间，为新的进程所用。即使进程退出，也不会立刻从内存中清理掉，这样可以加快下次启动进程的速度，所以内存一直是使用率很高的。那么如何查看Linux真实使用内存？ 执行命令：<code>free -h</code> 可以查看Linux VPS主机的内存使用情况：</p>

<span class="hljs-attribute">Mem</span>:           <span class="hljs-number">2.0</span>G        <span class="hljs-number">1.3</span>G        <span class="hljs-number">382</span>M        <span class="hljs-number">106</span>M        <span class="hljs-number">280</span>M        <span class="hljs-number">416</span>M
