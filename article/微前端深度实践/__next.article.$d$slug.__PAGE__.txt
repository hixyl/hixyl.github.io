4:T45f4,<p>在微前端架构从“跑通”走向“生产级”的过程中，单纯的嵌入只是第一步。要构建一个健壮的系统，我们必须解决<strong>通信闭环、状态保持、路由接管、资源路径</strong>这四个核心问题。</p>
<p>以下是针对 React 场景的深度实践技巧和完整代码配置。</p>
<hr>
<h3 id="一-全局数据通信data-communication"><a href="#一-全局数据通信data-communication">一、 全局数据通信（Data Communication）</a></h3>
<p>这是最核心的功能。Micro-app 提供了一套基于发布订阅模式的数据通道。</p>
<h4 id="1-核心逻辑"><a href="#1-核心逻辑">1. 核心逻辑</a></h4>
<ul>
<li>
<p><strong>下发（Host -> Child）：</strong> 主应用通过 <code>setData</code> 广播数据，子应用监听数据变化。</p>
</li>
<li>
<p><strong>反向（Child -> Host）：</strong> 子应用通过 <code>dispatch</code> 发送数据，主应用监听。</p>
</li>
<li>
<p><strong>注意点：</strong> 必须在组件卸载时解绑监听器，否则会导致内存泄漏。</p>
</li>
</ul>

<h3 id="二-路由接管与-baseroute"><a href="#二-路由接管与-baseroute">二、 路由接管与 BaseRoute</a></h3>
<p>如果子应用本身有多页路由（如使用了 <code>react-router-dom</code>），你需要告诉子应用：“你的根路径不是 <code>/</code>，而是主应用分配给你的 <code>/child-one</code>”。</p>
<h4 id="1-主应用配置"><a href="#1-主应用配置">1. 主应用配置</a></h4>
<p>假设我们要把子应用挂载在 <code>http://localhost:3000/child-one/*</code> 下。</p>
<p>TypeScript</p>
<pre><code class="hljs language-ts"><span class="hljs-comment">// main-app</span>
&#x3C;micro-app 
  name=<span class="hljs-string">"child-app"</span> 
  url=<span class="hljs-string">"http://localhost:4000/"</span> 
  baseroute=<span class="hljs-string">"/child-one"</span> <span class="hljs-comment">// 告诉子应用，你的地盘从这里开始</span>
>&#x3C;/micro-app>
</code></pre>
<h4 id="2-子应用配置"><a href="#2-子应用配置">2. 子应用配置</a></h4>
<p>子应用的路由必须识别这个 <code>basename</code>。</p>
<p>TypeScript</p>
<pre><code class="hljs language-ts"><span class="hljs-comment">// child-app/src/main.tsx</span>
<span class="hljs-keyword">import</span> { <span class="hljs-title class_">BrowserRouter</span> } <span class="hljs-keyword">from</span> <span class="hljs-string">'react-router-dom'</span>;

<span class="hljs-comment">// 动态获取：如果是独立运行，base就是 '/'；如果是微前端，就是 '/child-one'</span>
<span class="hljs-keyword">const</span> <span class="hljs-variable constant_">BASE_ROUTE</span> = <span class="hljs-variable language_">window</span>.<span class="hljs-property">__MICRO_APP_BASE_ROUTE__</span> || <span class="hljs-string">'/'</span>;

<span class="hljs-title class_">ReactDOM</span>.<span class="hljs-title function_">createRoot</span>(<span class="hljs-variable language_">document</span>.<span class="hljs-title function_">getElementById</span>(<span class="hljs-string">'root'</span>)!).<span class="hljs-title function_">render</span>(

<h3 id="三-性能优化keep-alive-保活"><a href="#三-性能优化keep-alive-保活">三、 性能优化：Keep-alive (保活)</a></h3>
<p>默认情况下，当你从子应用 Tab A 切到 Tab B，子应用会被销毁。再次切回来时需要重新加载资源、重新渲染，体验很差。</p>
<p><code>keep-alive</code> 属性可以让子应用在切走时只是“隐藏”而不是“销毁”。</p>
<p><strong>用法非常简单：</strong></p>
<p>TypeScript</p>
<pre><code class="hljs language-ts">&#x3C;micro-app 
  name=<span class="hljs-string">"child-app"</span> 
  url=<span class="hljs-string">"..."</span> 
  keep-alive <span class="hljs-comment">// 开启保活</span>
>&#x3C;/micro-app>
</code></pre>
<p>深度解读：</p>
<p>开启后，子应用的 componentWillUnmount (或者 useEffect 的 cleanup) 不会执行。</p>
<p>如果你需要感知“我被切走了”和“我又回来了”，需要使用 micro-app 提供的特定生命周期：</p>
<p>TypeScript</p>
<pre><code class="hljs language-ts"><span class="hljs-comment">// 子应用内部</span>
<span class="hljs-title function_">useEffect</span>(<span class="hljs-function">() =></span> {
  <span class="hljs-keyword">const</span> <span class="hljs-title function_">onMount</span> = (<span class="hljs-params"></span>) => <span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">'我被唤醒了 (created or activated)'</span>);
  <span class="hljs-keyword">const</span> <span class="hljs-title function_">onUnmount</span> = (<span class="hljs-params"></span>) => <span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">'我被隐藏了 (unmounted or deactivated)'</span>);

  <span class="hljs-variable language_">window</span>.<span class="hljs-title function_">addEventListener</span>(<span class="hljs-string">'appstate-change'</span>, <span class="hljs-function">(<span class="hljs-params"><span class="hljs-attr">e</span>: <span class="hljs-built_in">any</span></span>) =></span> {
    <span class="hljs-keyword">if</span> (e.<span class="hljs-property">detail</span>.<span class="hljs-property">appState</span> === <span class="hljs-string">'active'</span>) <span class="hljs-title function_">onMount</span>();
    <span class="hljs-keyword">if</span> (e.<span class="hljs-property">detail</span>.<span class="hljs-property">appState</span> === <span class="hljs-string">'deactive'</span>) <span class="hljs-title function_">onUnmount</span>();
  });
}, []);
</code></pre>

<h3 id="四-静态资源路径补全-public-path"><a href="#四-静态资源路径补全-public-path">四、 静态资源路径补全 (Public Path)</a></h3>
<p>这是新手最容易遇到的坑：<strong>子应用的图片/字体在独立运行时正常，嵌入主应用后 404。</strong></p>
<p><strong>原因：</strong> 浏览器请求相对路径资源（如 <code>/logo.png</code>）时，会基于地址栏的域名（主应用 localhost:3000）去请求，但资源实际在子应用服务器（localhost:4000）上。</p>
<p><strong>解法：</strong> 在子应用入口文件的最顶部（第一行）注入 public path。</p>
<p>TypeScript</p>
<pre><code class="hljs language-ts"><span class="hljs-comment">// child-app/src/main.ts (或 index.ts)</span>
<span class="hljs-comment">// 必须放在 import App 之前</span>
<span class="hljs-keyword">if</span> (<span class="hljs-variable language_">window</span>.<span class="hljs-property">__MICRO_APP_ENVIRONMENT__</span>) {
  <span class="hljs-comment">// eslint-disable-next-line @typescript-eslint/no-explicit-any</span>
  __webpack_public_path__ = <span class="hljs-variable language_">window</span>.<span class="hljs-property">__MICRO_APP_PUBLIC_PATH__</span>;
}

<span class="hljs-comment">// Vite 项目的特殊处理：</span>
<span class="hljs-comment">// Vite 在开发模式下处理 public path 比较特殊，通常建议写死图片绝对路径</span>
<span class="hljs-comment">// 或者在 vite.config.ts 中配置 base</span>
</code></pre>
<p>针对 Vite 的实用技巧：</p>
<p>在 vite.config.ts 中写一个简单的插件，自动转换 HTML 中的资源链接（如果 micro-app 的 iframe 模式处理不完美时）：</p>
<p>TypeScript</p>
<pre><code class="hljs language-ts"><span class="hljs-comment">// child-app/vite.config.ts</span>
<span class="hljs-keyword">export</span> <span class="hljs-keyword">default</span> <span class="hljs-title function_">defineConfig</span>({
  <span class="hljs-attr">base</span>: process.<span class="hljs-property">env</span>.<span class="hljs-property">NODE_ENV</span> === <span class="hljs-string">'production'</span> ? <span class="hljs-string">'http://your-cdn.com/child-app/'</span> : <span class="hljs-string">'/'</span>,
  <span class="hljs-comment">// ...其他配置</span>
})
</code></pre>

<h3 id="五-预加载-prefetch"><a href="#五-预加载-prefetch">五、 预加载 (PreFetch)</a></h3>
<p>为了极致体验，你可以在用户登录主应用时，就利用浏览器的空闲时间悄悄加载子应用的资源。</p>
<p>在主应用的入口文件 <code>main.tsx</code> 中配置：</p>
<p>TypeScript</p>
<pre><code class="hljs language-ts"><span class="hljs-comment">// main-app/src/main.tsx</span>
<span class="hljs-keyword">import</span> microApp <span class="hljs-keyword">from</span> <span class="hljs-string">'@micro-zoe/micro-app'</span>;

microApp.<span class="hljs-title function_">start</span>({
  <span class="hljs-attr">preFetchApps</span>: [
    { <span class="hljs-attr">name</span>: <span class="hljs-string">'child-app'</span>, <span class="hljs-attr">url</span>: <span class="hljs-string">'http://localhost:4000/'</span> }
  ]
});
</code></pre>
<h3 id="总结清单"><a href="#总结清单">总结清单</a></h3>
<ol>
<li>
<p><strong>通信</strong>：优先使用 <code>setData</code> 和 <code>addDataListener</code>，记得解绑。</p>
</li>
<li>
<p><strong>路由</strong>：子应用一定要配合设置 <code>basename</code>，值为 <code>window.__MICRO_APP_BASE_ROUTE__</code>。</p>
</li>
<li>
<p><strong>体验</strong>：对于复杂的子应用，务必开启 <code>keep-alive</code>。</p>
</li>
<li>
<p><strong>样式</strong>：推荐使用 <code>iframe</code> 属性开启强隔离（sandbox），这能省去90%的 CSS 冲突烦恼。</p>
</li>
<li>
<p><strong>类型</strong>：不要在 TS 类型上纠结太久，善用 <code>global.d.ts</code> 或 <code>window</code> 扩展来解决报错。</p>
</li>
