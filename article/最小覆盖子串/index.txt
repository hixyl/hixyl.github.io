1c:T351d,<h3 id="算法题目最小覆盖子串"><a href="#算法题目最小覆盖子串"><strong>算法题目：最小覆盖子串</strong></a></h3>
<p>给你一个字符串 <code>s</code> 和一个字符串 <code>t</code> 。请你找出在 <code>s</code> 中涵盖 <code>t</code> 所有字符的最小子串。如果 <code>s</code> 中不存在涵盖 <code>t</code> 所有字符的子串，则返回空字符串 <code>""</code> 。</p>
<p><strong>注意：</strong></p>
<ul>
<li>
<p>对于 <code>t</code> 中重复的字符，我们寻找的子字符串中该字符的数量必须不少于 <code>t</code> 中该字符的数量。</p>
</li>
<li>
<p>如果 <code>s</code> 中存在这样的子串，我们保证它是唯一的答案。</p>
</li>
</ul>
<hr>
<h4 id="示例-1"><a href="#示例-1"><strong>示例 1：</strong></a></h4>
<ul>
<li>
<p><strong>输入：</strong> <code>s = "ADOBECODEBANC"</code>, <code>t = "ABC"</code></p>
</li>
<li>
<p><strong>输出：</strong> <code>"BANC"</code></p>
</li>
<li>
<p><strong>解释：</strong> 最小覆盖子串 <code>"BANC"</code> 包含来自字符串 <code>t</code> 的 'A'、'B' 和 'C'。</p>
</li>
</ul>
<hr>
<h4 id="示例-2"><a href="#示例-2"><strong>示例 2：</strong></a></h4>
<ul>
<li>
<p><strong>输入：</strong> <code>s = "a"</code>, <code>t = "a"</code></p>
</li>
<li>
<p><strong>输出：</strong> <code>"a"</code></p>
</li>
<li>
<p><strong>解释：</strong> 整个字符串 <code>s</code> 是最小覆盖子串。</p>
</li>
</ul>
<hr>
<h4 id="示例-3"><a href="#示例-3"><strong>示例 3：</strong></a></h4>
<ul>
<li>
<p><strong>输入：</strong> <code>s = "a"</code>, <code>t = "aa"</code></p>
</li>
<li>
<p><strong>输出：</strong> <code>""</code></p>
</li>
<li>
<p><strong>解释：</strong> <code>t</code> 中两个字符 'a' 均应包含在 <code>s</code> 的子串中，因此没有符合条件的子字符串，返回空字符串。</p>
</li>
</ul>
<hr>
<h4 id="提示"><a href="#提示"><strong>提示：</strong></a></h4>
<ul>
<li>
<p><code>m == s.length</code></p>
</li>
<li>
<p><code>n == t.length</code></p>
</li>
<li>
<p>1≤m,n≤105</p>
</li>
<li>
<p><code>s</code> 和 <code>t</code> 由英文字母组成</p>
</li>
</ul>
<hr>
<p>这道“最小覆盖子串”问题是算法中一个非常经典的题目，它考察的是对字符串处理和窗口思想的理解。直接使用暴力法（即检查<code>s</code>的每一个子串）效率会非常低，无法通过测试。解决这个问题的最优方法是<strong>滑动窗口（Sliding Window）</strong>。</p>
<h3 id="解题思路滑动窗口"><a href="#解题思路滑动窗口">解题思路：滑动窗口</a></h3>
<p>滑动窗口可以理解为一个在字符串<code>s</code>上移动的“尺子”，这个“尺子”的左右两端就是我们所说的“窗口”。我们的目标是调整这个“尺子”的左右端点，使其框住的范围（即子串）恰好包含<code>t</code>中的所有字符，并找到这个“尺子”的最小长度。</p>
<p>具体步骤如下：</p>
<ol>
<li>
<p><strong>初始化需求列表</strong>：首先，我们需要知道我们需要哪些字符以及各需要多少个。我们可以用一个哈希表（在 JavaScript 中就是普通对象 <code>Object</code>）来记录字符串 <code>t</code> 中每个字符出现的次数。我们称之为 <code>needs</code>。</p>
</li>
<li>
<p><strong>定义窗口和指针</strong>：</p>
<ul>
<li>
<p>我们用 <code>left</code> 和 <code>right</code> 两个指针来表示窗口的左右边界，初始时都指向字符串 <code>s</code> 的开头（索引 <code>0</code>）。</p>
</li>
<li>
<p>我们还需要另一个哈希表 <code>window</code> 来记录当前窗口内包含了 <code>needs</code> 中字符的个数。</p>
</li>
<li>
<p>一个计数器 <code>validCount</code> 用来记录 <code>window</code> 中有多少个字符已经满足了 <code>needs</code> 的要求（即数量上达到了要求）。例如，如果 <code>t</code> 是 "AAB"，那么当窗口中 'A' 的数量达到 2 时，'A' 这个字符就算满足要求了，<code>validCount</code> 可以加 1。</p>
</li>
</ul>
</li>
<li>
<p><strong>移动窗口</strong>：</p>
<ul>
<li>
<p><strong>扩大窗口</strong>：我们不断地向右移动 <code>right</code> 指针，将新的字符加入窗口。</p>
<ul>
<li>
<p>每加入一个字符 <code>c</code>，就更新 <code>window</code> 中 <code>c</code> 的计数。</p>
</li>
<li>
<p>如果 <code>c</code> 是我们需要的字符（即 <code>c</code> 在 <code>needs</code> 中），并且它在 <code>window</code> 中的数量正好等于在 <code>needs</code> 中的数量，说明这个字符的要求已经满足，我们将 <code>validCount</code> 加 1。</p>
</li>
</ul>
</li>
<li>
<p><strong>缩小窗口</strong>：当 <code>validCount</code> 的值等于 <code>needs</code> 中不同字符的总数时（例如，<code>t</code>="AABC"，<code>needs</code>里有A,B,C三种字符，那么就是3），说明当前窗口已经是一个<strong>可行解</strong>（即它已经覆盖了 <code>t</code>）。</p>
<ul>
<li>
<p>此时，我们记录下这个窗口的长度，并和已知的最小长度比较，如果更小就更新最小长度和起始位置。</p>
</li>
<li>
<p>然后，我们尝试从左边缩小窗口，即向右移动 <code>left</code> 指针。</p>
</li>
<li>
<p>移出一个字符 <code>d</code>，并更新 <code>window</code> 中 <code>d</code> 的计数。</p>
</li>
<li>
<p>如果 <code>d</code> 是我们需要的字符，并且在移出之前它在 <code>window</code> 中的数量正好等于在 <code>needs</code> 中的数量，那么移出后，这个字符的要求就不再满足了，所以 <code>validCount</code> 要减 1。</p>
</li>
<li>
<p>我们持续缩小窗口，直到 <code>validCount</code> 不再满足条件，然后回到扩大窗口的步骤。</p>
</li>
</ul>
</li>
</ul>
</li>
<li>
<p><strong>循环与结束</strong>：</p>
<ul>
<li>
<p>我们重复第 3 步，直到 <code>right</code> 指针到达字符串 <code>s</code> 的末尾。</p>
</li>
<li>
<p>循环结束后，如果最小长度仍然是初始的极大值，说明没有找到任何符合条件的子串，返回 <code>""</code>。否则，根据记录的最小长度和起始位置，截取并返回结果。</p>
</li>
</ul>
</li>
</ol>
<hr>

        }

        <span class="hljs-comment">// 当窗口中所有需要的字符数量都满足了 t 的要求时</span>
        <span class="hljs-comment">// validCount === needsSize 表示找到了一个覆盖子串</span>
        <span class="hljs-keyword">while</span> (validCount === needsSize) {
            <span class="hljs-comment">// 当前窗口的长度</span>
            <span class="hljs-keyword">const</span> currentLen = right - left;
            <span class="hljs-comment">// 如果当前窗口长度小于已记录的最小长度，则更新</span>
            <span class="hljs-keyword">if</span> (currentLen &#x3C; minLen) {
                minLen = currentLen;
                start = left; <span class="hljs-comment">// 记录这个最小子串的起始位置</span>
            }

            <span class="hljs-comment">// d 是即将移出窗口的字符</span>
            <span class="hljs-keyword">const</span> d = s[left];
            <span class="hljs-comment">// 左移窗口</span>
            left++;

            <span class="hljs-comment">// 如果移出的字符 d 是 t 中需要的字符</span>
            <span class="hljs-keyword">if</span> (needs[d]) {
                <span class="hljs-comment">// 如果 d 在窗口中的数量原本是正好满足需求的</span>
                <span class="hljs-keyword">if</span> (<span class="hljs-variable language_">window</span>[d] === needs[d]) {
                    <span class="hljs-comment">// 那么移出后就不再满足了，validCount -1</span>
                    validCount--;
                }
                <span class="hljs-comment">// 更新 window 计数</span>
                <span class="hljs-variable language_">window</span>[d]--;
            }
        }
    }

    <span class="hljs-comment">// 如果 minLen 还是初始值，说明没有找到符合条件的子串</span>
    <span class="hljs-comment">// 否则，根据记录的 start 和 minLen 截取子串并返回</span>
    <span class="hljs-keyword">return</span> minLen === <span class="hljs-title class_">Infinity</span> ? <span class="hljs-string">""</span> : s.<span class="hljs-title function_">substring</span>(start, start + minLen);
};
</code></pre>
<h3 id="复杂度分析"><a href="#复杂度分析">复杂度分析</a></h3>
<ul>
<li>
<p><strong>时间复杂度</strong>: O(m+n)，其中 <code>m</code> 是字符串 <code>s</code> 的长度，<code>n</code> 是字符串 <code>t</code> 的长度。</p>
<ul>
<li>
<p>我们首先遍历 <code>t</code> 一次来构建 <code>needs</code> 哈希表，时间为 O(n)。</p>
</li>
<li>
<p>然后 <code>left</code> 和 <code>right</code> 指针都各自最多遍历 <code>s</code> 一次。虽然有嵌套的 <code>while</code> 循环，但 <code>left</code> 指针的移动次数不会超过 <code>right</code>，总体来看每个元素最多被访问两次（一次被 <code>right</code> 指针访问，一次被 <code>left</code> 指针访问）。所以滑动窗口部分的时间复杂度是 O(m)。</p>
</li>
<li>
<p>总时间复杂度为 O(m+n)。</p>
</li>
</ul>
</li>
<li>
<p><strong>空间复杂度</strong>: O(k)，其中 <code>k</code> 是字符串 <code>t</code> 中不同字符的数量（或者说是字符集的大小）。</p>
<ul>
<li>我们需要 <code>needs</code> 和 <code>window</code> 两个哈希表来存储字符计数，它们的大小最多为 <code>k</code>。</li>
</ul>
</li>
