18:T1ece,<p>在现代前端工程化项目中，Web Worker 不再是那种“需要手写原生路径”的边缘技术，而是作为解决 JavaScript 单线程性能瓶颈的核心工具。</p>
<p>要将其优雅地集成到项目中，主要涉及<strong>工程化接入</strong>、<strong>通信架构设计</strong>以及<strong>适用场景识别</strong>三个维度。</p>
<hr>
<h2 id="1-工程化接入打包与加载"><a href="#1-工程化接入打包与加载">1. 工程化接入：打包与加载</a></h2>
<p>在 Vite 或 Webpack 5 之前，处理 Worker 极其麻烦（需要专门的 loader）。现在，主流工具链已经内置了对 Worker 的支持。</p>
<h3 id="vite-中的接入"><a href="#vite-中的接入">Vite 中的接入</a></h3>
<p>Vite 使用 <code>?worker</code> 后缀或 <code>new URL()</code> 语法：</p>
<p>TypeScript</p>
<pre><code class="hljs language-ts"><span class="hljs-comment">// 1. 声明式引入</span>
<span class="hljs-keyword">import</span> <span class="hljs-title class_">MyWorker</span> <span class="hljs-keyword">from</span> <span class="hljs-string">'./worker.ts?worker'</span>;
<span class="hljs-keyword">const</span> worker = <span class="hljs-keyword">new</span> <span class="hljs-title class_">MyWorker</span>();

<span class="hljs-comment">// 2. 标准 ESM 引入（推荐，兼容性更好）</span>
<span class="hljs-keyword">const</span> worker = <span class="hljs-keyword">new</span> <span class="hljs-title class_">Worker</span>(
  <span class="hljs-keyword">new</span> <span class="hljs-title function_">URL</span>(<span class="hljs-string">'./worker.ts'</span>, <span class="hljs-keyword">import</span>.<span class="hljs-property">meta</span>.<span class="hljs-property">url</span>),
  { <span class="hljs-attr">type</span>: <span class="hljs-string">'module'</span> }
);
</code></pre>
<h3 id="webpack-5-中的接入"><a href="#webpack-5-中的接入">Webpack 5 中的接入</a></h3>
<p>Webpack 5 能够自动识别 <code>new Worker</code> 语法并生成独立的 chunk：</p>
<p>TypeScript</p>
<pre><code class="hljs language-arduino"><span class="hljs-type">const</span> worker = <span class="hljs-keyword">new</span> <span class="hljs-built_in">Worker</span>(<span class="hljs-keyword">new</span> <span class="hljs-built_in">URL</span>(<span class="hljs-string">'./worker.js'</span>, <span class="hljs-keyword">import</span>.meta.url));
</code></pre>
<hr>
<h2 id="2-通信架构设计从-postmessage-到-rpc"><a href="#2-通信架构设计从-postmessage-到-rpc">2. 通信架构设计：从 postMessage 到 RPC</a></h2>
<p>原生 <code>postMessage</code> 的问题在于它是基于<strong>监听/响应</strong>模式的，代码逻辑很容易变得支离破碎（像早期的 Ajax）。在工程化项目中，推荐使用 <strong>RPC（远程过程调用）</strong> 模式。</p>
<h3 id="推荐工具comlink"><a href="#推荐工具comlink">推荐工具：Comlink</a></h3>
<p>Google 团队开发的 <a href="https://github.com/GoogleChromeLabs/comlink">Comlink</a> 是目前的工业标准。它将复杂的 <code>postMessage</code> 封装成了类似异步函数调用的形式。</p>
<p><strong>Worker 侧 (<code>worker.ts</code>):</strong></p>
<p>TypeScript</p>
<pre><code class="hljs language-ts"><span class="hljs-keyword">import</span> * <span class="hljs-keyword">as</span> <span class="hljs-title class_">Comlink</span> <span class="hljs-keyword">from</span> <span class="hljs-string">'comlink'</span>;

<span class="hljs-keyword">const</span> api = {
  <span class="hljs-title function_">calculateBigData</span>(<span class="hljs-params"><span class="hljs-attr">data</span>: <span class="hljs-built_in">any</span>[]</span>) {
    <span class="hljs-comment">// 耗时计算</span>
    <span class="hljs-keyword">return</span> data.<span class="hljs-title function_">reduce</span>(<span class="hljs-function">(<span class="hljs-params">acc, val</span>) =></span> acc + val, <span class="hljs-number">0</span>);
  }
};

<span class="hljs-title class_">Comlink</span>.<span class="hljs-title function_">expose</span>(api);
</code></pre>
<p><strong>主线程侧 (<code>main.ts</code>):</strong></p>
<p>TypeScript</p>
<pre><code class="hljs language-ts"><span class="hljs-keyword">import</span> * <span class="hljs-keyword">as</span> <span class="hljs-title class_">Comlink</span> <span class="hljs-keyword">from</span> <span class="hljs-string">'comlink'</span>;

<span class="hljs-keyword">const</span> worker = <span class="hljs-keyword">new</span> <span class="hljs-title class_">Worker</span>(<span class="hljs-keyword">new</span> <span class="hljs-title function_">URL</span>(<span class="hljs-string">'./worker.ts'</span>, <span class="hljs-keyword">import</span>.<span class="hljs-property">meta</span>.<span class="hljs-property">url</span>));
<span class="hljs-keyword">const</span> api = <span class="hljs-title class_">Comlink</span>.<span class="hljs-title function_">wrap</span>(worker);

<span class="hljs-comment">// 像调用本地异步函数一样使用</span>
<span class="hljs-keyword">const</span> result = <span class="hljs-keyword">await</span> api.<span class="hljs-title function_">calculateBigData</span>([<span class="hljs-number">1</span>, <span class="hljs-number">2</span>, <span class="hljs-number">3</span>]);
</code></pre>

<h2 id="3-典型应用场景"><a href="#3-典型应用场景">3. 典型应用场景</a></h2>
<p>在工程化项目中，不应为了用而用，而应针对以下“重负载”任务进行解耦：</p>
<table>
<thead>
<tr>
<th><strong>场景</strong></th>
<th><strong>具体任务</strong></th>
<th><strong>收益</strong></th>
</tr>
</thead>
<tbody>
<tr>
<td><strong>大规模数据处理</strong></td>
<td>数万条 JSON 数据的过滤、排序、统计。</td>
<td>避免长任务（Long Task）导致页面卡顿。</td>
</tr>
<tr>
<td><strong>图像/视频处理</strong></td>
<td>Canvas 像素计算、图片压缩、滤镜生成。</td>
<td>保持动画流畅（OffscreenCanvas）。</td>
</tr>
<tr>
<td><strong>复杂算法</strong></td>
<td>加解密（RSA/AES）、Excel 导出生成、PDF 解析。</td>
<td>确保 UI 响应不受 CPU 密集任务影响。</td>
</tr>
<tr>
<td><strong>实时通信</strong></td>
<td>WebSocket 的长连接维护与心跳包解析。</td>
<td>减少主线程的消息处理压力。</td>
</tr>
</tbody>
</table>

<h2 id="4-工程化实践中的注意事项"><a href="#4-工程化实践中的注意事项">4. 工程化实践中的注意事项</a></h2>
<ul>
<li>
<p><strong>DOM 限制：</strong> Worker 线程无法直接操作 DOM，也无法使用 <code>window</code>。如果需要操作 UI，必须将数据传回主线程。</p>
</li>
<li>
<p><strong>Transferable Objects（可转移对象）：</strong> 在传输超大数据（如 <code>ArrayBuffer</code>）时，使用 <code>postMessage(data, [data])</code>。这样可以“转移”内存所有权而非复制内存，性能提升极大。</p>
</li>
<li>
<p><strong>资源释放：</strong> Worker 不是免费的，每个 Worker 都会消耗一定的系统资源。对于一次性任务，完成后务必调用 <code>worker.terminate()</code>。</p>
</li>
<li>
<p><strong>兼容性降级：</strong> 尽管目前支持率极高，但在极端环境下，可以通过判断 <code>window.Worker</code> 是否存在来决定是否降级到主线程同步执行。</p>
</li>
</ul>
<hr>
<p><strong>下一步建议：</strong></p>
<p>如果你现在的项目中有超过 100ms 的 CPU 耗时任务（可以通过 Chrome DevTools 的 Performance 面板观测到长红条），你可以尝试<strong>将最重的那部分计算函数提取到一个独立的 <code>.worker.ts</code> 文件中</strong>，并使用 Comlink 进行封装。</p>
