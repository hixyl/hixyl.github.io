4:T3f87,<p>我们来系统地梳理一下 JavaScript 中字符串（String）和数组（Array）的这些常用基础函数。</p>
<p>理解这两者差异的关键在于一个核心概念：<strong>字符串是不可变的（Immutable），而数组是可变的（Mutable）</strong>。记住这一点，很多函数的行为就变得顺理成章了。</p>
<hr>
<h3 id="1-字符串string--不可变的字符序列"><a href="#1-字符串string--不可变的字符序列">1. 字符串（String）- 不可变的字符序列</a></h3>
<p>当你对一个字符串进行“修改”操作（如切片、分割）时，你<strong>永远不会改变原始字符串</strong>。相反，你会得到一个<strong>新的</strong>字符串。</p>
<h4 id="11-长度-length"><a href="#11-长度-length">1.1 长度 (length)</a></h4>
<p>这是一个<strong>属性</strong>，不是函数，所以后面没有括号 <code>()</code> 。它返回字符串中字符的数量。</p>
<p>JavaScript</p>
<pre><code class="hljs language-ini">const <span class="hljs-attr">str</span> = <span class="hljs-string">'Hello, World!'</span><span class="hljs-comment">;</span>
console.log(str.length)<span class="hljs-comment">; // 输出: 13</span>
</code></pre>
<h4 id="12-切片-slicing"><a href="#12-切片-slicing">1.2 切片 (Slicing)</a></h4>
<p>用于从字符串中提取一部分，并返回一个<strong>新字符串</strong>。原始字符串不变。</p>
<p>有三个函数可以做这件事，但你应该<strong>优先使用 <code>slice()</code></strong>。</p>
<ul>
<li>
<p><strong><code>slice(startIndex, endIndex)</code></strong></p>
<ul>
<li>
<p>这是最常用、最推荐的方法。</p>
</li>
<li>
<p>它提取从 <code>startIndex</code> 开始，到 <code>endIndex</code> <strong>之前</strong>（不包括 <code>endIndex</code>）的部分。</p>
</li>
<li>
<p>支持<strong>负数索引</strong>，表示从字符串末尾开始计算。<code>-1</code> 指的是最后一个字符。</p>
</li>
<li>
<p>如果省略 <code>endIndex</code>，则会提取到字符串末尾。</p>
</li>
</ul>
<p>JavaScript</p>
<pre><code class="hljs language-python">const <span class="hljs-built_in">str</span> = <span class="hljs-string">'JavaScript'</span>;
console.log(<span class="hljs-built_in">str</span>.<span class="hljs-built_in">slice</span>(<span class="hljs-number">0</span>, <span class="hljs-number">4</span>));    // <span class="hljs-string">'Java'</span> (从索引<span class="hljs-number">0</span>到<span class="hljs-number">4</span>，不含<span class="hljs-number">4</span>)
console.log(<span class="hljs-built_in">str</span>.<span class="hljs-built_in">slice</span>(<span class="hljs-number">4</span>));       // <span class="hljs-string">'Script'</span> (从索引<span class="hljs-number">4</span>到末尾)
console.log(<span class="hljs-built_in">str</span>.<span class="hljs-built_in">slice</span>(-<span class="hljs-number">6</span>));      // <span class="hljs-string">'Script'</span> (从倒数第<span class="hljs-number">6</span>个字符到末尾)
console.log(<span class="hljs-built_in">str</span>.<span class="hljs-built_in">slice</span>(<span class="hljs-number">0</span>, -<span class="hljs-number">6</span>));   // <span class="hljs-string">'Java'</span> (从索引<span class="hljs-number">0</span>到倒数第<span class="hljs-number">6</span>个字符)
</code></pre>
</li>
<li>
<p><code>substring(startIndex, endIndex)</code></p>
<ul>
<li>
<p>功能与 <code>slice</code> 类似，但有两点关键不同：</p>
<ol>
<li>
<p><strong>不支持负数索引</strong>，任何负数都会被当作 <code>0</code>。</p>
</li>
<li>
<p>如果 <code>startIndex</code> 大于 <code>endIndex</code>，它会自动交换两者。</p>
</li>
</ol>
</li>
<li>
<p>由于这些行为可能导致混淆，一般不推荐使用。</p>
</li>
</ul>
</li>
<li>
<p><code>substr(startIndex, length)</code></p>
<ul>
<li>
<p><strong>已废弃（Deprecated）</strong>，不应该在新的代码中使用。</p>
</li>
<li>
<p>它的第二个参数是<strong>长度</strong>，而不是结束索引，这常常是错误的根源。</p>
</li>
</ul>
</li>
</ul>
<blockquote>
<p><strong>切片小结</strong>：一律使用 <code>slice()</code>。它功能强大、行为可预测，是你唯一需要熟练掌握的字符串切片函数。</p>
</blockquote>
<h4 id="13-分割-splitting"><a href="#13-分割-splitting">1.3 分割 (Splitting)</a></h4>
<p>使用 <code>split(separator)</code> 函数，将一个字符串按照指定的分隔符，拆分成一个<strong>数组</strong>。</p>
<ul>
<li>
<p><code>separator</code> 可以是字符串，也可以是正则表达式。</p>
</li>
<li>
<p>如果 <code>separator</code> 是空字符串 <code>''</code>，会将字符串的每个字符都拆分开。</p>
</li>
</ul>
<p>JavaScript</p>
<pre><code class="hljs language-perl">const sentence = <span class="hljs-string">'The-quick-brown-fox'</span>;
const words = sentence.<span class="hljs-keyword">split</span>(<span class="hljs-string">'-'</span>); <span class="hljs-regexp">//</span> 使用 <span class="hljs-string">'-'</span> 分割
console.log(words); <span class="hljs-regexp">//</span> 输出: [<span class="hljs-string">'The'</span>, <span class="hljs-string">'quick'</span>, <span class="hljs-string">'brown'</span>, <span class="hljs-string">'fox'</span>]

const letters = <span class="hljs-string">'abc'</span>.<span class="hljs-keyword">split</span>(<span class="hljs-string">''</span>); <span class="hljs-regexp">//</span> 使用空字符串分割
console.log(letters); <span class="hljs-regexp">//</span> 输出: [<span class="hljs-string">'a'</span>, <span class="hljs-string">'b'</span>, <span class="hljs-string">'c'</span>]

const csv = <span class="hljs-string">'one,two,three'</span>;
console.log(csv.<span class="hljs-keyword">split</span>(<span class="hljs-string">','</span>)); <span class="hljs-regexp">//</span> 输出: [<span class="hljs-string">'one'</span>, <span class="hljs-string">'two'</span>, <span class="hljs-string">'three'</span>]
</code></pre>
<p>这是一个<strong>单向</strong>操作：<code>String -> Array</code>。它的逆操作是数组的 <code>join()</code> 方法。</p>
<hr>
<h3 id="2-数组array--可变的数据列表"><a href="#2-数组array--可变的数据列表">2. 数组（Array）- 可变的数据列表</a></h3>
<p>数组是可变的，这意味着很多操作会<strong>直接修改原始数组</strong>。这是与字符串最本质的区别。</p>
<h4 id="21-长度-length"><a href="#21-长度-length">2.1 长度 (length)</a></h4>
<p>和字符串一样，<code>length</code> 是一个<strong>属性</strong>。但数组的 <code>length</code> 是<strong>可读可写</strong>的。你可以通过修改它来截断或延长数组。</p>
<p>JavaScript</p>
<pre><code class="hljs language-ini">const <span class="hljs-attr">arr</span> = [<span class="hljs-number">10</span>, <span class="hljs-number">20</span>, <span class="hljs-number">30</span>, <span class="hljs-number">40</span>, <span class="hljs-number">50</span>]<span class="hljs-comment">;</span>
console.log(arr.length)<span class="hljs-comment">; // 输出: 5</span>

// 截断数组 (这是一个修改操作)
<span class="hljs-attr">arr.length</span> = <span class="hljs-number">3</span><span class="hljs-comment">;</span>
console.log(arr)<span class="hljs-comment">; // 输出: [10, 20, 30]</span>

// 延长数组，新增的项会是 empty (空位)
<span class="hljs-attr">arr.length</span> = <span class="hljs-number">5</span><span class="hljs-comment">;</span>
console.log(arr)<span class="hljs-comment">; // 输出: [10, 20, 30, &#x3C;2 empty items>]</span>
</code></pre>
<h4 id="22-切片-slicing---不修改原数组"><a href="#22-切片-slicing---不修改原数组">2.2 切片 (Slicing) - 不修改原数组</a></h4>
<p><code>slice(startIndex, endIndex)</code></p>
<ul>
<li>
<p>数组的 <code>slice()</code> 和字符串的 <code>slice()</code> <strong>几乎完全一样</strong>！</p>
</li>
<li>
<p>它提取数组的一部分，返回一个<strong>新数组</strong>，并且<strong>不会修改原始数组</strong>。</p>
</li>
<li>
<p>同样支持负数索引。</p>
</li>
</ul>
<p>JavaScript</p>
<pre><code class="hljs language-css">const originalArr = <span class="hljs-selector-attr">[<span class="hljs-string">'a'</span>, <span class="hljs-string">'b'</span>, <span class="hljs-string">'c'</span>, <span class="hljs-string">'d'</span>, <span class="hljs-string">'e'</span>]</span>;
const newArr = originalArr<span class="hljs-selector-class">.slice</span>(<span class="hljs-number">1</span>, <span class="hljs-number">4</span>); // 从索引<span class="hljs-number">1</span>到<span class="hljs-number">4</span> (不含<span class="hljs-number">4</span>)

console<span class="hljs-selector-class">.log</span>(newArr);        // 输出: [<span class="hljs-string">'b'</span>, <span class="hljs-string">'c'</span>, <span class="hljs-string">'d'</span>]
console.<span class="hljs-built_in">log</span>(originalArr);   // 输出: [<span class="hljs-string">'a'</span>, <span class="hljs-string">'b'</span>, <span class="hljs-string">'c'</span>, <span class="hljs-string">'d'</span>, <span class="hljs-string">'e'</span>] (原始数组不变)
</code></pre>
<h4 id="23-分割修改-splicing---修改原数组"><a href="#23-分割修改-splicing---修改原数组">2.3 分割/修改 (Splicing) - 修改原数组</a></h4>
<p>你问题中提到的“数组的分割”，在JS语境下最接近的其实是 <code>splice()</code> 函数。它是一个非常强大的“瑞士军刀”，可以同时实现删除、插入和替换元素，但它会<strong>直接修改原始数组</strong>。</p>
<p><strong><code>splice(startIndex, deleteCount, ...itemsToAdd)</code></strong></p>
<ul>
<li>
<p><code>startIndex</code>: 开始修改的位置。</p>
</li>
<li>
<p><code>deleteCount</code>: 从 <code>startIndex</code> 开始，要删除的元素数量。</p>
</li>
<li>
<p><code>...itemsToAdd</code>: (可选) 要在 <code>startIndex</code> 位置插入的新元素。</p>
</li>
<li>
<p><strong>返回值</strong>：一个包含<strong>被删除元素</strong>的数组。如果没有删除任何元素，则返回空数组。</p>
</li>
</ul>
<p><strong>使用场景:</strong></p>
<ol>
<li>
<p><strong>仅删除</strong>:</p>
<p>JavaScript</p>
<pre><code class="hljs language-css">const arr = <span class="hljs-selector-attr">[<span class="hljs-string">'a'</span>, <span class="hljs-string">'b'</span>, <span class="hljs-string">'c'</span>, <span class="hljs-string">'d'</span>, <span class="hljs-string">'e'</span>]</span>;
const deleted = arr<span class="hljs-selector-class">.splice</span>(<span class="hljs-number">1</span>, <span class="hljs-number">2</span>); // 从索引<span class="hljs-number">1</span>开始，删除<span class="hljs-number">2</span>个元素
console<span class="hljs-selector-class">.log</span>(deleted);   // 输出: [<span class="hljs-string">'b'</span>, <span class="hljs-string">'c'</span>]
console.<span class="hljs-built_in">log</span>(arr);       // 输出: [<span class="hljs-string">'a'</span>, <span class="hljs-string">'d'</span>, <span class="hljs-string">'e'</span>] (原数组被修改)
</code></pre>
</li>
<li>
<p><strong>仅插入</strong>:</p>
<p>JavaScript</p>
<pre><code class="hljs language-perl">const arr = [<span class="hljs-string">'a'</span>, <span class="hljs-string">'d'</span>, <span class="hljs-string">'e'</span>];
arr.splice(<span class="hljs-number">1</span>, <span class="hljs-number">0</span>, <span class="hljs-string">'x'</span>, <span class="hljs-string">'y'</span>); <span class="hljs-regexp">//</span> 从索引<span class="hljs-number">1</span>开始，删除<span class="hljs-number">0</span>个元素，然后插入 <span class="hljs-string">'x'</span> 和 <span class="hljs-string">'y'</span>
console.log(arr); <span class="hljs-regexp">//</span> 输出: [<span class="hljs-string">'a'</span>, <span class="hljs-string">'x'</span>, <span class="hljs-string">'y'</span>, <span class="hljs-string">'d'</span>, <span class="hljs-string">'e'</span>] (原数组被修改)
</code></pre>
</li>
<li>
<p><strong>替换 (删除并插入)</strong>:</p>
<p>JavaScript</p>
<pre><code class="hljs language-perl">const arr = [<span class="hljs-string">'a'</span>, <span class="hljs-string">'b'</span>, <span class="hljs-string">'c'</span>, <span class="hljs-string">'d'</span>, <span class="hljs-string">'e'</span>];
arr.splice(<span class="hljs-number">1</span>, <span class="hljs-number">2</span>, <span class="hljs-string">'z'</span>); <span class="hljs-regexp">//</span> 从索引<span class="hljs-number">1</span>开始，删除<span class="hljs-number">2</span>个元素(<span class="hljs-string">'b'</span>,<span class="hljs-string">'c'</span>)，然后在该位置插入 <span class="hljs-string">'z'</span>
console.log(arr); <span class="hljs-regexp">//</span> 输出: [<span class="hljs-string">'a'</span>, <span class="hljs-string">'z'</span>, <span class="hljs-string">'d'</span>, <span class="hljs-string">'e'</span>] (原数组被修改)
</code></pre>
</li>
</ol>

<h3 id="总结与记忆技巧"><a href="#总结与记忆技巧">总结与记忆技巧</a></h3>
<p>混淆主要发生在 <code>slice</code> 和 <code>splice</code> 之间，以及它们在字符串和数组上的行为。</p>
<table>
<thead>
<tr>
<th>操作</th>
<th>字符串 (String) - 不可变</th>
<th>数组 (Array) - 可变</th>
<th>如何记忆避免混淆</th>
</tr>
</thead>
<tbody>
<tr>
<td><strong>长度</strong></td>
<td><code>str.length</code> (只读)</td>
<td><code>arr.length</code> (可读写)</td>
<td>数组更“灵活”，长度可以被你手动改变。</td>
</tr>
<tr>
<td><strong>切片 (取一部分)</strong></td>
<td><strong><code>slice()</code></strong> (返回新字符串)</td>
<td><strong><code>slice()</code></strong> (返回新数组)</td>
<td><strong><code>slice</code> 是安全的“切片”</strong>。它就像从蛋糕上切下一块，既得到了新的一块，也没毁掉原来的蛋糕。<strong>行为在字符串和数组上高度一致，都是创建副本</strong>。</td>
</tr>
<tr>
<td><strong>分割/修改</strong></td>
<td><strong><code>split()</code></strong> (字符串 -> 数组)</td>
<td><strong><code>splice()</code></strong> (修改原数组)</td>
<td><strong>1. 拼写上的差异</strong>：<code>splice</code> 比 <code>slice</code> 多一个 <code>p</code>，你可以把 <code>p</code> 想象成 <strong>Permanent</strong> (永久的) 或者 <strong>Powerful</strong> (强大的)，暗示它会造成<strong>永久性、更强力</strong>的修改。  <br><strong>2. 功能上的对立</strong>： <code>split</code> 是把一个东西（字符串）拆成一堆（数组），而 <code>splice</code> 是在一堆东西（数组）里进行“手术”（增/删/改）。它们的目标和应用场景完全不同。</td>
</tr>
</tbody>
</table>
<p><strong>核心记忆法则：</strong></p>
<ol>
<li>
<p><strong>不变的字符串 vs 可变的数组</strong>：这是所有区别的根源。</p>
</li>
<li>
<p><strong><code>slice</code> 是君子</strong>：它只拷贝，不动你的原数据。无论用在字符串还是数组上，都很“礼貌”。这是你的首选“切片”工具。</p>
</li>
<li>
<p><strong><code>splice</code> 是刺客</strong>：它直接对原数组动刀（增、删、改）。名字里的 <code>p</code> 提醒你它的破坏性（Permanent change）。它的返回值是被它“干掉”的元素。</p>
</li>
<li>
<p><strong><code>split</code> 和 <code>join</code> 是一对</strong>：<code>string.split()</code> 把字符串变成数组，<code>array.join()</code> 把数组变回字符串。它们是数据形态转换的桥梁。</p>
</li>
</ol>
<p>当你准备操作数据时，先问自己一个问题：“我希望修改原始数据，还是得到一个副本？”</p>
<ul>
<li>
<p>想得到副本？用 <code>slice()</code>。</p>
</li>
<li>
<p>想直接修改原数组？用 <code>splice()</code>。</p>
</li>
<li>
<p>想把字符串拆成数组？用 <code>split()</code>。</p>
</li>
</ul>
