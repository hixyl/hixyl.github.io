<h3 id="1-keep-alive-是什么"><a href="#1-keep-alive-是什么">1. <code>keep-alive</code> 是什么？</a></h3>
<p><code>keep-alive</code> 是 Vue 内置的一个抽象组件。它的核心功能是<strong>缓存</strong>那些被它包裹、且暂时不需要渲染在页面上的组件实例，而不是直接销毁它们。</p>
<p>想象一下你电脑上的应用程序。关闭一个程序（比如 Word），下次再打开需要重新加载；而将它最小化，它其实还在后台运行，下次点击就能立刻恢复到之前的状态。<code>keep-alive</code> 就扮演了“最小化”而不是“关闭”的角色。</p>
<p><strong>它的主要用途和优点：</strong></p>
<ol>
<li>
<p><strong>性能优化</strong>：对于一些创建成本较高的组件（例如，需要请求数据、进行复杂计算后才能渲染的组件），<code>keep-alive</code> 可以避免它们被反复创建和销毁，显著提升性能。</p>
</li>
<li>
<p><strong>保留组件状态</strong>：当组件被切换时，它内部的状态（如 data、props、computed 等）和用户已经进行的操作（如输入的文字、滚动的距离）都会被完整地保留下来。这对于提升用户体验至关重要，比如在多个表单页签之间切换时，用户不希望已经填写的内容丢失。</p>
</li>
<li>
<p><strong>避免重复请求</strong>：由于组件实例没有被销毁，其 <code>created</code> 或 <code>mounted</code> 生命周期钩子不会重复触发，因此可以避免重复发送网络请求。</p>
</li>
</ol>
<p>它有三个主要的 props：</p>
<ul>
<li>
<p><code>include</code>: 字符串或正则表达式。只有名称匹配的组件才会被缓存。</p>
</li>
<li>
<p><code>exclude</code>: 字符串或正则表达式。任何名称匹配的组件都不会被缓存。</p>
</li>
<li>
<p><code>max</code>: 数字。用于指定最多可以缓存多少个组件实例。一旦达到这个数字，Vue 会采用 LRU (Least Recently Used，最近最少使用) 策略，移除最久没有被访问的缓存组件。</p>
</li>
</ul>
<h3 id="2-缓存组件中的定时器问题及解决方案"><a href="#2-缓存组件中的定时器问题及解决方案">2. 缓存组件中的定时器问题及解决方案</a></h3>
<p>这是一个非常经典的问题。问题的根源在于，<code>keep-alive</code> 改变了组件的生命周期。</p>
<p><strong>问题分析：</strong></p>
<p>当一个被 <code>keep-alive</code> 包裹的组件从视图中被“隐藏”时，它并<strong>不会</strong>触发 <code>beforeUnmount</code> / <code>beforeDestroy</code> 这类销毁钩子。它只是被“停用”（deactivated）。如果你在 <code>mounted</code> 或 <code>created</code> 钩子中启动了一个 <code>setInterval</code> 或 <code>setTimeout</code>，并且打算在 <code>beforeUnmount</code> 中用 <code>clearInterval</code> 或 <code>clearTimeout</code> 来清除它，那么这个清除操作将永远不会发生。</p>
<p>结果就是，这个组件虽然在界面上看不到了，但它内部的定时器依然在后台“孜孜不倦”地运行。这会导致：</p>
<ul>
<li>
<p><strong>内存泄漏</strong>：定时器和它闭包中的变量无法被垃圾回收。</p>
</li>
<li>
<p><strong>不必要的计算</strong>：定时器函数持续执行，消耗 CPU 资源。</p>
</li>
<li>
<p><strong>逻辑错误</strong>：定时器可能会继续执行一些你意想不到的操作，比如在后台更新数据，甚至尝试操作一个已经不存在于 DOM 中的元素而导致报错。</p>
</li>
</ul>
<p><strong>解决方案：使用 <code>activated</code> 和 <code>deactivated</code></strong></p>
<p>为了解决这个问题，<code>keep-alive</code> 专门提供了两个新的生命周期钩子：</p>
<ul>
<li>
<p><code>activated</code>：在被缓存的组件<strong>被激活</strong>（即重新插入到 DOM 中）时调用。</p>
</li>
<li>
<p><code>deactivated</code>：在被缓存的组件<strong>被停用</strong>（即从 DOM 中移除）时调用。</p>
</li>
</ul>
<p>因此，正确的做法是，在组件被激活时开启定时器，在组件被停用时清除它。</p>
<p>同时，<strong>必须</strong>在组件最终被销毁的钩子（<code>beforeUnmount</code> / <code>beforeDestroy</code>）里也进行一次清除。这是为了防止组件因为 <code>keep-alive</code> 的 <code>max</code> 限制而被移除缓存，或者整个 <code>keep-alive</code> 组件被销毁时，定时器得不到清理，从而造成内存泄漏。</p>

<h3 id="3-keep-alive-的实现原理"><a href="#3-keep-alive-的实现原理">3. <code>keep-alive</code> 的实现原理</a></h3>
<p>理解其原理需要对 Vue 的虚拟 DOM (VNode) 和渲染过程有一定认识。</p>
<p><code>keep-alive</code> 的核心在于它<strong>重写了自身的 <code>render</code> 函数</strong>，并在其中进行<strong>VNode 缓存</strong>和<strong>手动管理</strong>。</p>
<p>整个过程可以分解为以下几个关键步骤：</p>
<ol>
<li>
<p>初始化缓存：</p>
<p>keep-alive 在其 created 生命周期钩子中，会初始化一个 cache 对象（用于存储 VNode）和一个 keys 数组（用于支持 LRU 策略）。this.cache = {}, this.keys = []。</p>
</li>
<li>
<p>获取子组件 VNode：</p>
<p>keep-alive 是一个抽象组件，它本身不渲染任何 DOM 元素，而是去渲染它的默认插槽（this.$slots.default）里的第一个子组件。在其 render 函数中，它会拿到这个子组件的 VNode。</p>
</li>
<li>
<p>缓存命中检查：</p>
<p>render 函数会根据子组件 VNode 生成一个唯一的缓存键（key）。这个 key 通常是组件注册的 name 选项，或者组件自身设置的 key 属性。然后，keep-alive 会用这个 key 去 this.cache 对象里查找。</p>
</li>
<li>
<p><strong>处理缓存命中 (Cache Hit)</strong>：</p>
<ul>
<li>
<p>如果在 <code>this.cache</code> 中找到了对应的 VNode，意味着这个组件实例已经被缓存。</p>
</li>
<li>
<p><code>keep-alive</code> 不会重新创建组件，而是直接复用这个 VNode 对应的组件实例（<code>vnode.componentInstance</code>）。</p>
</li>
<li>
<p>它会将这个实例的 <code>$el</code> (真实 DOM 元素) 直接插入到父容器中。</p>
</li>
<li>
<p>同时，它会将这个 <code>key</code> 从 <code>this.keys</code> 数组中移动到末尾，表示它刚刚被访问过（用于 LRU）。</p>
</li>
<li>
<p>最后，调用该组件实例的 <code>activated</code> 钩子。</p>
</li>
</ul>
</li>
<li>
<p><strong>处理缓存未命中 (Cache Miss)</strong>：</p>
<ul>
<li>
<p>如果在 <code>this.cache</code> 中没有找到对应的 VNode，说明这是一个新组件或者是一个之前被移出缓存的组件。</p>
</li>
<li>
<p><code>keep-alive</code> 会正常渲染这个组件，并将其 VNode 存储到 <code>this.cache</code> 中，<code>key</code> 也存入 <code>this.keys</code> 数组。</p>
</li>
<li>
<p><strong>LRU 策略</strong>：此时，它会检查缓存的组件数量是否超过了 <code>max</code> 限制。如果超过了，它会取出 <code>this.keys</code> 数组的第一个元素（也就是最久未被访问的 <code>key</code>），从 <code>this.cache</code> 中删除对应的 VNode，并<strong>真正地销毁</strong>该组件实例（调用其 <code>$destroy</code> 方法）。</p>
</li>
</ul>
</li>
<li>
<p>组件停用：</p>
<p>当一个原本被渲染的组件需要被切换掉时（例如，路由跳转），keep-alive 的 render 函数会发现新的子组件 VNode 和旧的不同。它不会销毁旧的组件实例，而是仅仅将其保留在 this.cache 中，并调用其 deactivated 钩子。然后，再挂载新的组件。</p>
</li>
</ol>
