4:T3afd,<p>本文档旨在解析一个基于 React 的交互功能：用户在一段不可编辑的文本内容上，通过鼠标划词选中，将选中的文本添加为一个“关键词”，并使该段落中所有与关键词匹配的文本都高亮显示。</p>
<p>此功能的核心在于巧妙地结合了浏览器原生 API、React 的状态管理和动态渲染机制。</p>
<h4 id="一-核心逻辑概览"><a href="#一-核心逻辑概览"><strong>一、 核心逻辑概览</strong></a></h4>
<p>整个功能的实现可以分解为三个主要步骤：</p>
<ol>
<li>
<p><strong>捕获用户选择</strong>: 监听鼠标事件，当用户在文本上完成划词动作时，获取选中的文本内容及其在屏幕上的位置。</p>
</li>
<li>
<p><strong>状态驱动交互</strong>: 将选中的文本作为“关键词”存入 React 的 state 中。同时，根据鼠标位置，动态显示一个操作浮窗（Popover），引导用户确认操作。</p>
</li>
<li>
<p><strong>动态渲染高亮</strong>: 监听“关键词” state 的变化。当 state 更新时，重新计算并渲染文本内容，将所有匹配关键词的部分用特定样式的 <code>&#x3C;span></code> 标签包裹，从而实现高亮效果。</p>
</li>
</ol>
<h4 id="二-技术实现详解"><a href="#二-技术实现详解"><strong>二、 技术实现详解</strong></a></h4>
<h5 id="1-捕获用户选择-handlemouseup-事件"><a href="#1-捕获用户选择-handlemouseup-事件"><strong>1. 捕获用户选择 (<code>handleMouseUp</code> 事件)</strong></a></h5>
<p>这是整个交互的起点。我们不在乎用户拖拽的过程，只关心最终选定的结果，因此 <code>onMouseUp</code> 事件是最佳的触发时机。</p>
<ul>
<li>
<p><strong>事件绑定</strong>: 在展示话术内容的 <code>&#x3C;div></code> 元素上绑定 <code>onMouseUp={handleMouseUp}</code>。</p>
</li>
<li>
<p><strong>获取选中对象</strong>: 使用浏览器原生 API <code>window.getSelection()</code> 来获取一个 <code>Selection</code> 对象，该对象代表用户当前选中的文本范围。</p>
</li>
<li>
<p><strong>提取文本与位置</strong>:</p>
<ul>
<li>
<p>通过 <code>selection.toString().trim()</code> 可以轻松获得用户选中的字符串。</p>
</li>
<li>
<p>通过 <code>selection.getRangeAt(0).getBoundingClientRect()</code> 可以获取选中区域的 <code>DOMRect</code> 对象，其中包含了 <code>x</code>, <code>y</code>, <code>width</code>, <code>height</code>, <code>top</code>, <code>bottom</code>, <code>left</code>, <code>right</code> 等精确的屏幕位置信息。</p>
</li>
</ul>
</li>
<li>
<p><strong>显示操作浮窗</strong>: 当确定用户选中了文本后，利用获取到的位置信息（如此处代码中的 <code>rect.bottom</code> 和 <code>rect.left</code>），更新一个 <code>popover</code> 状态，使其 <code>visible</code> 变为 <code>true</code> 并在计算好的位置上渲染出来。</p>
</li>
</ul>
<p>JavaScript</p>

<ul>
<li>

<h5 id="3-动态渲染与高亮实现-rendercontentwithhighlights"><a href="#3-动态渲染与高亮实现-rendercontentwithhighlights"><strong>3. 动态渲染与高亮实现 (<code>renderContentWithHighlights</code>)</strong></a></h5>
<p>这是整个功能最核心、最精妙的部分。它负责将一段普通字符串，根据 <code>fillInBlanks</code> 列表，转换成一个包含高亮 <code>&#x3C;span></code> 的 React 元素数组。</p>
<ul>
<li>
<p><strong>核心思路</strong>: 使用正则表达式匹配所有关键词，并利用 <code>String.prototype.split()</code> 方法的特性来分割字符串。</p>
</li>
<li>
<p><strong>实现步骤</strong>:</p>
<ol>
<li>
<p><strong>准备关键词</strong>: 从 <code>fillInBlanks</code> state 中获取所有需要高亮的词。</p>
</li>
<li>
<p><strong>排序（关键优化）</strong>: <strong>将关键词按长度从长到短排序</strong>。这是一个至关重要的细节，可以防止复合词的匹配问题。例如，如果同时有 “话术” 和 “命中话术” 两个关键词，如果不排序，<code>"命中话术"</code> 可能会先被 <code>"话术"</code> 匹配分割，导致 <code>"命中"</code> 成为普通文本。排序后，会优先匹配更长的 <code>"命中话术"</code>。</p>
</li>
<li>
<p><strong>构建正则表达式</strong>:</p>
<ul>
<li>
<p>对每个关键词进行特殊字符转义，防止关键词中的 <code>.</code>、<code>(</code> 等字符干扰正则。</p>
</li>
<li>
<p>使用 <code>|</code> (或) 将所有关键词连接起来，形成一个如 <code>(命中话术|话术|填空)</code> 的正则表达式。<strong>注意两边的括号 <code>()</code></strong>，这是为了创建一个捕获组（Capturing Group）。</p>
</li>
</ul>
</li>
<li>
<p><strong>分割字符串</strong>: 调用 <code>content.split(regex)</code>。当 <code>split</code> 的参数是带有捕获组的正则表达式时，其返回的数组中会包含被匹配到的分隔符（也就是我们的关键词）。</p>
<ul>
<li>例如：<code>'这是命中话术'.split(/(命中话术)/)</code> 会得到 <code>['这是', '命中话术', '']</code>。</li>
</ul>
</li>
<li>
<p><strong>渲染 JSX</strong>: 遍历分割后的数组 <code>parts</code>。检查每一部分 <code>part</code> 是否存在于我们的关键词列表中。</p>
<ul>
<li>
<p>如果是，渲染成 <code>&#x3C;span className={css.highlightedText}>{part}&#x3C;/span></code>。</p>
</li>
<li>
<p>如果不是，直接渲染该文本 <code>part</code>。</p>
</li>
</ul>
</li>
</ol>
</li>
</ul>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-comment">// 关键代码片段 3: 渲染高亮</span>

<h4 id="三-总结与提炼"><a href="#三-总结与提炼"><strong>三、 总结与提炼</strong></a></h4>
<p>这个功能是一个优秀的前端交互案例，其精华在于：</p>
<ul>
<li>
<p><strong>组合利用原生 API</strong>: 充分利用了 <code>window.getSelection</code> 来处理复杂的文本选中逻辑，而不是自己造轮子。</p>
</li>
<li>
<p><strong>清晰的数据流</strong>: 严格遵循 React 的单向数据流，<code>fillInBlanks</code> state 是驱动所有UI变化的唯一可信源。</p>
</li>
<li>
<p><strong>巧妙的字符串处理</strong>: <code>split</code> 方法配合带捕获组的正则表达式，是实现文本分割与动态插入标签的最高效、最优雅的方案之一。</p>
</li>
<li>
<p><strong>关注细节与体验</strong>: 对关键词进行降序排序处理了匹配的边界情况；操作后清除浏览器默认选中样式，提升了交互的流畅度。</p>
</li>
</ul>
