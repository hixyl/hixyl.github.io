1c:T226c,<p>在前端开发中，处理引用数据类型（对象、数组）时，深拷贝是一个绕不开的核心概念。它的本质是在内存的堆区重新开辟一块空间，完整地复制目标对象及其所有嵌套的子对象，从而彻底切断与原对象的引用联系。</p>
<p>为了在不同业务场景中做出最合理的选择，我们可以将前端实现深拷贝的方法从原生 API、历史 hack 方案到工程化实现进行系统梳理：</p>
<h3 id="1-现代原生标准structuredclone"><a href="#1-现代原生标准structuredclone">1. 现代原生标准：<code>structuredClone()</code></a></h3>
<p>这是目前浏览器原生提供的深拷贝全局方法，它基于结构化克隆算法（Structured Clone Algorithm），也是当下大部分常规业务的首选。</p>
<ul>
<li>
<p><strong>适用场景</strong>：现代浏览器环境（兼容性良好，Node.js 17+ 也已支持），需要拷贝包含内置对象（如 <code>Map</code>、<code>Set</code>、<code>Date</code>、<code>RegExp</code>、<code>ArrayBuffer</code> 等）或存在循环引用的复杂数据。</p>
</li>
<li>
<p><strong>优势</strong>：</p>
<ul>
<li>
<p>原生 API，性能优异。</p>
</li>
<li>
<p>完美支持循环引用，不会导致栈溢出。</p>
</li>
<li>
<p>支持绝大多数 JavaScript 内置数据类型。</p>
</li>
</ul>
</li>
<li>
<p><strong>局限性</strong>：</p>
<ul>
<li>
<p><strong>无法拷贝函数</strong>（会抛出 <code>DataCloneError</code> 异常）。</p>
</li>
<li>
<p>无法拷贝 DOM 节点。</p>
</li>
<li>
<p>无法保留对象的原型链（克隆出的对象不再是原类的实例，而是普通对象 <code>Object</code>）。</p>
</li>
</ul>
</li>
</ul>
<h3 id="2-经典的快捷方案jsonparsejsonstringify"><a href="#2-经典的快捷方案jsonparsejsonstringify">2. 经典的快捷方案：<code>JSON.parse(JSON.stringify())</code></a></h3>
<p>这是早期前端最常用的深拷贝方法，本质上是利用 JSON 的序列化和反序列化来实现内存的重新分配。</p>
<ul>
<li>
<p><strong>适用场景</strong>：纯粹的、不包含特殊类型和循环引用的简单数据载体（如后端返回的纯 JSON 树状结构）。</p>
</li>
<li>
<p><strong>优势</strong>：代码极其简短，执行速度在处理纯数据时表现尚可，兼容性完美。</p>
</li>
<li>
<p><strong>致命缺陷（边界情况多）</strong>：</p>
<ul>
<li>
<p><strong>丢失数据</strong>：忽略 <code>undefined</code>、<code>Symbol</code>、函数。如果它们在数组中，会被转换为 <code>null</code>；如果在对象中，该键值对直接消失。</p>
</li>
<li>
<p><strong>类型畸变</strong>：<code>Date</code> 对象会被转换为字符串，<code>RegExp</code> 和 <code>Error</code> 对象会被转换为空对象 <code>{}</code>，<code>NaN</code>、<code>Infinity</code> 会被转为 <code>null</code>。</p>
</li>
<li>
<p><strong>报错</strong>：遇到循环引用会直接抛出 <code>TypeError</code>。</p>
</li>
</ul>
</li>
</ul>
<h3 id="3-第一性原理实现手写递归拷贝"><a href="#3-第一性原理实现手写递归拷贝">3. 第一性原理实现：手写递归拷贝</a></h3>
<p>当现有 API 无法满足特殊的业务逻辑（例如必须拷贝函数，或者保留特定的原型链）时，我们需要从底层逻辑出发手写递归。其核心逻辑是：<strong>遍历目标对象的所有属性，遇到基本类型直接返回，遇到引用类型则递归调用自身，同时利用哈希表记录已拷贝的对象以打破循环。</strong></p>
<p>JavaScript</p>
<pre><code class="hljs language-javascript"><span class="hljs-keyword">function</span> <span class="hljs-title function_">deepClone</span>(<span class="hljs-params">target, map = <span class="hljs-keyword">new</span> <span class="hljs-built_in">WeakMap</span>()</span>) {
  <span class="hljs-comment">// 1. 处理基础类型和 null</span>
  <span class="hljs-keyword">if</span> (target === <span class="hljs-literal">null</span> || <span class="hljs-keyword">typeof</span> target !== <span class="hljs-string">'object'</span>) {
    <span class="hljs-keyword">return</span> target;
  }

  <span class="hljs-comment">// 2. 处理循环引用</span>
  <span class="hljs-keyword">if</span> (map.<span class="hljs-title function_">has</span>(target)) {

  }

  <span class="hljs-comment">// 3. 处理特定内置对象 (可以根据需要扩展 RegExp, Date 等)</span>
  <span class="hljs-keyword">if</span> (target <span class="hljs-keyword">instanceof</span> <span class="hljs-title class_">Date</span>) <span class="hljs-keyword">return</span> <span class="hljs-keyword">new</span> <span class="hljs-title class_">Date</span>(target);
  <span class="hljs-keyword">if</span> (target <span class="hljs-keyword">instanceof</span> <span class="hljs-title class_">RegExp</span>) <span class="hljs-keyword">return</span> <span class="hljs-keyword">new</span> <span class="hljs-title class_">RegExp</span>(target);

  <span class="hljs-comment">// 4. 初始化克隆对象，保持原型链</span>
  <span class="hljs-keyword">const</span> cloneTarget = <span class="hljs-title class_">Array</span>.<span class="hljs-title function_">isArray</span>(target) ? [] : <span class="hljs-title class_">Object</span>.<span class="hljs-title function_">create</span>(<span class="hljs-title class_">Object</span>.<span class="hljs-title function_">getPrototypeOf</span>(target));
  
  <span class="hljs-comment">// 5. 记录到 WeakMap 中</span>
  map.<span class="hljs-title function_">set</span>(target, cloneTarget);

  <span class="hljs-comment">// 6. 递归处理属性 (包括 Symbol 属性)</span>
  <span class="hljs-title class_">Reflect</span>.<span class="hljs-title function_">ownKeys</span>(target).<span class="hljs-title function_">forEach</span>(<span class="hljs-function"><span class="hljs-params">key</span> =></span> {
    cloneTarget[key] = <span class="hljs-title function_">deepClone</span>(target[key], map);
  });

  <span class="hljs-keyword">return</span> cloneTarget;
}

<ul>
<li><strong>核心细节</strong>：使用 <code>WeakMap</code> 而不是 <code>Map</code> 来存储对象映射关系，是为了避免引起内存泄漏。<code>WeakMap</code> 的键是弱引用，当原对象被垃圾回收时，其映射关系也会自动解除。</li>
</ul>
<h3 id="4-稳健的工程化选择第三方库如-lodash-的-_clonedeep"><a href="#4-稳健的工程化选择第三方库如-lodash-的-_clonedeep">4. 稳健的工程化选择：第三方库（如 Lodash 的 <code>_.cloneDeep</code>）</a></h3>
<p>在大型企业级项目中，如果你不想自己维护庞大且复杂的深拷贝工具函数，引入成熟的第三方库是降低风险的最佳方式。</p>
<ul>
<li>
<p><strong>适用场景</strong>：复杂的老旧项目，或者对各种边界情况（包括各种奇怪的类型对象、Buffer 等）有极高稳定要求的场景。</p>
</li>
<li>
<p><strong>优势</strong>：久经沙场，几乎处理了 JavaScript 中所有能想到的边界情况。</p>
</li>
<li>
<p><strong>局限性</strong>：会增加打包后的代码体积（虽然可以通过按需引入缓解），在处理简单对象时，由于内部判断逻辑过于庞大，性能开销反而比原生方法大。</p>
</li>
</ul>
