<h3 id="objectcreate-的深入解析"><a href="#objectcreate-的深入解析"><code>Object.create()</code> 的深入解析</a></h3>
<p><code>Object.create()</code> 是一个非常基础且强大的方法，它允许我们创建一个新对象，并<strong>精确地指定这个新对象的原型（prototype）</strong>。这是理解 <code>Object.create()</code> 的核心。相较于通过 <code>new</code> 关键字或对象字面量 <code>{}</code> 创建对象，<code>Object.create()</code> 给了你对原型链最直接的控制权。</p>
<h4 id="语法"><a href="#语法">语法</a></h4>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-title class_">Object</span>.<span class="hljs-title function_">create</span>(proto, propertiesObject)
</code></pre>
<p>它接收两个参数：</p>
<ol>
<li>
<p><code>proto</code>: <strong>必需参数</strong>。一个对象，它将成为新创建对象的原型。它可以是任何对象，甚至可以是 <code>null</code>。</p>
</li>
<li>
<p><code>propertiesObject</code>: <strong>可选参数</strong>。一个对象，用于为新对象定义额外的自有属性（own properties），其格式与 <code>Object.defineProperties()</code> 的第二个参数相同。</p>
</li>
</ol>
<hr>
<h4 id="核心用法与场景"><a href="#核心用法与场景">核心用法与场景</a></h4>
<h5 id="1-实现纯粹的原型式继承"><a href="#1-实现纯粹的原型式继承">1. 实现纯粹的原型式继承</a></h5>
<p>这是 <code>Object.create()</code> 最经典、最主要的用途。它能让你清晰地建立对象间的继承关系。</p>
<p>假设我们有一个“动物”原型，它有一个 <code>breathe</code> 方法：</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">const</span> animal = {
  <span class="hljs-attr">isAlive</span>: <span class="hljs-literal">true</span>,
  <span class="hljs-title function_">breathe</span>(<span class="hljs-params"></span>) {
    <span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">'inhale, exhale'</span>);
  }
};

<span class="hljs-comment">// 创建一个 dog 对象，它的原型是 animal</span>
<span class="hljs-keyword">const</span> dog = <span class="hljs-title class_">Object</span>.<span class="hljs-title function_">create</span>(animal);
dog.<span class="hljs-property">breed</span> = <span class="hljs-string">'Labrador'</span>; <span class="hljs-comment">// 为 dog 添加自有属性</span>

<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(dog.<span class="hljs-property">isAlive</span>);      <span class="hljs-comment">// true (继承自 animal)</span>
dog.<span class="hljs-title function_">breathe</span>();                 <span class="hljs-comment">// "inhale, exhale" (继承自 animal)</span>
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(dog.<span class="hljs-title function_">hasOwnProperty</span>(<span class="hljs-string">'breed'</span>));   <span class="hljs-comment">// true (breed 是 dog 自己的属性)</span>
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(dog.<span class="hljs-title function_">hasOwnProperty</span>(<span class="hljs-string">'isAlive'</span>)); <span class="hljs-comment">// false (isAlive 是原型链上的属性)</span>
</code></pre>
<p><strong>背后发生了什么？</strong></p>
<p><code>dog</code> 对象本身是空的（除了我们后来添加的 <code>breed</code> 属性）。当我们访问 <code>dog.isAlive</code> 时，JavaScript 引擎在 <code>dog</code> 对象上找不到它，于是就会沿着它的原型链向上查找，在 <code>animal</code> 对象上找到了 <code>isAlive</code> 并返回其值。这个查找链条是：<code>dog ---> animal ---> Object.prototype ---> null</code>。</p>
<h5 id="2-创建一个纯净的对象-objectcreatenull"><a href="#2-创建一个纯净的对象-objectcreatenull">2. 创建一个“纯净”的对象 (<code>Object.create(null)</code>)</a></h5>
<p>这是一个非常重要的特例。通常，我们用 <code>{}</code> 创建的对象，其原型默认是 <code>Object.prototype</code>，因此它天生就带有一些方法，如 <code>toString()</code>, <code>hasOwnProperty()</code> 等。</p>
<p>但在某些场景下，我们希望创建一个完全干净、不继承任何东西的对象，把它当作一个纯粹的“哈希表”或“字典”来使用，以避免潜在的键名冲突。</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-comment">// 创建一个“纯净”的 map 对象</span>
<span class="hljs-keyword">const</span> map = <span class="hljs-title class_">Object</span>.<span class="hljs-title function_">create</span>(<span class="hljs-literal">null</span>);

map.<span class="hljs-property">name</span> = <span class="hljs-string">'My Map'</span>;
map.<span class="hljs-property">toString</span> = <span class="hljs-string">'a string'</span>; <span class="hljs-comment">// 这里的 toString 只是一个普通的属性键</span>

<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(map.<span class="hljs-property">name</span>); <span class="hljs-comment">// "My Map"</span>

<span class="hljs-comment">// 因为 map 的原型是 null，它没有任何继承的方法</span>
<span class="hljs-comment">// console.log(map.toString()); // 这会抛出错误: Uncaught TypeError: map.toString is not a function</span>
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(map[<span class="hljs-string">'toString'</span>]); <span class="hljs-comment">// "a string"</span>

<span class="hljs-comment">// 同样，它也没有 hasOwnProperty 方法</span>
<span class="hljs-comment">// console.log(map.hasOwnProperty('name')); // 错误: map.hasOwnProperty is not a function</span>
</code></pre>
<p>为什么这很有用？</p>
<p>如果用户输入的数据中恰好有一个键是 hasOwnProperty，而你又把它存入一个普通对象 let obj = {} 中，当你调用 obj.hasOwnProperty 时，你可能无法区分这是用户数据还是原型上的方法。使用 Object.create(null) 创建的对象则完全没有这个问题。</p>
<h5 id="3-使用第二个参数定义属性"><a href="#3-使用第二个参数定义属性">3. 使用第二个参数定义属性</a></h5>
<p>第二个可选参数允许你在创建对象时，就以非常精确的方式定义其自有属性。这提供了比简单赋值更强大的控制力。</p>
<p>属性描述符（Property Descriptors）有以下几个键：</p>
<ul>
<li>
<p><code>value</code>: 属性的值。</p>
</li>
<li>
<p><code>writable</code>: 是否可写（<code>true</code>/<code>false</code>）。</p>
</li>
<li>
<p><code>enumerable</code>: 是否可被 <code>for...in</code> 循环或 <code>Object.keys()</code> 枚举（<code>true</code>/<code>false</code>）。</p>
</li>
<li>
<p><code>configurable</code>: 是否可被删除或更改其特性（<code>true</code>/<code>false</code>）。</p>
</li>
</ul>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">const</span> user = <span class="hljs-title class_">Object</span>.<span class="hljs-title function_">create</span>(<span class="hljs-literal">null</span>, {
  <span class="hljs-comment">// 定义一个数据属性</span>
  <span class="hljs-attr">name</span>: {
    <span class="hljs-attr">value</span>: <span class="hljs-string">'John Doe'</span>,
    <span class="hljs-attr">writable</span>: <span class="hljs-literal">true</span>,
    <span class="hljs-attr">enumerable</span>: <span class="hljs-literal">true</span>
  },
  <span class="hljs-comment">// 定义一个只读的、不可枚举的属性</span>
  <span class="hljs-attr">id</span>: {
    <span class="hljs-attr">value</span>: <span class="hljs-number">123</span>,
    <span class="hljs-attr">writable</span>: <span class="hljs-literal">false</span>,
    <span class="hljs-attr">enumerable</span>: <span class="hljs-literal">false</span>
  }
});

<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(user.<span class="hljs-property">name</span>);      <span class="hljs-comment">// "John Doe"</span>
user.<span class="hljs-property">name</span> = <span class="hljs-string">'Jane Doe'</span>;      <span class="hljs-comment">// 可以修改</span>
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(user.<span class="hljs-property">name</span>);      <span class="hljs-comment">// "Jane Doe"</span>

<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(user.<span class="hljs-property">id</span>);        <span class="hljs-comment">// 123</span>
<span class="hljs-comment">// user.id = 456;            // 在严格模式下会报错，非严格模式下静默失败</span>

<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-title class_">Object</span>.<span class="hljs-title function_">keys</span>(user)); <span class="hljs-comment">// ["name"] (因为 id 的 enumerable 是 false)</span>
</code></pre>

<h3 id="object-上的其他重要静态函数"><a href="#object-上的其他重要静态函数">Object 上的其他重要静态函数</a></h3>
<p><code>Object</code> 构造函数本身也挂载了许多非常有用的静态方法。我们可以将它们分类来理解。</p>
<h4 id="一-对象的创建与修改"><a href="#一-对象的创建与修改">一、 对象的创建与修改</a></h4>
<p>除了 <code>Object.create()</code>，还有：</p>
<ol>
<li>
<p><strong><code>Object.assign(target, ...sources)</code></strong></p>
<ul>
<li>
<p><strong>作用</strong>：将一个或多个源对象（<code>sources</code>）的所有<strong>可枚举的自有属性</strong>复制到目标对象（<code>target</code>）。它会返回修改后的 <code>target</code> 对象。</p>
</li>
<li>
<p><strong>关键点</strong>：这是一个<strong>浅拷贝</strong>（Shallow Copy）。如果源对象的属性值是另一个对象，那么复制的是该对象的引用，而不是对象本身。</p>
</li>
<li>
<p><strong>示例</strong>：</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">const</span> defaults = { <span class="hljs-attr">a</span>: <span class="hljs-number">1</span>, <span class="hljs-attr">b</span>: <span class="hljs-number">2</span> };
<span class="hljs-keyword">const</span> options = { <span class="hljs-attr">b</span>: <span class="hljs-number">3</span>, <span class="hljs-attr">c</span>: <span class="hljs-number">4</span> };
<span class="hljs-keyword">const</span> merged = <span class="hljs-title class_">Object</span>.<span class="hljs-title function_">assign</span>({}, defaults, options); <span class="hljs-comment">// { a: 1, b: 3, c: 4 }</span>
<span class="hljs-comment">// options.b 覆盖了 defaults.b</span>
</code></pre>
</li>
</ul>
</li>
<li>
<p><strong><code>Object.defineProperty(obj, prop, descriptor)</code></strong></p>
<ul>
<li>
<p><strong>作用</strong>：在一个对象上直接定义一个新属性，或修改一个现有属性。</p>
</li>
<li>
<p><strong>关键点</strong>：提供对属性特性的精细控制（writable, enumerable, configurable）。这是实现数据劫持（如 Vue 2.x）的基础。</p>
</li>
</ul>
</li>
<li>
<p><strong><code>Object.defineProperties(obj, props)</code></strong></p>
<ul>
<li><strong>作用</strong>：在一个对象上定义或修改多个属性。<code>Object.create()</code> 的第二个参数内部就使用了这个逻辑。</li>
</ul>
</li>
</ol>
<h4 id="二-获取对象的属性信息自省"><a href="#二-获取对象的属性信息自省">二、 获取对象的属性信息（自省）</a></h4>
<p>这些方法用于检查对象自身的结构。</p>
<ol>
<li>
<p><strong><code>Object.keys(obj)</code></strong></p>
<ul>
<li><strong>作用</strong>：返回一个数组，包含对象自身的所有<strong>可枚举属性</strong>的<strong>键名（字符串）</strong>。</li>
</ul>
</li>
<li>
<p><strong><code>Object.values(obj)</code></strong></p>
<ul>
<li><strong>作用</strong>：返回一个数组，包含对象自身的所有<strong>可枚举属性</strong>的<strong>值</strong>。</li>
</ul>
</li>
<li>
<p><strong><code>Object.entries(obj)</code></strong></p>
<ul>
<li>
<p><strong>作用</strong>：返回一个数组，每个成员是对象自身的一个 <code>[key, value]</code> 键值对数组。对于将对象转换为 <code>Map</code> 或进行 <code>for...of</code> 循环非常有用。</p>
</li>
<li>
<p><strong>示例</strong>：</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">const</span> obj = { <span class="hljs-attr">a</span>: <span class="hljs-number">1</span>, <span class="hljs-attr">b</span>: <span class="hljs-number">2</span> };
<span class="hljs-keyword">for</span> (<span class="hljs-keyword">const</span> [key, value] <span class="hljs-keyword">of</span> <span class="hljs-title class_">Object</span>.<span class="hljs-title function_">entries</span>(obj)) {
  <span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">`<span class="hljs-subst">${key}</span>: <span class="hljs-subst">${value}</span>`</span>);
}
</code></pre>
</li>
</ul>
</li>
<li>
<p><strong><code>Object.getOwnPropertyNames(obj)</code></strong></p>
<ul>
<li><strong>作用</strong>：返回一个数组，包含对象自身的**所有属性（包括不可枚举的属性，但不包括 Symbol 属性）**的键名。</li>
</ul>
</li>
<li>
<p><strong><code>Object.getOwnPropertySymbols(obj)</code></strong></p>
<ul>
<li><strong>作用</strong>：返回一个数组，包含对象自身的所有 Symbol 属性。</li>
</ul>
</li>
<li>
<p><strong><code>Object.getOwnPropertyDescriptor(obj, prop)</code></strong></p>
<ul>
<li><strong>作用</strong>：返回指定对象上一个自有属性的属性描述符。如果属性不存在，则返回 <code>undefined</code>。</li>
</ul>
</li>
<li>
<p><strong><code>Object.getOwnPropertyDescriptors(obj)</code></strong></p>
<ul>
<li><strong>作用</strong>：返回一个对象，包含了指定对象所有自有属性的描述符。</li>
</ul>
</li>
</ol>
<h4 id="三-原型链的操作"><a href="#三-原型链的操作">三、 原型链的操作</a></h4>
<ol>
<li>
<p><strong><code>Object.getPrototypeOf(obj)</code></strong></p>
<ul>
<li><strong>作用</strong>：返回指定对象的原型。这是获取一个对象原型的标准、推荐方法（优于非标准的 <code>__proto__</code>）。</li>
</ul>
</li>
<li>
<p><strong><code>Object.setPrototypeOf(obj, prototype)</code></strong></p>
<ul>
<li>
<p><strong>作用</strong>：设置一个对象的原型。</p>
</li>
<li>
<p><strong>注意</strong>：这是一个非常消耗性能的操作，因为它会改变对象的整个继承链。在性能敏感的代码中应避免使用。最好在对象创建之初就用 <code>Object.create()</code> 来确定其原型。</p>
</li>
</ul>
</li>
</ol>
<h4 id="四-控制对象的状态使其不可变"><a href="#四-控制对象的状态使其不可变">四、 控制对象的状态（使其不可变）</a></h4>
<p>这些方法用于锁定对象，防止其被修改。</p>
<ol>
<li>
<p><strong><code>Object.preventExtensions(obj)</code></strong></p>
<ul>
<li>
<p><strong>作用</strong>：让一个对象<strong>不能再添加</strong>新的属性。已有属性仍然可以被修改或删除。</p>
</li>
<li>
<p>检查方法：<code>Object.isExtensible(obj)</code></p>
</li>
</ul>
</li>
<li>
<p><strong><code>Object.seal(obj)</code></strong></p>
<ul>
<li>
<p><strong>作用</strong>：<strong>密封</strong>一个对象。这会 <code>preventExtensions</code>，并且把所有现有属性标记为 <code>configurable: false</code>。这意味着你不能添加新属性，也不能删除或重新配置现有属性，但<strong>可以修改</strong>现有属性的值（如果它们是 <code>writable: true</code> 的话）。</p>
</li>
<li>
<p>检查方法：<code>Object.isSealed(obj)</code></p>
</li>
</ul>
</li>
<li>
<p><strong><code>Object.freeze(obj)</code></strong></p>
<ul>
<li>
<p><strong>作用</strong>：<strong>冻结</strong>一个对象。这是最严格的级别，它会 <code>seal</code> 对象，并将其所有数据属性的 <code>writable</code> 特性设为 <code>false</code>。对象将变得完全不可变（仅限第一层，是浅冻结）。</p>
</li>
<li>
<p>检查方法：<code>Object.isFrozen(obj)</code></p>
</li>
</ul>
</li>
</ol>
<h4 id="五-实用工具"><a href="#五-实用工具">五、 实用工具</a></h4>
<ol>
<li>
<p><strong><code>Object.is(value1, value2)</code></strong></p>
<ul>
<li>
<p><strong>作用</strong>：判断两个值是否是相同的值。它比 <code>===</code> 更严格。</p>
</li>
<li>
<p><strong>与 <code>===</code> 的区别</strong>：</p>
<ul>
<li>
<p><code>Object.is(NaN, NaN)</code> 返回 <code>true</code> (而 <code>NaN === NaN</code> 返回 <code>false</code>)。</p>
</li>
<li>
<p><code>Object.is(+0, -0)</code> 返回 <code>false</code> (而 <code>+0 === -0</code> 返回 <code>true</code>)。</p>
</li>
</ul>
</li>
</ul>
</li>
<li>
<p><strong><code>Object.fromEntries(iterable)</code></strong></p>
<ul>
<li>
<p><strong>作用</strong>：<code>Object.entries()</code> 的逆操作。它接收一个 <code>[key, value]</code> 形式的迭代器（如数组或 <code>Map</code>），并返回一个新的对象。</p>
</li>
<li>
<p><strong>示例</strong>：</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">const</span> entries = <span class="hljs-keyword">new</span> <span class="hljs-title class_">Map</span>([
  [<span class="hljs-string">'foo'</span>, <span class="hljs-string">'bar'</span>],
  [<span class="hljs-string">'baz'</span>, <span class="hljs-number">42</span>]
]);
<span class="hljs-keyword">const</span> obj = <span class="hljs-title class_">Object</span>.<span class="hljs-title function_">fromEntries</span>(entries); <span class="hljs-comment">// { foo: "bar", baz: 42 }</span>
</code></pre>
</li>
</ul>
</li>
</ol>
<h3 id="总结"><a href="#总结">总结</a></h3>
<p><code>Object</code> 上的这些静态方法为 JavaScript 开发者提供了操作和理解这门语言核心——对象的全套工具。</p>
<ul>
<li>
<p><strong><code>Object.create()</code></strong> 是基石，它让你从<strong>原型</strong>这个根源上控制对象的创建。</p>
</li>
<li>
