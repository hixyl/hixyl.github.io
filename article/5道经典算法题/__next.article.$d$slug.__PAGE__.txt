4:Te1d0,<p>好的，我们来深入探讨一下前端开发面试中那些最经典、最核心的算法题目。</p>
<p>不同于后端或纯算法岗位，前端的算法面试更侧重于考察那些与日常工作场景紧密相关的问题。比如：处理树状结构（DOM、组件树）、应对高频事件、管理异步流程、高效处理数据等。因此，一个优秀的前端工程师不仅要懂“增删改查”，更要理解这些操作背后的效率和逻辑。</p>
<p>下面我将问题分为五大经典类别，每个类别下选择最具代表性的题目，并提供从问题分析、代码实现到场景应用的“一条龙”式彻底讲解。</p>
<hr>
<h3 id="类别一数组与字符串-array--string---数据处理基石"><a href="#类别一数组与字符串-array--string---数据处理基石">类别一：数组与字符串 (Array &#x26; String) - 数据处理基石</a></h3>
<p>这是最基础也是最高频的类别。前端每天都在和接口返回的数组、用户输入的字符串打交道。</p>
<h4 id="经典题目两数之和-two-sum"><a href="#经典题目两数之和-two-sum">经典题目：两数之和 (Two Sum)</a></h4>
<p>题目描述</p>
<p>给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 的那 两个 整数，并返回它们的数组下标。</p>
<p>你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。</p>
<p>示例:</p>
<p>输入: nums = [2, 7, 11, 15], target = 9</p>
<p>输出: [0, 1]</p>
<p>解释: 因为 nums[0] + nums[1] == 9，所以返回 [0, 1]。</p>
<p>为什么这道题对前端如此重要？</p>
<p>这道题考察的是最基本的数据查找和匹配能力。在实际业务中，我们经常需要：</p>
<ul>
<li>
<p>在商品列表中找到价格能凑成某个优惠券满减金额的两个商品。</p>
</li>
<li>
<p>从两个不同的数据源（比如用户信息和订单信息）中，通过某个共同的 ID 找到关联项。</p>
</li>
<li>
<p>这道题的解法演进（从暴力到高效）能很好地体现一个工程师对时间复杂度的理解。</p>
</li>
</ul>
<p><strong>思路演进与代码详解</strong></p>
<p><strong>解法1：暴力枚举（不推荐，但应了解）</strong></p>
<p>最直观的思路是，用两层循环，遍历所有可能的数字组合。</p>
<ul>
<li>
<p><strong>思路</strong>: 第一个指针 <code>i</code> 从头开始，第二个指针 <code>j</code> 从 <code>i+1</code> 开始，检查 <code>nums[i] + nums[j]</code> 是否等于 <code>target</code>。</p>
</li>
<li>
<p><strong>时间复杂度</strong>: O(n2)，因为是嵌套循环。</p>
</li>
<li>
<p><strong>空间复杂度</strong>: O(1)，没有使用额外的存储空间。</p>
</li>
</ul>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-comment">/**
 * 暴力解法 - O(n^2)
 * <span class="hljs-doctag">@param</span> {<span class="hljs-type">number[]</span>} nums 整数数组
 * <span class="hljs-doctag">@param</span> {<span class="hljs-type">number</span>} target 目标值
 * <span class="hljs-doctag">@returns</span> {<span class="hljs-type">number[]</span>} 两个数的下标
 */</span>
<span class="hljs-keyword">function</span> <span class="hljs-title function_">twoSum_bruteForce</span>(<span class="hljs-params">nums, target</span>) {
  <span class="hljs-comment">// 外层循环，从第一个元素开始</span>
  <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> i = <span class="hljs-number">0</span>; i &#x3C; nums.<span class="hljs-property">length</span>; i++) {
    <span class="hljs-comment">// 内层循环，从外层循环的下一个元素开始，避免重复和对自己求和</span>
    <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> j = i + <span class="hljs-number">1</span>; j &#x3C; nums.<span class="hljs-property">length</span>; j++) {
      <span class="hljs-comment">// 检查两个数的和是否等于目标值</span>
      <span class="hljs-keyword">if</span> (nums[i] + nums[j] === target) {
        <span class="hljs-keyword">return</span> [i, j]; <span class="hljs-comment">// 找到后立即返回</span>
      }
    }
  }
}
</code></pre>
<p><strong>解法2：哈希表 / Map（最优解）</strong></p>
<p>面试官期待的答案。我们可以通过空间换时间，将查找效率从 O(n) 降低到 O(1)。</p>
<ul>
<li>
<p><strong>思路</strong>: 遍历数组，对于每个元素 <code>num</code>，我们不再是向后查找，而是去一个“登记处”（哈希表）查询我们“需要”的那个数字 (<code>target - num</code>) 是否已经出现过。</p>
<ol>
<li>
<p>创建一个 Map（或普通对象）来存储已经遍历过的数字及其下标。</p>
</li>
<li>
<p>遍历 <code>nums</code> 数组。</p>
</li>
<li>
<p>在遍历到 <code>nums[i]</code> 时，计算出我们需要的“另一半”：<code>complement = target - nums[i]</code>。</p>
</li>
<li>
<p>在 Map 中查找是否存在 <code>complement</code>。</p>
</li>
<li>
<p>如果存在，说明我们找到了配对，直接返回 <code>map.get(complement)</code> 和当前的下标 <code>i</code>。</p>
</li>
<li>
<p>如果不存在，就把当前的 <code>nums[i]</code> 和它的下标 <code>i</code> 存入 Map 中，供后续的数字进行查询。</p>
</li>
</ol>
</li>
<li>
<p><strong>时间复杂度</strong>: O(n)。我们只需要遍历数组一次。Map 的插入和查找操作平均时间复杂度都是 O(1)。</p>
</li>
<li>
<p><strong>空间复杂度</strong>: O(n)。在最坏的情况下，我们需要把所有元素都存入 Map 中。</p>
</li>
</ul>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-comment">/**
 * 哈希表解法 - O(n)
 * <span class="hljs-doctag">@param</span> {<span class="hljs-type">number[]</span>} nums 整数数组
 * <span class="hljs-doctag">@param</span> {<span class="hljs-type">number</span>} target 目标值
 * <span class="hljs-doctag">@returns</span> {<span class="hljs-type">number[]</span>} 两个数的下标
 */</span>
<span class="hljs-keyword">function</span> <span class="hljs-title function_">twoSum</span>(<span class="hljs-params">nums, target</span>) {
  <span class="hljs-comment">// 创建一个 Map，用于存储 { 数字: 下标 } 的键值对</span>
  <span class="hljs-keyword">const</span> map = <span class="hljs-keyword">new</span> <span class="hljs-title class_">Map</span>();

  <span class="hljs-comment">// 仅需一次遍历</span>
  <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> i = <span class="hljs-number">0</span>; i &#x3C; nums.<span class="hljs-property">length</span>; i++) {
    <span class="hljs-keyword">const</span> currentNum = nums[i];
    <span class="hljs-comment">// 计算我们需要寻找的“另一半”</span>
    <span class="hljs-keyword">const</span> complement = target - currentNum;

    <span class="hljs-comment">// 检查 Map 中是否已经存在“另一半”</span>
    <span class="hljs-keyword">if</span> (map.<span class="hljs-title function_">has</span>(complement)) {
      <span class="hljs-comment">// 如果存在，说明找到了配对</span>
      <span class="hljs-comment">// complement 的下标已经存在 map 中，currentNum 的下标就是当前的 i</span>
      <span class="hljs-keyword">return</span> [map.<span class="hljs-title function_">get</span>(complement), i];
    }

    <span class="hljs-comment">// 如果没找到，就把当前数字和它的下标存入 Map</span>
    <span class="hljs-comment">// 供后续的元素进行配对查询</span>
    map.<span class="hljs-title function_">set</span>(currentNum, i);
  }

  <span class="hljs-comment">// 如果遍历完都没有找到，可以根据题目要求返回空数组或抛出错误</span>
  <span class="hljs-comment">// (题目假设总会有一个解，所以这里理论上不会执行到)</span>
}

<span class="hljs-comment">// --- 测试 ---</span>
<span class="hljs-keyword">const</span> nums = [<span class="hljs-number">2</span>, <span class="hljs-number">7</span>, <span class="hljs-number">11</span>, <span class="hljs-number">15</span>];
<span class="hljs-keyword">const</span> target = <span class="hljs-number">9</span>;
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">`最优解找到的下标: [<span class="hljs-subst">${twoSum(nums, target)}</span>]`</span>); <span class="hljs-comment">// 输出: 最优解找到的下标: [0,1]</span>
</code></pre>
<p><strong>彻底讲明白</strong>: 核心思想是<strong>用哈希表将“查找”这个耗时的操作变成近乎瞬时的操作</strong>。当我们遍历到第 <code>i</code> 个元素时，我们不再需要向后遍历 <code>n-i</code> 次去寻找配对，而是直接问哈希表：“嘿，我需要的那个数你见过吗？” 这个“问”的动作，时间成本是 O(1)。这就是典型的“空间换时间”策略，在前端性能优化中非常常见（例如，缓存计算结果）。</p>

<h3 id="类别二树与-dom-tree--dom---前端核心战场"><a href="#类别二树与-dom-tree--dom---前端核心战场">类别二：树与 DOM (Tree &#x26; DOM) - 前端核心战场</a></h3>
<p>DOM 本身就是一棵树，React/Vue 的组件结构也是一棵树。因此，树的遍历和操作是前端工程师的必备技能。</p>
<h4 id="经典题目树的深度优先遍历-dfs-与广度优先遍历-bfs"><a href="#经典题目树的深度优先遍历-dfs-与广度优先遍历-bfs">经典题目：树的深度优先遍历 (DFS) 与广度优先遍历 (BFS)</a></h4>
<p>题目描述</p>
<p>给定一个树状结构（比如一个嵌套的对象，或者真实的 DOM 树），实现两种遍历方式：</p>
<ol>
<li>
<p><strong>深度优先遍历 (DFS, Depth-First Search)</strong>: 尽可能深地搜索树的分支。</p>
</li>
<li>
<p><strong>广度优先遍历 (BFS, Breadth-First Search)</strong>: 又叫层次遍历，从上到下，从左到右逐层访问节点。</p>
</li>
</ol>
<p><strong>为什么这道题对前端如此重要？</strong></p>
<ul>
<li>
<p><strong>DOM 操作</strong>: <code>querySelectorAll</code> 的底层实现就类似于树的遍历。当你需要寻找页面上所有 <code>class</code> 为 <code>highlight</code> 的 <code>div</code> 元素时，就是在进行树的遍历和筛选。</p>
</li>
<li>
<p><strong>组件通信</strong>: 在 React 或 Vue 中，有时需要从父组件向深层子组件传递信息（Context API），或者反向通信，这都涉及到组件树的遍历逻辑。</p>
</li>
<li>
<p><strong>数据处理</strong>: 从 API 获取的 JSON 数据经常是嵌套的树状结构，你需要遍历这棵树来提取或转换数据。</p>
</li>
</ul>
<p><strong>代码详解</strong></p>
<p>我们用一个简单的对象来模拟树结构：</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">const</span> tree = {
  <span class="hljs-attr">value</span>: <span class="hljs-string">'A'</span>,
  <span class="hljs-attr">children</span>: [
    {
      <span class="hljs-attr">value</span>: <span class="hljs-string">'B'</span>,
      <span class="hljs-attr">children</span>: [
        { <span class="hljs-attr">value</span>: <span class="hljs-string">'D'</span>, <span class="hljs-attr">children</span>: [] },
        { <span class="hljs-attr">value</span>: <span class="hljs-string">'E'</span>, <span class="hljs-attr">children</span>: [] }
      ]
    },
    {
      <span class="hljs-attr">value</span>: <span class="hljs-string">'C'</span>,
      <span class="hljs-attr">children</span>: [
        { <span class="hljs-attr">value</span>: <span class="hljs-string">'F'</span>, <span class="hljs-attr">children</span>: [] }
      ]
    }
  ]
};
</code></pre>
<p><strong>解法1：深度优先遍历 (DFS) - 递归实现</strong></p>
<p>递归是实现 DFS 最自然、最简洁的方式。</p>
<ul>
<li>
<p><strong>思路</strong>:</p>
<ol>
<li>
<p>定义一个函数 <code>dfs(node)</code>，接收一个节点作为参数。</p>
</li>
<li>
<p>首先处理当前节点（比如，打印它的值）。</p>
</li>
<li>
<p>然后遍历当前节点的 <code>children</code> 数组。</p>
</li>
<li>
<p>对每个 <code>child</code>，递归调用 <code>dfs(child)</code>。</p>
</li>
</ol>
</li>
<li>
<p><strong>特点</strong>: 实现简单，代码易读。但如果树的层级非常深，可能导致调用栈溢出（Stack Overflow）。</p>
</li>
</ul>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-comment">/**
 * 深度优先遍历 (DFS) - 递归实现
 * <span class="hljs-doctag">@param</span> {<span class="hljs-type">object</span>} node 树节点
 * <span class="hljs-doctag">@param</span> {<span class="hljs-type">Array</span>} result 存储遍历结果的数组
 */</span>
<span class="hljs-keyword">function</span> <span class="hljs-title function_">dfs</span>(<span class="hljs-params">node, result = []</span>) {
  <span class="hljs-keyword">if</span> (!node) <span class="hljs-keyword">return</span>; <span class="hljs-comment">// 递归的基线条件</span>

  <span class="hljs-comment">// 1. 处理当前节点</span>
  result.<span class="hljs-title function_">push</span>(node.<span class="hljs-property">value</span>);
  <span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">`DFS访问: <span class="hljs-subst">${node.value}</span>`</span>);

  <span class="hljs-comment">// 2. 递归遍历子节点</span>
  <span class="hljs-keyword">if</span> (node.<span class="hljs-property">children</span> &#x26;&#x26; node.<span class="hljs-property">children</span>.<span class="hljs-property">length</span> > <span class="hljs-number">0</span>) {
    <span class="hljs-keyword">for</span> (<span class="hljs-keyword">const</span> child <span class="hljs-keyword">of</span> node.<span class="hljs-property">children</span>) {
      <span class="hljs-title function_">dfs</span>(child, result); <span class="hljs-comment">// 对每个子节点递归调用</span>
    }
  }

  <span class="hljs-keyword">return</span> result;
}

<span class="hljs-comment">// --- 测试 DFS ---</span>
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">'--- 开始深度优先遍历 ---'</span>);
<span class="hljs-keyword">const</span> dfsResult = <span class="hljs-title function_">dfs</span>(tree);
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">`DFS 最终结果: [<span class="hljs-subst">${dfsResult.join(<span class="hljs-string">', '</span>)}</span>]`</span>); <span class="hljs-comment">// A, B, D, E, C, F</span>
</code></pre>
<p><strong>解法2：广度优先遍历 (BFS) - 队列实现</strong></p>
<p>BFS 不能简单地用递归实现，它需要一个辅助的数据结构——队列（Queue）。</p>
<ul>
<li>
<p><strong>思路</strong>:</p>
<ol>
<li>
<p>创建一个队列，并把根节点放进去。</p>
</li>
<li>
<p>当队列不为空时，执行循环：</p>
<p>a. 从队列头部取出一个节点 currentNode。</p>
<p>b. 处理 currentNode（比如，打印它的值）。</p>
<p>c. 将 currentNode 的所有子节点依次加入队列的尾部。</p>
</li>
<li>
<p>重复步骤 2，直到队列为空。</p>
</li>
</ol>
</li>
<li>
<p><strong>特点</strong>: 能保证按层级顺序访问。实现上比递归 DFS 稍复杂，但不会有栈溢出的风险。可以用来找最短路径。</p>
</li>
</ul>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-comment">/**
 * 广度优先遍历 (BFS) - 队列实现
 * <span class="hljs-doctag">@param</span> {<span class="hljs-type">object</span>} node 树节点
 * <span class="hljs-doctag">@returns</span> {<span class="hljs-type">Array</span>} 遍历结果数组
 */</span>
<span class="hljs-keyword">function</span> <span class="hljs-title function_">bfs</span>(<span class="hljs-params">node</span>) {
  <span class="hljs-keyword">if</span> (!node) <span class="hljs-keyword">return</span> [];

  <span class="hljs-keyword">const</span> queue = [node]; <span class="hljs-comment">// 1. 创建队列，放入根节点</span>
  <span class="hljs-keyword">const</span> result = [];

  <span class="hljs-comment">// 2. 当队列不为空时循环</span>
  <span class="hljs-keyword">while</span> (queue.<span class="hljs-property">length</span> > <span class="hljs-number">0</span>) {
    <span class="hljs-comment">// a. 从队首取出一个节点</span>
    <span class="hljs-keyword">const</span> currentNode = queue.<span class="hljs-title function_">shift</span>(); <span class="hljs-comment">// shift() 方法模拟出队</span>

    <span class="hljs-comment">// b. 处理当前节点</span>
    result.<span class="hljs-title function_">push</span>(currentNode.<span class="hljs-property">value</span>);
    <span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">`BFS访问: <span class="hljs-subst">${currentNode.value}</span>`</span>);

    <span class="hljs-comment">// c. 将其所有子节点入队</span>
    <span class="hljs-keyword">if</span> (currentNode.<span class="hljs-property">children</span> &#x26;&#x26; currentNode.<span class="hljs-property">children</span>.<span class="hljs-property">length</span> > <span class="hljs-number">0</span>) {
      <span class="hljs-keyword">for</span> (<span class="hljs-keyword">const</span> child <span class="hljs-keyword">of</span> currentNode.<span class="hljs-property">children</span>) {
        queue.<span class="hljs-title function_">push</span>(child); <span class="hljs-comment">// push() 方法模拟入队</span>
      }
    }
  }

  <span class="hljs-keyword">return</span> result;
}

<p><strong>彻底讲明白</strong>:</p>
<ul>
<li>
<p><strong>DFS</strong> 的精髓是“一条路走到黑，走不通再回头”。它的数据结构是<strong>栈 (Stack)</strong>。对于递归实现，这个栈就是函数的<strong>调用栈</strong>。你也可以用一个真实的栈来迭代实现 DFS。</p>
</li>
<li>
<p><strong>BFS</strong> 的精髓是“一层一层地毯式搜索”。它的数据结构是<strong>队列 (Queue)</strong>。它能保证最先被访问的节点的邻居也一定比后被访问的节点的邻居要早被访问。这使得 BFS 成为在无权图中寻找最短路径的不二之选（例如，找到从A节点到B节点最少需要经过几层）。</p>
</li>
</ul>

<h3 id="类别三异步编程-asynchronous-programming---前端命脉"><a href="#类别三异步编程-asynchronous-programming---前端命脉">类别三：异步编程 (Asynchronous Programming) - 前端命脉</a></h3>
<p>JavaScript 是单线程的，但浏览器环境是事件驱动的。处理网络请求、定时器、用户交互等异步任务是前端的日常。</p>
<h4 id="经典题目实现-promiseall"><a href="#经典题目实现-promiseall">经典题目：实现 Promise.all</a></h4>
<p>题目描述</p>
<p>Promise.all(promises) 方法接收一个 promise 的可迭代对象（比如数组），并返回一个单独的 Promise。这个返回的 Promise 会在所有输入的 promise 都兑现（fulfilled）后兑现，其值为一个包含所有兑现值的数组。如果输入的任何一个 promise 被拒绝（rejected），则返回的 Promise 会立即被拒绝，并带有第一个被拒绝的 promise 的原因。</p>
<p>请你实现一个功能类似的 <code>myPromiseAll</code> 函数。</p>
<p><strong>为什么这道题对前端如此重要？</strong></p>
<ul>
<li>
<p><strong>并发请求</strong>: 页面初始化时，我们经常需要同时请求多个接口（比如用户信息、商品列表、配置信息），并在所有数据都返回后再渲染页面。<code>Promise.all</code> 是实现这一需求的核心工具。</p>
</li>
<li>
<p><strong>理解 Promise</strong>: 能否写出 <code>Promise.all</code> 是检验面试者是否真正理解 Promise 状态、执行流程和错误处理的“试金石”。</p>
</li>
</ul>
<p><strong>代码详解</strong></p>
<ul>
<li>
<p><strong>核心思路</strong>:</p>
<ol>
<li>
<p><code>myPromiseAll</code> 函数返回一个新的 <code>Promise</code>。</p>
</li>
<li>
<p>它需要一个计数器 <code>count</code> 来记录已完成的 promise 数量，以及一个 <code>results</code> 数组来按顺序存储结果。</p>
</li>
<li>
<p>遍历输入的 <code>promises</code> 数组。对于每个 <code>promise</code>：</p>
<ul>
<li>
<p>使用 <code>.then()</code> 来处理它。</p>
</li>
<li>
<p>当 <code>promise</code> 成功时，将结果存入 <code>results</code> 数组的<strong>对应位置</strong>（而不是 <code>push</code>，因为 promise 完成顺序不确定），并将计数器 <code>count</code> 加一。</p>
</li>
<li>
<p>检查 <code>count</code> 是否等于 <code>promises</code> 的总数。如果是，说明所有 promise 都已完成，此时 <code>resolve</code> 新 <code>Promise</code>，并传入 <code>results</code> 数组。</p>
</li>
</ul>
</li>
<li>
<p>如果遍历过程中的任何一个 <code>promise</code> 失败了（进入 <code>.catch</code> 或 <code>.then</code> 的第二个参数），则立即 <code>reject</code> 新 <code>Promise</code>，并把这个错误原因传出去。</p>
</li>
</ol>
</li>
</ul>

<ul>
<li>
<p><strong>并发</strong>: <code>forEach</code> 循环是同步的，它会立即启动所有 promise 的执行，这就实现了并发。</p>
</li>
<li>

</li>
<li>

</li>
</ul>

<h3 id="类别四性能与优化-performance--optimization---体现工程能力"><a href="#类别四性能与优化-performance--optimization---体现工程能力">类别四：性能与优化 (Performance &#x26; Optimization) - 体现工程能力</a></h3>
<p>一个功能跑得起来和跑得好是两回事。性能优化相关的算法题能直接体现面试者的工程素养。</p>
<h4 id="经典题目实现防抖-debounce-和节流-throttle"><a href="#经典题目实现防抖-debounce-和节流-throttle">经典题目：实现防抖 (Debounce) 和节流 (Throttle)</a></h4>
<p>题目描述</p>
<p>编写两个高阶函数：debounce 和 throttle。</p>
<ul>
<li>
<p><strong>防抖 (Debounce)</strong>: 在事件被触发 n 秒后再执行回调，如果在这 n 秒内又被触发，则重新计时。</p>
</li>
<li>
<p><strong>节流 (Throttle)</strong>: 规定在一个单位时间内，只能触发一次函数。如果这个单位时间内触发多次函数，只有一次生效。</p>
</li>
</ul>
<p>为什么这道题对前端如此重要？</p>
<p>这是前端性能优化的“标配”技能。</p>
<ul>
<li>
<p><strong>防抖应用</strong>:</p>
<ul>
<li>
<p>搜索框输入建议：用户停止输入一小段时间后再发请求，而不是每输入一个字母都发请求。</p>
</li>
<li>
<p>窗口 <code>resize</code> 事件：只在用户调整完窗口大小后才重新计算布局。</p>
</li>
</ul>
</li>
<li>
<p><strong>节流应用</strong>:</p>
<ul>
<li>
<p><code>scroll</code> 事件：监听页面滚动，但每隔 200ms 才触发一次回调（比如计算元素是否进入视口）。</p>
</li>
<li>
<p>鼠标 <code>mousemove</code> 事件：实现拖拽功能时，不需要对每 1px 的移动都做响应。</p>
</li>
</ul>
</li>
</ul>
<p><strong>代码详解</strong></p>
<p><strong>实现防抖 (Debounce)</strong></p>
<ul>
<li><strong>思路</strong>: 利用定时器 <code>setTimeout</code>。每次触发事件时，都先清除上一个定时器，然后设置一个新的定时器。只有当最后一次触发后，定时器能够完整地执行完，回调函数才会被调用。</li>
</ul>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-comment">/**

<span class="hljs-keyword">let</span> scrollCount = <span class="hljs-number">0</span>;
<span class="hljs-keyword">const</span> <span class="hljs-title function_">handleScroll</span> = (<span class="hljs-params"></span>) => {
  scrollCount++;

};

<span class="hljs-keyword">const</span> throttledScrollHandler = <span class="hljs-title function_">throttle</span>(handleScroll, <span class="hljs-number">1000</span>);

<span class="hljs-comment">// 模拟高频滚动</span>
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">"\n--- 测试节流 ---"</span>);
<span class="hljs-variable language_">console</span>.<span class="hljs-title function_">log</span>(<span class="hljs-string">"模拟2秒内高频触发滚动..."</span>);

</code></pre>
<p><strong>彻底讲明白</strong>:</p>
<ul>
<li>
<p><strong>防抖</strong>的核心是<strong>等待</strong>和<strong>重置</strong>。它关心的是“最后一次”操作。就像电梯关门，只要有人按开门按钮，关门计时就重置。</p>
</li>
<li>
<p><strong>节流</strong>的核心是<strong>锁定</strong>和<strong>解锁</strong>。它关心的是“固定频率”。就像技能冷却，放了一次技能后，必须等 CD 转好才能放第二次。</p>
</li>
<li>
<p>选择哪个取决于业务需求：如果你只关心最终结果（比如用户输完的最终文本），用防抖；如果你需要以固定频率对过程进行响应（比如滚动时的动画），用节流。</p>
</li>
</ul>

<h3 id="类别五综合应用---virtual-dom-diff"><a href="#类别五综合应用---virtual-dom-diff">类别五：综合应用 - Virtual DOM Diff</a></h3>
<p>这是前几类知识的集大成者，也是现代前端框架的灵魂。能讲明白 Diff 算法，说明你对前端的渲染机制有非常深刻的理解。</p>
<h4 id="经典题目简化版的-virtual-dom-diff-算法"><a href="#经典题目简化版的-virtual-dom-diff-算法">经典题目：简化版的 Virtual DOM Diff 算法</a></h4>
<p>题目描述</p>
<p>请描述 Virtual DOM 的 Diff 算法的核心思想，并用代码模拟一个简化版的 patch(oldVNode, newVNode) 函数，该函数能够找出两个虚拟节点树的差异，并生成一个描述这些差异的“补丁”对象。</p>
<p><strong>简化假设</strong>:</p>
<ol>
<li>
<p>只考虑节点类型（<code>tag</code>）不同和文本内容（<code>text</code>）不同的情况。</p>
</li>
<li>
<p>不考虑 <code>props</code> 的 diff。</p>
</li>
<li>
<p>不考虑复杂的子节点移动，只考虑子节点列表的同级替换、删除和新增。</p>
</li>
</ol>
<p><strong>为什么这道题对前端如此重要？</strong></p>
<ul>
<li>
<p><strong>框架基石</strong>: 这是 React、Vue 等框架实现高性能渲染的核心。理解它，就等于理解了为什么我们不再需要手动操作繁琐且昂贵的 DOM。</p>
</li>
<li>
<p><strong>抽象思维</strong>: VDOM 和 Diff 算法是前端领域一个非常成功的抽象设计，它将 UI 描述（VNode）和 UI 实现（DOM）解耦，考察的是工程师的抽象和设计能力。</p>
</li>
</ul>
<p><strong>思路与代码详解</strong></p>
<p><strong>核心思想</strong></p>
<ol>
<li>
<p><strong>同级比较</strong>: Diff 算法是逐层比较的，不会跨层级移动节点。如果 <code>div</code> 下的 <code>p</code> 变成了 <code>section</code> 下的 <code>p</code>，它会被当做“删除旧 <code>p</code>”和“新增新 <code>p</code>”来处理。</p>
</li>
<li>
<p><strong>类型判断</strong>:</p>
<ul>
<li>
<p>如果新旧节点类型不同（比如 <code>div</code> 变成了 <code>p</code>），直接替换整个旧节点及其子树，不做后续比较。</p>
</li>
<li>
<p>如果都是文本节点，比较文本内容是否变化。</p>
</li>
</ul>
</li>
<li>
<p><strong>子节点比较 (Key 的重要性)</strong>:</p>
<ul>
<li>
<p>这是最复杂的部分。当父节点相同时，需要比较它们的子节点列表。</p>
</li>
<li>
<p>为了高效地识别出哪些子节点是新增、删除或移动的，现代框架引入了 <code>key</code>。通过 <code>key</code>，我们可以快速匹配新旧列表中的同一个节点，从而避免不必要的重建，只需移动即可。</p>
</li>
<li>
<p>在我们的简化版中，我们先不实现复杂的 key 匹配，只做一个简单的对比。</p>
</li>
</ul>
</li>
</ol>

<p>彻底讲明白:</p>
<p>这个简化版的 diff 函数递归地遍历了新旧两棵 VDOM 树。</p>
<ol>
<li>
<p>它返回一个 <code>patches</code> 对象，这个对象的结构<strong>模仿了原 VDOM 树的结构</strong>。<code>patches</code> 对象只在有差异的地方记录差异信息。</p>
</li>
<li>
<p>在真实的 <code>patch</code> 过程中，会有一个独立的函数（比如 <code>applyPatches(domRoot, patches)</code>），它会拿着这个 <code>patches</code> 对象去对真实的 DOM 进行<strong>最小化</strong>的操作。例如，看到 <code>type: 'REPLACE'</code>，它就知道调用 <code>parentElement.replaceChild()</code>；看到 <code>type: 'TEXT'</code>，它就知道修改 <code>node.textContent</code>。</p>
</li>
<li>
<p><strong>关键点</strong>: <code>diff</code> 过程本身<strong>不操作 DOM</strong>，它只是一个纯粹的 JavaScript 对象比较过程，因此速度非常快。昂贵的操作被推迟到最后一步，并且只执行必要的部分。这就是 VDOM 提升性能的秘密。在面试中，能把这一点（计算与执行分离）讲清楚，会非常加分。</p>
</li>
</ol>
