    <html><head></head><body><h2>一、概述</h2>
<p>WordPress 是网络上最流行的开源博客系统和 CMS。它基于 PHP 和 MySQL。它的功能可以通过数以千计的免费插件和主题进行扩展。 在本教程中，我们将在 Apache2 服务器上安装 WordPress 并创建我们的第一篇文章。</p>
<h2>二、安装依赖</h2>
<p>要安装 PHP 和 Apache，使用以下命令：</p>
<pre><code>sudo apt update
sudo apt install apache2 <span class="hljs-string">\</span>
                 ghostscript <span class="hljs-string">\</span>
                 libapache2-mod-php <span class="hljs-string">\</span>
                 mysql-server <span class="hljs-string">\</span>
                 php <span class="hljs-string">\</span>
                 php-bcmath <span class="hljs-string">\</span>
                 php-curl <span class="hljs-string">\</span>
                 php-imagick <span class="hljs-string">\</span>
                 php-intl <span class="hljs-string">\</span>
                 php-json <span class="hljs-string">\</span>
                 php-mbstring <span class="hljs-string">\</span>
                 php-mysql <span class="hljs-string">\</span>
                 php-xml <span class="hljs-string">\</span>
                 php-zip
</code></pre>
<p>&nbsp; &nbsp;</p>
<h2>三、安装WordPress</h2>
<p>我们将使用来自<a href="http://wordpress.org/">WordPress.org</a>的版本，而不是 Ubuntu 存档中的 APT 包，因为这是上游 WordPress 的首选方法。这也将减少 WordPress 支持志愿者无法预料并因此无法提供帮助的“陷阱”问题。 创建安装目录并从<a href="http://wordpress.org/">WordPress.org</a>下载文件： &nbsp;</p>
<pre><code>sudo mkdir -p <span class="hljs-regexp">/srv/</span>www
sudo chown www-data: <span class="hljs-regexp">/srv/</span>www
curl https:<span class="hljs-regexp">//</span>wordpress.org<span class="hljs-regexp">/latest.tar.gz  sudo -u www-data tar zx -C /</span>srv/www
</code></pre>
<p>&nbsp;</p>
<p>请注意，这会将所有权设置为 user www-data，这可能是不安全的，例如当您的服务器托管具有不同维护者的多个站点时。您应该调查在这种情况下使用每个网站的用户，并使文件仅对这些用户可读和可写。这将需要配置 PHP-FPM 为每个站点启动一个单独的实例，每个实例都作为站点的用户帐户运行。在这样的设置中，wp-config.php应该（阅读：如果你做的不同，你需要一个很好的理由）对站点所有者和组是只读的，并且其他权限设置为无访问权限（chmod 400）。但是，这超出了本指南的范围。</p>
<h2>四、为 WordPress 配置 Apache</h2>
<p>为 WordPress 创建 Apache 站点。使用命令<code>vim <span class="hljs-regexp">/etc/</span>apache2<span class="hljs-regexp">/sites-available/</span>wordpress.conf</code>&nbsp;键入以下内容：</p>
<pre><code><span class="hljs-section">&lt;VirtualHost *<span class="hljs-number">:80</span>&gt;</span>
    <span class="hljs-attribute">DocumentRoot</span> /srv/www/wordpress
    <span class="hljs-section">&lt;Directory /srv/www/wordpress&gt;</span>
        <span class="hljs-attribute">Options</span> FollowSymLinks
        <span class="hljs-attribute">AllowOverride</span> Limit Options FileInfo
        <span class="hljs-attribute">DirectoryIndex</span> index.php
        <span class="hljs-attribute">Require</span> <span class="hljs-literal">all</span> granted
    <span class="hljs-section">&lt;/Directory&gt;</span>
    <span class="hljs-section">&lt;Directory /srv/www/wordpress/wp-content&gt;</span>
        <span class="hljs-attribute">Options</span> FollowSymLinks
        <span class="hljs-attribute">Require</span> <span class="hljs-literal">all</span> granted
    <span class="hljs-section">&lt;/Directory&gt;</span>
<span class="hljs-section">&lt;/VirtualHost&gt;</span>
</code></pre>
<p>启用网站：<code><span class="hljs-attribute">sudo</span> a2enmod wordpress</code>&nbsp; &nbsp; 启用 URL 重写：<code>sudo a2ensite <span class="hljs-built_in">rewrite</span></code>&nbsp; (使用以下命令禁用默认的“It Works”站点：<code>sudo a2dissite <span class="hljs-number">000</span>-<span class="hljs-keyword">default</span></code> ) 或者，您可以编辑我们的配置文件以添加 WordPress 安装将响应请求的主机名，而不是禁用“它可以工作”页面。这个主机名必须以某种方式映射到您的机器上，例如通过 DNS，或编辑客户端系统的<code><span class="hljs-regexp">/etc/</span>hosts</code>文件（在 Windows 上相当于<code><span class="hljs-name">C</span>:\Windows\System32\drivers\etc\hosts</code>）。添加<code><span class="hljs-attribute">ServerName</span></code>如下：</p>
<pre><code><span class="hljs-variable">&lt;VirtualHost *:80&gt;</span>
    ServerName hostname.example.com
    ... <span class="hljs-comment"># the rest of the VHost configuration</span>
<span class="hljs-variable">&lt;/VirtualHost&gt;</span>
</code></pre>
<p>最后，重新加载 apache2 以应用所有这些更改：<code>sudo service apache2 <span class="hljs-keyword">reload</span></code></p>

<h2>六、配置WordPress连接数据库</h2>
<p>现在，让我们配置 WordPress 以使用该数据库。首先，将示例配置文件复制到<code>wp-config.php</code>： <code>sudo -u www-data cp <span class="hljs-regexp">/srv/</span>www<span class="hljs-regexp">/wordpress/</span>wp-config-sample.php <span class="hljs-regexp">/srv/</span>www<span class="hljs-regexp">/wordpress/</span>wp-config.php</code> 接下来，在配置文件中设置数据库凭据（不要替换<code><span class="hljs-attribute">database_name_here</span></code>或<code><span class="hljs-attribute">username_here</span></code>&nbsp;在下面的命令中。_请_替换<code><span class="hljs-section">&lt;your-password&gt;</span></code>为您的数据库密码。）： <code>sudo -u www-data sed -i <span class="hljs-string">'s/database_name_here/wordpress/'</span> <span class="hljs-regexp">/srv/</span>www<span class="hljs-regexp">/wordpress/</span>wp-config.php sudo -u www-data sed -i <span class="hljs-string">'s/username_here/wordpress/'</span> <span class="hljs-regexp">/srv/</span>www<span class="hljs-regexp">/wordpress/</span>wp-config.php sudo -u www-data sed -i <span class="hljs-string">'s/password_here/&lt;your-password&gt;/'</span> <span class="hljs-regexp">/srv/</span>www<span class="hljs-regexp">/wordpress/</span>wp-config.php</code> 最后，在终端会话中打开 nano 中的配置文件： <code>sudo -u www-data nano <span class="hljs-regexp">/srv/</span>www<span class="hljs-regexp">/wordpress/</span>wp-config.php</code> 找到以下内容：</p>
<pre><code>define( 'AUTH_KEY', 'put your unique phrase here' )<span class="hljs-comment">; </span>
define( 'SECURE_AUTH_KEY', 'put your unique phrase here' )<span class="hljs-comment">; </span>

<p>删除这些行（ctrl+k将在您每次按序列时删除一行）。然后替换为<a href="https://api.wordpress.org/secret-key/1.1/salt/">https://api.wordpress.org/secret-key/1.1/salt/</a>的内容。（此地址是一个随机生成器，每次打开时都会返回完全随机的密钥。）此步骤对于确保您的站点不易受到“已知秘密”攻击非常重要。 <code>ctrl+<span class="hljs-keyword">x</span></code>&nbsp;保存并关闭配置文件，然后输入<code><span class="hljs-attribute">y enter</span></code>&nbsp;</p>
<h2>七、配置WordPress</h2>
<p>在浏览器中打开<a href="http://localhost/">http://localhost/ 。</a>您将被要求提供新站点的名称、用户名、密码和电子邮件地址。请注意，您在此处选择的用户名和密码用于 WordPress，并且不提供对服务器任何其他部分的访问权限 - 选择与我们为 WordPress 使用而配置的 MySQL（数据库）凭据不同的用户名和密码，并且与您登录计算机或服务器的桌面或外壳的凭据不同。您可以选择是否要让您的网站被搜索引擎索引。</p>
