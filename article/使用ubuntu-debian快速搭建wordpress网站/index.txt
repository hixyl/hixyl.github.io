1d:T2f55,<h2 id="一概述"><a href="#一概述">一、概述</a></h2>
<p>WordPress 是网络上最流行的开源博客系统和 CMS。它基于 PHP 和 MySQL。它的功能可以通过数以千计的免费插件和主题进行扩展。 在本教程中，我们将在 Apache2 服务器上安装 WordPress 并创建我们的第一篇文章。</p>
<h2 id="二安装依赖"><a href="#二安装依赖">二、安装依赖</a></h2>
<p>要安装 PHP 和 Apache，使用以下命令：</p>
<pre><code class="hljs language-bash"><span class="hljs-built_in">sudo</span> apt update
<span class="hljs-built_in">sudo</span> apt install apache2 \
                 ghostscript \
                 libapache2-mod-php \
                 mysql-server \
                 php \
                 php-bcmath \
                 php-curl \
                 php-imagick \
                 php-intl \
                 php-json \
                 php-mbstring \
                 php-mysql \
                 php-xml \
                 php-zip
</code></pre>
<p>   </p>
<h2 id="三安装wordpress"><a href="#三安装wordpress">三、安装WordPress</a></h2>
<p>我们将使用来自<a href="http://wordpress.org/">WordPress.org</a>的版本，而不是 Ubuntu 存档中的 APT 包，因为这是上游 WordPress 的首选方法。这也将减少 WordPress 支持志愿者无法预料并因此无法提供帮助的“陷阱”问题。 创建安装目录并从<a href="http://wordpress.org/">WordPress.org</a>下载文件：  </p>
<pre><code class="hljs language-bash"><span class="hljs-built_in">sudo</span> <span class="hljs-built_in">mkdir</span> -p /srv/www
<span class="hljs-built_in">sudo</span> <span class="hljs-built_in">chown</span> www-data: /srv/www
curl https://wordpress.org/latest.tar.gz  <span class="hljs-built_in">sudo</span> -u www-data tar zx -C /srv/www
</code></pre>
<p> </p>
<p>请注意，这会将所有权设置为 user www-data，这可能是不安全的，例如当您的服务器托管具有不同维护者的多个站点时。您应该调查在这种情况下使用每个网站的用户，并使文件仅对这些用户可读和可写。这将需要配置 PHP-FPM 为每个站点启动一个单独的实例，每个实例都作为站点的用户帐户运行。在这样的设置中，wp-config.php应该（阅读：如果你做的不同，你需要一个很好的理由）对站点所有者和组是只读的，并且其他权限设置为无访问权限（chmod 400）。但是，这超出了本指南的范围。</p>
<h2 id="四为-wordpress-配置-apache"><a href="#四为-wordpress-配置-apache">四、为 WordPress 配置 Apache</a></h2>
<p>为 WordPress 创建 Apache 站点。使用命令<code>vim /etc/apache2/sites-available/wordpress.conf</code> 键入以下内容：</p>

    <span class="hljs-tag">&#x3C;/<span class="hljs-name">Directory</span>></span>

    <span class="hljs-tag">&#x3C;/<span class="hljs-name">Directory</span>></span>

<p>启用网站：<code>sudo a2enmod wordpress</code>    启用 URL 重写：<code>sudo a2ensite rewrite</code>  (使用以下命令禁用默认的“It Works”站点：<code>sudo a2dissite 000-default</code> ) 或者，您可以编辑我们的配置文件以添加 WordPress 安装将响应请求的主机名，而不是禁用“它可以工作”页面。这个主机名必须以某种方式映射到您的机器上，例如通过 DNS，或编辑客户端系统的<code>/etc/hosts</code>文件（在 Windows 上相当于<code>C:\Windows\System32\drivers\etc\hosts</code>）。添加<code>ServerName</code>如下：</p>
<pre><code class="hljs language-php-template"><span class="xml"><span class="hljs-tag">&#x3C;<span class="hljs-name">VirtualHost</span> *<span class="hljs-attr">:80</span>></span>

    ... # the rest of the VHost configuration
<span class="hljs-tag">&#x3C;/<span class="hljs-name">VirtualHost</span>></span>

<p>最后，重新加载 apache2 以应用所有这些更改：<code>sudo service apache2 reload</code></p>
