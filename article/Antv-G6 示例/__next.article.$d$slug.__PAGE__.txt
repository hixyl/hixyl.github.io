<h3 id="第三部分完整解释说明"><a href="#第三部分完整解释说明">第三部分：完整解释说明</a></h3>
<h4 id="1-初始化机制依赖注入的桥梁"><a href="#1-初始化机制依赖注入的桥梁">1. 初始化机制：依赖注入的桥梁</a></h4>
<p>整个系统的关键在于 <code>index.js</code> 的 <code>useEffect</code> 中。</p>
<ul>
<li>
<p><strong>G6 独立王国</strong>：G6 作为一个 Canvas 库，它不认识 React 的 State 或 Context。</p>
</li>
<li>
<p><strong>依赖注入</strong>：通过 <code>registerEAMapBehavior(addUnit)</code>，我们将 React 内部定义的函数（拥有访问 API 能力、修改 React 状态能力的函数）传递给了 G6 的配置层。</p>
</li>
<li>
<p><strong>结果</strong>：当 G6 内部捕获到点击事件时，它实际上是在执行 React 组件上下文中的代码。</p>
</li>
</ul>
<h4 id="2-渲染机制canvas-图形组装-draw"><a href="#2-渲染机制canvas-图形组装-draw">2. 渲染机制：Canvas 图形组装 (<code>draw</code>)</a></h4>
<p>在 <code>EAMapUtil.js</code> 中，节点渲染不是写 HTML，而是像搭积木一样：</p>
<ul>
<li>
<p><strong>Group（组）</strong>：每个节点是一个 <code>Group</code>，相当于 HTML 的 <code>div</code>。</p>
</li>
<li>
<p><strong>Shape（图形）</strong>：在组里画 <code>rect</code>（卡片背景）、<code>text</code>（文字）、<code>circle</code>（按钮背景）。</p>
</li>
<li>
<p><strong>命名查找</strong>：给图形设置 <code>name</code> 属性至关重要（如 <code>operateGroupWhenExpanded</code>）。因为 Canvas 无法像 DOM 那样用 <code>querySelector</code>，G6 依赖 <code>group.find(item => item.get('name') === 'xxx')</code> 来找到特定的图形进行操作（比如改变颜色、隐藏）。</p>
</li>
</ul>
<h4 id="3-状态管理命令式-vs-声明式"><a href="#3-状态管理命令式-vs-声明式">3. 状态管理：命令式 vs 声明式</a></h4>
<ul>
<li>
<p><strong>React (声明式)</strong>：我们习惯 <code>setState({ visible: true })</code> 然后 React 自动重绘。</p>
</li>
<li>
<p><strong>G6 (命令式)</strong>：在 Canvas 里，改变状态需要手动操作。</p>
<ul>
<li>
<p><strong>步骤 1</strong>：React 调用 <code>graph.setItemState(node, 'Expanded', true)</code>。</p>
</li>
<li>
<p><strong>步骤 2</strong>：G6 触发 <code>EAMapUtil.js</code> 里的 <code>setState</code> 方法。</p>
</li>
<li>
<p>步骤 3：在 setState 里，我们需要手动写逻辑：“找到那个加号圆圈，调用 .hide() 方法”。</p>
<p>这更像原本的 jQuery 或原生 DOM 操作逻辑。</p>
</li>
</ul>
</li>
</ul>
<h4 id="4-数据流转mutable-model可变模型"><a href="#4-数据流转mutable-model可变模型">4. 数据流转：Mutable Model（可变模型）</a></h4>
<p>React 开发者通常习惯 Immutable（不可变）数据，但在 G6 中，操作通常是 Mutable（可变）的。</p>
<ul>
<li>
<p><strong>流程</strong>：<code>addUnit</code> 函数中，我们直接获取了节点的数据模型 <code>EAItem.getModel()</code>，然后直接执行了 <code>currentEAItem.children.push(...)</code>。</p>
</li>
<li>
<p><strong>同步</strong>：修改了 JS 对象后，G6 并不知道数据变了。必须调用 <code>graph.updateChildren</code> 或 <code>graph.changeData</code>，G6 才会根据新的数据模型重新计算布局并重绘。</p>
</li>
</ul>
<h4 id="5-交互控制事件代理"><a href="#5-交互控制事件代理">5. 交互控制：事件代理</a></h4>
<p>代码中通过 <code>registerBehavior</code> 实现了事件代理。</p>
<ul>
<li>
<p>我们不给每个“加号”绑定 click 事件（那样太重了）。</p>
</li>
<li>
<p>我们监听整个画布或节点的 <code>node:click</code>。</p>
</li>
<li>
<p>通过 <code>target.get("name")</code> 或 <code>target.get("parent").get("name")</code> 来判断用户到底点在了卡片的哪个位置（是点在空白处选中，还是点在加号上展开）。这就是所谓的“命中检测”。</p>
</li>
</ul>

<p>这段代码的核心是<strong>利用 G6 强大的图渲染能力，结合 React 的业务逻辑处理能力</strong>。</p>

<li>
<p><strong>EAMapUtil.js</strong> 是<strong>皮肤</strong>（定义长相）和<strong>神经末梢</strong>（感知点击）。</p>
</li>
<li>
<p>index.js 是大脑（决定数据怎么加载、何时更新）。</p>
<p>两者通过配置回调连接，数据变更通过直接修改 Model + 显式通知 Graph 更新来完成。</p>
