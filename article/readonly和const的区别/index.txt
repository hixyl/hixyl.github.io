<h2 id="2-深入理解-const"><a href="#2-深入理解-const">2. 深入理解 <code>const</code></a></h2>
<p><code>const</code> 是 JavaScript 的原生关键字。它保证的是<strong>变量名和内存地址之间的绑定关系</strong>不可变。</p>
<ul>
<li>
<p><strong>基本类型：</strong> 值确实不可变（如数字、字符串）。</p>
</li>
<li>
<p><strong>引用类型（对象/数组）：</strong> <strong>内容是可变的</strong>，只是不能把变量重新指向另一个对象。</p>
</li>
</ul>
<p>TypeScript</p>
<pre><code class="hljs language-ts"><span class="hljs-keyword">const</span> user = { <span class="hljs-attr">name</span>: <span class="hljs-string">"Alice"</span> };

user.<span class="hljs-property">name</span> = <span class="hljs-string">"Bob"</span>; <span class="hljs-comment">// ✅ 允许！因为 user 依然指向同一个对象</span>
<span class="hljs-comment">// user = { name: "Charlie" }; // ❌ 报错：无法分配到常数</span>
</code></pre>
<hr>
<h2 id="3-深入理解-readonly"><a href="#3-深入理解-readonly">3. 深入理解 <code>readonly</code></a></h2>
<p><code>readonly</code> 是 TypeScript 提供的类型系统修饰符。它更像是一个“只读协议”，告诉编译器：“这个属性一旦确定，别让任何人动它”。</p>
<ul>
<li>
<p><strong>灵活性：</strong> 它允许在构造函数中进行逻辑计算后再赋值，而 <code>const</code> 必须在声明那一刻写死。</p>
</li>
<li>
<p><strong>编译后消失：</strong> 编译成 JS 后，<code>readonly</code> 会消失，它不影响运行时的行为。</p>
</li>
</ul>
<p>TypeScript</p>
<pre><code class="hljs language-ts"><span class="hljs-keyword">class</span> <span class="hljs-title class_">Robot</span> {
    <span class="hljs-keyword">readonly</span> <span class="hljs-attr">serialNumber</span>: <span class="hljs-built_in">string</span>;

    <span class="hljs-title function_">constructor</span>(<span class="hljs-params"><span class="hljs-attr">id</span>: <span class="hljs-built_in">number</span></span>) {
        <span class="hljs-comment">// 允许在构造函数内初始化</span>
        <span class="hljs-variable language_">this</span>.<span class="hljs-property">serialNumber</span> = <span class="hljs-string">`SN-<span class="hljs-subst">${id}</span>-<span class="hljs-subst">${<span class="hljs-built_in">Math</span>.random()}</span>`</span>;
    }

    <span class="hljs-title function_">updateId</span>(<span class="hljs-params"></span>) {
        <span class="hljs-comment">// this.serialNumber = "NEW-ID"; // ❌ 报错：只读属性不可修改</span>
    }
}
</code></pre>

<h2 id="4-组合拳最严格的限制"><a href="#4-组合拳最严格的限制">4. 组合拳：最严格的限制</a></h2>
<p>如果你想要一个变量既不能重新指向，其内部属性也不能被修改，你需要结合两者使用：</p>
<p>TypeScript</p>
<pre><code class="hljs language-ts"><span class="hljs-keyword">interface</span> <span class="hljs-title class_">Point</span> {
    <span class="hljs-keyword">readonly</span> <span class="hljs-attr">x</span>: <span class="hljs-built_in">number</span>;
    <span class="hljs-keyword">readonly</span> <span class="hljs-attr">y</span>: <span class="hljs-built_in">number</span>;
}

<span class="hljs-keyword">const</span> <span class="hljs-attr">p1</span>: <span class="hljs-title class_">Point</span> = { <span class="hljs-attr">x</span>: <span class="hljs-number">10</span>, <span class="hljs-attr">y</span>: <span class="hljs-number">20</span> };

<span class="hljs-comment">// p1 = { x: 1, y: 2 }; // ❌ const 阻止了重新赋值</span>
<span class="hljs-comment">// p1.x = 5;            // ❌ readonly 阻止了修改属性</span>
</code></pre>
<hr>
<h2 id="5-什么时候用哪个"><a href="#5-什么时候用哪个">5. 什么时候用哪个？</a></h2>
<ul>
<li>
<p>如果你在声明一个<strong>局部的临时变量</strong>，或者一个<strong>全局的配置常量</strong>，使用 <code>const</code>。</p>
</li>
<li>
<p>如果你在定义<strong>接口、类或对象字面量</strong>，且希望外部调用者不要随意修改里面的成员，使用 <code>readonly</code>。</p>
</li>
<li>
<p>如果你正在处理一个<strong>数组且不希望它被 push/pop/splice</strong>，使用 <code>ReadonlyArray&#x3C;T></code>。</p>
</li>
</ul>
