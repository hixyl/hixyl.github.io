4:T23cd,<h3 id="题目描述"><a href="#题目描述">题目描述</a></h3>
<p>给定一个未排序的整数数组 <code>nums</code> ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。</p>
<p><strong>进阶：</strong> 你可以设计并实现时间复杂度为 $O(n)$ 的解决方案吗？</p>
<hr>
<h3 id="示例-1"><a href="#示例-1">示例 1</a></h3>
<blockquote>
<p><strong>输入：</strong> <code>nums = [100, 4, 200, 1, 3, 2]</code></p>
<p><strong>输出：</strong> <code>4</code></p>
<p><strong>解释：</strong> 最长数字连续序列是 <code>[1, 2, 3, 4]</code>。它的长度为 4。</p>
</blockquote>
<h3 id="示例-2"><a href="#示例-2">示例 2</a></h3>
<blockquote>
<p><strong>输入：</strong> <code>nums = [0, 3, 7, 2, 5, 8, 4, 6, 0, 1]</code></p>
<p><strong>输出：</strong> <code>9</code></p>
</blockquote>
<h3 id="示例-3"><a href="#示例-3">示例 3</a></h3>
<blockquote>
<p><strong>输入：</strong> <code>nums = [1, 0, 1, 2]</code></p>
<p><strong>输出：</strong> <code>3</code></p>
<p><strong>解释：</strong> 即使数组中有重复数字，连续序列依然按照数值递增计算（如 0, 1, 2）。</p>
</blockquote>
<hr>
<h3 id="提示"><a href="#提示">提示</a></h3>
<ul>
<li>
<p>$0 \le \text{nums.length} \le 10^5$</p>
</li>
<li>
<p>$-10^9 \le \text{nums}[i] \le 10^9$</p>
</li>
</ul>
<hr>
<h3 id="相关信息"><a href="#相关信息">相关信息</a></h3>
<ul>
<li>
<p><strong>标签：</strong> 数组 (Array)、哈希表 (Hash Table)、并查集 (Union Find)</p>
</li>
<li>
<p><strong>难度：</strong> 中等 (Medium)</p>
</li>
<li>
<p><strong>常见面试公司：</strong> Facebook, Google, Amazon, ByteDance</p>
</li>
</ul>
<hr>
<p>这个问题要求在 $O(n)$ 时间复杂度内找到未排序数组中最长的连续序列。解决这个问题的核心难点在于：<strong>如何在不排序（排序通常需要 $O(n \log n)$）的情况下，快速识别出哪些数字可以凑成连续的一串。</strong></p>
<h3 id="核心思路哈希集合-set-与序列起点判断"><a href="#核心思路哈希集合-set-与序列起点判断">核心思路：哈希集合 (Set) 与“序列起点”判断</a></h3>
<p>由于我们需要 $O(n)$ 的复杂度，这意味着我们最多只能对数组进行常数次的遍历。使用 <strong>哈希表（在 JS 中是 <code>Set</code>）</strong> 是最理想的选择，因为它能提供 $O(1)$ 的查找效率。</p>
<h4 id="1-为什么不能直接遍历"><a href="#1-为什么不能直接遍历">1. 为什么不能直接遍历？</a></h4>
<p>如果我们对每个数字都向后查找是否存在 <code>num + 1</code>, <code>num + 2</code>... 那么对于像 <code>[1, 2, 3, 4, 5]</code> 这样的数组，我们在处理 1 的时候会查一遍，处理 2 的时候又会查一遍，导致效率退化到 $O(n^2)$。</p>
<h4 id="2-关键优化只从起点开始计算"><a href="#2-关键优化只从起点开始计算">2. 关键优化：只从“起点”开始计算</a></h4>
<p>一个连续序列的<strong>起点</strong>有一个特征：<strong>它的前一个数字（<code>num - 1</code>）不在数组中。</strong></p>
<ul>
<li>
<p>如果 <code>num - 1</code> 在数组中，说明当前的 <code>num</code> 只是某个序列的中间部分，我们直接跳过，不做重复计算。</p>
</li>
<li>
<p>如果 <code>num - 1</code> 不在数组中，说明 <code>num</code> 是一个序列的开头。此时，我们才开始循环检查 <code>num + 1</code>, <code>num + 2</code>... 直到序列断开。</p>
</li>
</ul>
<p>这样保证了<strong>每个元素最多只会被访问两次</strong>（一次在 Set 中被检查，一次在作为某个序列的一部分被计数），从而达到 $O(n)$。</p>
<hr>
<h3 id="javascript-代码实现"><a href="#javascript-代码实现">JavaScript 代码实现</a></h3>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-comment">/**
 * <span class="hljs-doctag">@param</span> {<span class="hljs-type">number[]</span>} <span class="hljs-variable">nums</span>
 * <span class="hljs-doctag">@return</span> {<span class="hljs-type">number</span>}
 */</span>
<span class="hljs-keyword">var</span> longestConsecutive = <span class="hljs-keyword">function</span>(<span class="hljs-params">nums</span>) {
    <span class="hljs-keyword">if</span> (nums.<span class="hljs-property">length</span> === <span class="hljs-number">0</span>) <span class="hljs-keyword">return</span> <span class="hljs-number">0</span>;

    <span class="hljs-comment">// 1. 将所有数字存入 Set，去除重复值并提供 O(1) 查找</span>
    <span class="hljs-keyword">const</span> numSet = <span class="hljs-keyword">new</span> <span class="hljs-title class_">Set</span>(nums);
    <span class="hljs-keyword">let</span> longestStreak = <span class="hljs-number">0</span>;

    <span class="hljs-comment">// 2. 遍历 Set 中的每一个数字</span>
    <span class="hljs-keyword">for</span> (<span class="hljs-keyword">const</span> num <span class="hljs-keyword">of</span> numSet) {
        
        <span class="hljs-comment">// 3. 判断当前数字是否是一个序列的“起点”</span>
        <span class="hljs-comment">// 只有当 num - 1 不存在时，才说明 num 是起点</span>
        <span class="hljs-keyword">if</span> (!numSet.<span class="hljs-title function_">has</span>(num - <span class="hljs-number">1</span>)) {
            <span class="hljs-keyword">let</span> currentNum = num;
            <span class="hljs-keyword">let</span> currentStreak = <span class="hljs-number">1</span>;

            <span class="hljs-comment">// 4. 从起点开始，不断查找下一个连续数字</span>
            <span class="hljs-keyword">while</span> (numSet.<span class="hljs-title function_">has</span>(currentNum + <span class="hljs-number">1</span>)) {
                currentNum += <span class="hljs-number">1</span>;
                currentStreak += <span class="hljs-number">1</span>;
            }

            <span class="hljs-comment">// 5. 更新全局最长长度</span>
            longestStreak = <span class="hljs-title class_">Math</span>.<span class="hljs-title function_">max</span>(longestStreak, currentStreak);
        }
    }

    <span class="hljs-keyword">return</span> longestStreak;

<h3 id="逻辑拆解与示例分析"><a href="#逻辑拆解与示例分析">逻辑拆解与示例分析</a></h3>
<p>以 <code>nums = [100, 4, 200, 1, 3, 2]</code> 为例：</p>
<ol>
<li>
<p><strong>放入 Set</strong>：得到 <code>{100, 4, 200, 1, 3, 2}</code>。</p>
</li>
<li>
<p><strong>遍历到 100</strong>：</p>
<ul>
<li>
<p>检查 <code>99</code> 是否在 Set 中？不在。</p>
</li>
<li>
<p><code>100</code> 是起点。往后找 <code>101</code>，不在。</p>
</li>
<li>
<p>当前长度 <code>1</code>，<code>longestStreak = 1</code>。</p>
</li>
</ul>
</li>
<li>
<p><strong>遍历到 4</strong>：</p>
<ul>
<li>
<p>检查 <code>3</code> 是否在 Set 中？<strong>在</strong>。</p>
</li>
<li>
<p>说明 <code>4</code> 肯定不是起点（它属于以 1 或更小数字开头的序列），<strong>直接跳过</strong>。</p>
</li>
</ul>
</li>
<li>
<p><strong>遍历到 200</strong>：</p>
<ul>
<li>
<p>检查 <code>199</code> 不在。<code>200</code> 是起点。往后找 <code>201</code> 不在。</p>
</li>
<li>
<p><code>longestStreak</code> 仍为 <code>1</code>。</p>
</li>
</ul>
</li>
<li>
<p><strong>遍历到 1</strong>：</p>
<ul>
<li>
<p>检查 <code>0</code> 是否在 Set 中？不在。</p>
</li>
<li>
<p><code>1</code> 是起点！开启 <code>while</code> 循环：</p>
<ul>
<li>
<p>找到 <code>2</code>，长度变 2；</p>
</li>
<li>
<p>找到 <code>3</code>，长度变 3；</p>
</li>
<li>
<p>找到 <code>4</code>，长度变 4；</p>
</li>
<li>
<p>找到 <code>5</code>？不在，循环结束。</p>
</li>
</ul>
</li>
<li>
<p>更新 <code>longestStreak = 4</code>。</p>
</li>
</ul>
</li>
<li>
<p><strong>后续遍历</strong>（如 3, 2）：都会因为其前驱数字存在而直接跳过。</p>
</li>
</ol>
<h3 id="复杂度分析"><a href="#复杂度分析">复杂度分析</a></h3>
<ul>
<li>
<p><strong>时间复杂度：$O(n)$</strong></p>
<ul>
<li>
<p>建立 <code>Set</code> 需要 $O(n)$。</p>
</li>
<li>
<p>虽然代码里有嵌套循环，但 <code>if (!numSet.has(num - 1))</code> 确保了只有序列的第一个元素会触发内部的 <code>while</code> 循环。数组中的每个元素在整个过程中最多被处理两次。</p>
</li>
</ul>
</li>
<li>
<p><strong>空间复杂度：$O(n)$</strong></p>
<ul>
<li>我们需要一个 <code>Set</code> 来存储所有的数字。</li>
</ul>
</li>
</ul>
