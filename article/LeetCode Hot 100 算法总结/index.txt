<h3 id="核心算法模式总结"><a href="#核心算法模式总结">核心算法模式总结</a></h3>
<p>可以将 Hot 100 的题目大致归为以下几个核心类别。掌握了它们，你就掌握了通关的钥匙。</p>
<h4 id="1-双指针-two-pointers"><a href="#1-双指针-two-pointers">1. 双指针 (Two Pointers)</a></h4>
<p>双指针是一种极其高效的技巧，通过维护两个指针在数组或链表中的移动，来解决问题。它能巧妙地将一些 O(N2) 的暴力解法优化到 O(N)。</p>
<ul>
<li>
<p><strong>核心思想</strong>: 用两个指针（通常命名为 <code>left</code> 和 <code>right</code>，或 <code>slow</code> 和 <code>fast</code>）从不同位置（同向、反向或分离）开始，根据特定条件移动它们，直到它们相遇或交错，从而完成任务。</p>
</li>
<li>
<p><strong>常见模式</strong>:</p>
<ul>
<li>
<p><strong>快慢指针</strong>: 主要用于解决链表问题，如判断环、寻找中点、寻找倒数第K个节点。一个指针每次走一步（slow），另一个指针每次走两步（fast）。</p>
</li>
<li>
<p><strong>左右指针</strong>: 主要用于有序数组，通过从两端向中间逼近来寻找目标和或满足条件的区间。</p>
</li>
<li>
<p><strong>滑动窗口</strong>: 见下一个类别，它是双指针思想的一种特例和延伸。</p>
</li>
</ul>
</li>
<li>
<p><strong>典型例题</strong>:</p>
<ul>
<li>
<p><a href="https://leetcode.cn/problems/container-with-most-water/">11. 盛最多水的容器</a>: 典型的左右指针，从两端开始，每次移动指向较短板子的指针。</p>
</li>
<li>
<p><a href="https://leetcode.cn/problems/linked-list-cycle-ii/">142. 环形链表 II</a>: 快慢指针的经典应用，通过数学推导确定环的入口。</p>
</li>
<li>
<p><a href="https://leetcode.cn/problems/move-zeroes/">283. 移动零</a>: 快慢指针（或同向双指针），一个指针（slow）指向新数组的末尾，另一个指针（fast）遍历原数组。</p>
</li>
</ul>
</li>
<li>
<p><strong>JavaScript 通用模板 (左右指针)</strong>:</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">function</span> <span class="hljs-title function_">twoPointers</span>(<span class="hljs-params">arr</span>) {
    <span class="hljs-keyword">let</span> left = <span class="hljs-number">0</span>;
    <span class="hljs-keyword">let</span> right = arr.<span class="hljs-property">length</span> - <span class="hljs-number">1</span>;
    <span class="hljs-keyword">let</span> result;

    <span class="hljs-keyword">while</span> (left &#x3C; right) {
        <span class="hljs-comment">// 根据 arr[left] 和 arr[right] 的关系进行逻辑判断</span>
        <span class="hljs-keyword">if</span> (<span class="hljs-title function_">condition</span>(arr[left], arr[right])) {
            <span class="hljs-comment">// 移动其中一个指针</span>
            left++;
        } <span class="hljs-keyword">else</span> {
            right--;
        }
    }
    <span class="hljs-keyword">return</span> result;
}
</code></pre>
</li>
</ul>
<h4 id="2-滑动窗口-sliding-window"><a href="#2-滑动窗口-sliding-window">2. 滑动窗口 (Sliding Window)</a></h4>
<p>滑动窗口是双指针思想的集大成者，专门用于解决数组或字符串中的“连续子数组”或“连续子串”问题，如寻找最长的、最短的、满足特定条件的子数组/子串。</p>
<ul>
<li>
<p><strong>核心思想</strong>: 维护一个由 <code>left</code> 和 <code>right</code> 指针界定的“窗口”。<code>right</code> 指针负责扩大窗口，纳入新元素；<code>left</code> 指针负责在窗口不满足条件时收缩窗口，排除旧元素。整个过程 <code>left</code> 和 <code>right</code> 都只向右移动，保证了 O(N) 的时间复杂度。</p>
</li>
<li>
<p><strong>典型例题</strong>:</p>
<ul>
<li>
<p><a href="https://leetcode.cn/problems/longest-substring-without-repeating-characters/">3. 无重复字符的最长子串</a>: 窗口内维持没有重复的字符。</p>
</li>
<li>
<p><a href="https://leetcode.cn/problems/minimum-window-substring/">76. 最小覆盖子串</a>: 窗口需要覆盖目标字符串 <code>t</code> 中的所有字符。</p>
</li>
<li>
<p><a href="https://leetcode.cn/problems/find-all-anagrams-in-a-string/">438. 找到字符串中所有字母异位词</a>: 维持一个固定大小的窗口，判断窗口内的字符计数是否与目标一致。</p>
</li>
</ul>
</li>
<li>
<p><strong>JavaScript 通用模板</strong>:</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">function</span> <span class="hljs-title function_">slidingWindow</span>(<span class="hljs-params">s, t</span>) {

<h4 id="3-树的遍历-tree-traversal"><a href="#3-树的遍历-tree-traversal">3. 树的遍历 (Tree Traversal)</a></h4>
<p>二叉树问题在 Hot 100 中占据了相当大的比重。其核心解法万变不离其宗，都基于深度优先搜索（DFS）和广度优先搜索（BFS）。</p>
<ul>
<li>
<p><strong>核心思想</strong>:</p>
<ul>
<li>
<p><strong>DFS (深度优先搜索)</strong>: 深入到树的某一分支的尽头，再回溯。通常用<strong>递归</strong>或<strong>栈</strong>实现。前序、中序、后序遍历是其基础。</p>
</li>
<li>
<p><strong>BFS (广度优先搜索)</strong>: 从根节点开始，一层一层地向外扩展。通常用<strong>队列</strong>实现。</p>
</li>
</ul>
</li>
<li>
<p><strong>解题思路</strong>: 绝大多数树的问题，都可以思考：</p>
<ol>
<li>
<p>我对于当前这个节点 <code>root</code>，需要做什么？</p>
</li>
<li>
<p>我需要从我的左、右子树拿到什么信息，才能完成 <code>root</code> 节点的工作？</p>
</li>
<li>
<p>我应该向上返回什么信息，以供我的父节点使用？</p>
<p>这个思考过程，本质上就是定义递归函数的签名和返回值。</p>
</li>
</ol>
</li>
<li>
<p><strong>典型例题</strong>:</p>
<ul>
<li>
<p><a href="https://leetcode.cn/problems/binary-tree-inorder-traversal/">94. 二叉树的中序遍历</a>: DFS 的基础，递归或迭代（用栈）实现。</p>
</li>
<li>
<p><a href="https://leetcode.cn/problems/binary-tree-level-order-traversal/">102. 二叉树的层序遍历</a>: BFS 的直接应用。</p>
</li>
<li>
<p><a href="https://leetcode.cn/problems/maximum-depth-of-binary-tree/">104. 二叉树的最大深度</a>: DFS 的经典应用，<code>maxDepth(root) = 1 + max(maxDepth(root.left), maxDepth(root.right))</code>。</p>
</li>
<li>

<h4 id="4-动态规划-dynamic-programming-dp"><a href="#4-动态规划-dynamic-programming-dp">4. 动态规划 (Dynamic Programming, DP)</a></h4>
<p>动态规划是 Hot 100 中的重中之重，也是很多人觉得困难的部分。但其本质有章可循。</p>
<ul>
<li>
<p><strong>核心思想</strong>: 将一个大问题分解为相互重叠的子问题，通过求解子问题来得到大问题的解。为了避免重复计算，会用一个 DP 表（通常是一维或二维数组）来存储子问题的解。</p>
</li>
<li>
<p><strong>解题五步曲</strong>:</p>
<ol>
<li>
<p><strong>定义 <code>dp</code> 数组的含义</strong>: 这是最关键的一步。例如，<code>dp[i]</code> 表示以 <code>nums[i]</code> 结尾的最长递增子序列的长度。</p>
</li>
<li>
<p><strong>找出状态转移方程</strong>: 思考 <code>dp[i]</code> 如何由之前的 <code>dp[j]</code> (其中 <code>j &#x3C; i</code>) 推导出来。这是核心。</p>
</li>
<li>
<p><strong>确定 <code>dp</code> 数组的初始化</strong>: <code>dp[0]</code> 的值是什么？或者整个数组的初始值是什么？</p>
</li>
<li>
<p><strong>确定遍历顺序</strong>: 是从前到后，还是从后到前？</p>
</li>
<li>
<p><strong>确定最终返回值</strong>: 最终结果是 <code>dp[n-1]</code> 还是 <code>max(dp)</code>？</p>
</li>
</ol>
</li>
<li>
<p><strong>典型例题</strong>:</p>
<ul>
<li>
<p><a href="https://leetcode.cn/problems/maximum-subarray-sum/">53. 最大子数组和</a>: <code>dp[i]</code> 定义为以 <code>nums[i]</code> 结尾的最大子数组和。状态转移：<code>dp[i] = max(nums[i], nums[i] + dp[i-1])</code>。</p>
</li>
<li>
<p><a href="https://leetcode.cn/problems/longest-increasing-subsequence/">300. 最长递增子序列</a>: <code>dp[i]</code> 定义为以 <code>nums[i]</code> 结尾的最长递增子序列的长度。</p>
</li>
<li>
<p><a href="https://leetcode.cn/problems/climbing-stairs/">70. 爬楼梯</a>: <code>dp[i]</code> 定义为爬到第 <code>i</code> 阶的方法数。状态转移：<code>dp[i] = dp[i-1] + dp[i-2]</code>。</p>
</li>
<li>
<p><a href="https://leetcode.cn/problems/longest-palindromic-substring/">5. 最长回文子串</a>: <code>dp[i][j]</code> 定义为字符串 <code>s</code> 从索引 <code>i</code> 到 <code>j</code> 的子串是否是回文串。</p>
</li>
</ul>
</li>
<li>
<p><strong>JavaScript 通用模板 (一维 DP)</strong>:</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">function</span> <span class="hljs-title function_">dynamicProgramming</span>(<span class="hljs-params">input</span>) {
    <span class="hljs-keyword">const</span> n = input.<span class="hljs-property">length</span>;
    <span class="hljs-comment">// 1. 定义 dp 数组</span>
    <span class="hljs-keyword">const</span> dp = <span class="hljs-keyword">new</span> <span class="hljs-title class_">Array</span>(n).<span class="hljs-title function_">fill</span>(initial_value);

    <span class="hljs-comment">// 3. 初始化</span>
    dp[<span class="hljs-number">0</span>] = <span class="hljs-comment">/* ... */</span>;

    <span class="hljs-comment">// 4. 遍历</span>
    <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> i = <span class="hljs-number">1</span>; i &#x3C; n; i++) {
        <span class="hljs-comment">// 2. 状态转移方程</span>
        <span class="hljs-comment">// 通常会再嵌套一个循环来寻找之前的状态</span>
        <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> j = <span class="hljs-number">0</span>; j &#x3C; i; j++) {
            dp[i] = <span class="hljs-comment">/* 根据 dp[j] 和 input[i] 计算 */</span>;
        }
    }

    <span class="hljs-comment">// 5. 返回值</span>
    <span class="hljs-keyword">return</span> <span class="hljs-comment">/* dp[n-1] 或 Math.max(...dp) */</span>;
}

</li>
</ul>
<h4 id="5-回溯算法-backtracking"><a href="#5-回溯算法-backtracking">5. 回溯算法 (Backtracking)</a></h4>
<p>回溯法是暴力搜索的优化，可以看作是“有纪律的”暴力枚举。它在 DFS 的框架上，增加了“状态重置”（也叫“剪枝”）的操作。非常适合解决组合、排列、子集、棋盘等问题。</p>
<ul>
<li>
<p><strong>核心思想</strong>: 像在一棵“解空间树”上进行 DFS。从根节点出发，尝试一个选择，进入下一层决策；如果发现这个选择走不通（或已经走到头），就<strong>撤销</strong>这个选择（回溯），尝试同一层的其他选择。</p>
</li>
<li>
<p><strong>解题框架</strong>:</p>
<ol>
<li>
<p><strong>函数签名</strong>: <code>backtrack(路径, 选择列表)</code></p>
</li>
<li>
<p><strong>结束条件</strong>: 满足什么条件时，将“路径”加入结果集，并返回。</p>
</li>
<li>
<p><strong>循环和选择</strong>: 遍历当前可用的“选择列表”。</p>
</li>
<li>
<p><strong>做选择</strong>: 将当前选择加入“路径”。</p>
</li>
<li>
<p><strong>递归</strong>: <code>backtrack(新路径, 新选择列表)</code>。</p>
</li>
<li>
<p><strong>撤销选择</strong>: 将刚才加入“路径”的选择移除，恢复现场，以便进行下一次循环尝试。</p>
</li>
</ol>
</li>
<li>
<p><strong>典型例题</strong>:</p>
<ul>
<li>
<p><a href="https://leetcode.cn/problems/permutations/">46. 全排列</a>: 数字不能重复使用。</p>
</li>
<li>
<p><a href="https://leetcode.cn/problems/subsets/">78. 子集</a>: 经典的求所有子集问题。</p>
</li>
<li>
<p><a href="https://leetcode.cn/problems/generate-parentheses/">22. 括号生成</a>: 通过控制左右括号的数量进行剪枝。</p>
</li>
<li>
<p><a href="https://leetcode.cn/problems/combination-sum/">39. 组合总和</a>: 数字可以重复使用。</p>
</li>
</ul>
</li>
<li>
<p><strong>JavaScript 通用模板</strong>:</p>
<p>JavaScript</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">function</span> <span class="hljs-title function_">backtracking</span>(<span class="hljs-params">nums</span>) {
    <span class="hljs-keyword">const</span> result = [];
    <span class="hljs-keyword">const</span> path = []; <span class="hljs-comment">// 记录当前路径</span>

    <span class="hljs-keyword">function</span> <span class="hljs-title function_">backtrack</span>(<span class="hljs-params">startIndex</span>) { <span class="hljs-comment">// startIndex 用于去重和避免重复组合</span>
        <span class="hljs-comment">// 收集结果，注意需要拷贝 path</span>
        <span class="hljs-comment">// result.push([...path]); // 如果是求子集，在这里收集</span>

        <span class="hljs-comment">// 终止条件</span>
        <span class="hljs-keyword">if</span> (path.<span class="hljs-property">length</span> === <span class="hljs-comment">/* 目标长度 or 其他条件 */</span>) {
            result.<span class="hljs-title function_">push</span>([...path]);
            <span class="hljs-keyword">return</span>;
        }

        <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> i = startIndex; i &#x3C; nums.<span class="hljs-property">length</span>; i++) {
            <span class="hljs-comment">// 做选择</span>
            path.<span class="hljs-title function_">push</span>(nums[i]);

            <span class="hljs-title function_">backtrack</span>(i + <span class="hljs-number">1</span>); <span class="hljs-comment">// i+1 表示数字不可重复使用</span>
            <span class="hljs-comment">// 撤销选择</span>
            path.<span class="hljs-title function_">pop</span>();
        }
    }

    <span class="hljs-title function_">backtrack</span>(<span class="hljs-number">0</span>);
    <span class="hljs-keyword">return</span> result;
}
</code></pre>
</li>
</ul>
<h3 id="学习路径建议"><a href="#学习路径建议">学习路径建议</a></h3>
<p>我认为最高效的学习路径是：</p>
<ol>
<li>
<p><strong>数组/链表基础 -> 双指针/滑动窗口</strong>: 这是最基础且效果最立竿见影的。先从数组和链表的基本操作开始，然后迅速切入双指针和滑动窗口，解决 Hot 100 中近 20% 的题目。</p>
</li>
<li>
<p><strong>哈希表 (Map/Set)</strong>: 哈希表是优化时间复杂度的神器，是几乎所有算法模式的“润滑剂”。例如，在滑动窗口中记录字符数，在两数之和中寻找目标值。务必熟练使用 <code>Map</code> 和 <code>Set</code>。</p>
</li>
<li>
<p><strong>树 -> DFS/BFS</strong>: 掌握树的两种遍历方式，并理解递归的本质。练习 Hot 100 中所有树的题目，你会发现它们的解法高度统一。</p>
</li>
<li>
<p><strong>回溯算法</strong>: 在理解了 DFS 的基础上学习回溯，你会发现它就是 DFS 的一个标准框架。专门练习组合、排列、子集问题。</p>
</li>
<li>
<p><strong>动态规划</strong>: 这是攻坚阶段。不要急于求成，从最简单的爬楼梯、最大子数组和开始，真正理解“五步曲”，然后逐步挑战更复杂的二维 DP 和背包问题。</p>
</li>
</ol>
<p><strong>我的最终建议是：不要孤立地刷题，而要带着“模式识别”的眼光去刷。</strong></p>
<p>每当你看到一道题，先不要急着写代码，而是问自己：</p>
<ul>
<li>
<p>这道题的输入和输出是什么？</p>
</li>
<li>
<p>它看起来像我学过的哪种模式？是求连续子数组问题吗（滑动窗口）？是求最优解吗（动态规划）？是求所有可能性吗（回溯）？</p>
</li>
<li>
<p>如果套用这个模式的通用模板，我需要修改哪些部分？</p>
</li>
</ul>
